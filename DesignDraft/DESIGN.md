# DESIGN.md

# XFVerse UI Design Guide

本文件用于说明 XFVerse UI 重构的整体视觉方向、设计原则、主题规则、组件风格和 Codex 实现时必须遵守的 UI 约束。

XFVerse 是一个基于 WPF + .NET 的 AI 智能影音库桌面软件，核心功能包括 WebDAV 影音库管理、影片识别、播放器、观影统计、用户画像和 AI 推荐。

---

## 1. Product Identity

XFVerse 的产品气质是：

- 影音感
- 柔和感
- 现代桌面软件
- 智能影音库
- 个人媒体空间
- 商业化、干净、克制

界面不追求强烈科技感，也不做赛博朋克、强霓虹、过度玻璃拟态或大面积炫光。

整体视觉应服务于：

- 影片浏览
- 影片管理
- 播放体验
- 观影洞察
- AI 个性化推荐

---

## 2. Design Principles

### 必须遵守

- 页面结构、滚动区域、组件位置、信息层级以无边记设计稿为准
- 不擅自改变页面结构
- 不擅自减少功能区
- 不擅自调整组件顺序
- 只允许修正明显不合理的间距、对齐和视觉层级
- 基础控件优先使用 WPF UI
- 支持浅色 / 深色双主题
- 颜色、字体、间距、样式统一放入 ResourceDictionary
- 不在页面中硬编码颜色
- 新 UI 可以新建一套 Views / Controls，不强行沿用旧 UI 文件

### 视觉关键词

- 干净
- 克制
- 柔和
- 影音感
- 清晰层级
- 中性色为主
- 灰粉 / 浅粉点缀
- 低饱和状态色
- 信息密度根据页面类型调整

### 禁止方向

- 赛博朋克
- 强霓虹
- 过度玻璃拟态
- 大面积粉色背景
- 所有图标都使用粉色
- 所有按钮都使用主色
- 复杂拟物风
- 强烈发光效果
- 每个页面单独设计一套视觉规则

## 3. Base UI System

基础 UI 库使用：

- WPF UI

使用原则：

- 按钮、输入框、下拉框、导航、菜单、弹窗、滚动条、基础主题优先使用 WPF UI 默认样式
- 不从零重写基础控件样式
- WPF UI 默认图标优先使用
- 不足部分再补 Fluent / IconPack
- 所有自定义样式和颜色必须放入统一资源文件
- 不允许每个页面单独硬编码颜色和样式

---

## 4. Theme System

XFVerse 支持浅色 / 深色双主题。

### 浅色主题

浅色主题方向：

- 页面背景：淡粉白
- 基础卡片：白色
- 重点卡片：轻微渐变
- 文字：深灰 / 深蓝灰
- 整体对比度：中等，清楚但不刺眼

浅色主题原则：

- 背景可以带非常轻微的粉白氛围
- 普通内容区保持干净
- 卡片之间通过轻微阴影、边框和留白区分层级
- 不使用大面积高饱和粉色
- 重点卡片可以使用轻微粉白 / 紫粉渐变点缀

### 深色主题

深色主题方向：

- 主界面：深灰
- 播放器：黑灰
- 普通页面卡片：深灰卡片
- 播放器浮层：半透明黑色 / 深灰浮层
- 主色在深色下可以稍微更亮
- 主色仅用于按钮、进度条、选中态等重点位置
- 整体对比度：中等，清楚但不刺眼

深色主题原则：

- 不使用纯黑作为主界面背景
- 深色页面主要通过背景层级、边框和透明度区分结构
- 深色主题少用阴影，多用边框和层级
- 播放器区域可比普通页面更深，以保证沉浸感

## 5. Color Roles

### 主色

主色方向：

- 灰粉为主
- 浅粉作为关键点缀
- 整体以中性色承托

主色使用位置：

- 主按钮
- 当前选中的导航项
- 当前选中的 Tab
- 开关开启状态
- 播放进度条
- 普通音量条
- 当前播放源高亮
- AI 功能局部点缀
- 重要高亮状态

主色不应大量用于：

- 页面大面积背景
- 普通正文
- 所有图标
- 所有标签
- 所有按钮
- 错误 / 警告状态

## 6. Business Status Colors

以下为颜色归属规则，具体色值由 WPF UI Accent 和资源文件统一定义。

### 影片状态

- 已看：蓝色
- 未看：淡灰边框，不填充
- 想看：浅粉色
- 喜爱：浅粉色，但比“想看”更深
- 不想看：低饱和橙红，不刺眼

### 播放源状态

- 默认播放源：低饱和绿色
- 当前播放源：主色灰粉高亮 + 轻微边框
- 未识别影片 / 播放源：灰色为主，橙色小提示
- 识别失败：低饱和红，不要强警报感

### AI 状态

- AI 推荐 / AI 画像：灰粉 + 柔和紫 + 少量星光 / 渐变点缀

### 扫描 / 系统状态

- 扫描中：蓝色或主色轻提示
- 扫描成功：低饱和绿色
- 扫描失败：低饱和红
- 缓存警告：低饱和橙色

---

## 7. Movie Tag Colors

影片标签分为三类：

- 类型标签
- 情绪标签
- 场景标签

颜色规则：

- 类型标签：低饱和灰蓝
- 情绪标签：以灰粉 / 粉紫为主，但根据情绪变化
- 场景标签：低饱和紫灰

情绪标签补充规则：

- 积极情绪和消极情绪需要有区分
- 情绪标签可以在灰粉 / 粉紫体系内变化
- 不要偏离整体灰粉柔和风格
- 不要使用过高饱和度颜色

标签样式：

- 普通标签：淡色底 + 彩色文字
- 重要状态标签：实心浅色底
- 标签整体轻量、圆润、可读
- 不同标签类型需要保持可区分性

## 9. Density, Radius, Elevation

### UI 密度

整体密度：

- 核心内容：标准密度
- 详情页 / 列表页：偏紧凑
- 用户画像 / 观影洞察页：偏宽松

执行原则：

- 信息密集页面不要过度留白
- AI 画像和洞察页面需要更强的呼吸感
- 播放源列表、扫描日志、结果列表需要保证信息读取效率
- 具体布局以无边记设计为准

### 圆角

圆角规则：

- 基础圆角跟随 WPF UI 默认
- 不额外定义一套完全独立的圆角系统
- 特殊业务组件可在 WPF UI 基础上微调

执行原则：

- 普通按钮、输入框、菜单、弹窗跟随 WPF UI
- 业务卡片、影片卡片、用户画像卡可以适当更圆润
- 标签保持胶囊感或轻圆角
- 不同页面不要出现明显冲突的圆角风格

### 阴影和边框

浅色主题：

- 使用轻微阴影
- 使用浅色边框辅助分层
- 卡片浮起感要克制

深色主题：

- 少用阴影
- 多用边框、透明度、背景层级
- 不做强烈发光效果

整体原则：

- 阴影不可过重
- 不做强烈拟物风
- 浮层、弹窗、菜单可以有更明显的层级
- 普通卡片保持轻量

## 10. Layout and Scrolling

页面布局：

- 页面结构完全按无边记设计执行
- 不重新设计页面结构
- 不擅自调整组件顺序
- 不擅自减少功能区

滚动规则：

- 无边记中有滚动条的区域，即代表该区域可滚动
- 未标记滚动的区域不要擅自改为滚动区域
- 长页面、列表、日志、播放源列表等滚动行为以无边记为准

导航规则：

- 主导航形式按无边记设计
- 基础导航控件和样式使用 WPF UI 导航控件
- 导航选中态使用主色灰粉 / 浅粉点缀
- 展开导航栏状态下，导航栏右上角是关闭导航栏按钮
- 关闭导航栏状态下，页面左上角是展开导航栏按钮
- 导航栏展开 / 关闭按钮不是 App Logo
- App Logo 只用于侧边栏品牌区、关于 XFVerse 行、应用图标、任务栏图标等品牌位置
- 影片详情页左上角是返回按钮，不是 App Logo
- 播放器左上角是关闭播放器 / 返回按钮，不是 App Logo

## 11. Cards and Surfaces

卡片结构：

- 跟随无边记设计
- 只统一视觉，不重新定义结构

卡片使用原则：

- 统计页、画像页、设置页可以更卡片化
- 影片详情页按无边记结构，不重新定结构
- 扫描任务页按无边记结构
- 设置页按无边记结构
- 卡片之间保持清晰层级

卡片视觉：

- 浅色主题：白色 / 轻微渐变 / 轻阴影
- 深色主题：深灰卡片 / 边框 / 层级区分
- 重点卡片可使用轻微渐变或装饰
- 普通卡片不要过度装饰

---

## 12. Icons

图标使用规则：

- WPF UI 默认图标优先
- 不足部分再补 Fluent / IconPack
- 全软件保持统一图标风格
- 不混用多套风格差异明显的图标

图标颜色：

- 普通图标使用中性色
- 选中图标使用主色
- 状态图标使用对应语义色
- 危险图标使用低饱和红 / 橙红
- 成功图标使用低饱和绿
- 不要所有图标都使用浅粉色

## 13. Avatar

默认头像不使用图片资源。

默认头像由 WPF / XAML 生成：

- 圆形头像容器
- 偏纯色的灰粉 / 紫灰背景
- 可以有极轻微渐变，但渐变不要明显
- 居中显示用户名首字母
- 如果用户名为空，则显示 WPF UI 默认用户图标

要求：

- 中性
- 不区分性别
- 不使用真人图像
- 不使用 AI 人物头像
- 不过度卡通
- 适配深色和浅色主题

## 15. Empty States

空状态不使用独立插画图片。

空状态统一封装为 EmptyState 组件，包含：

- WPF UI 默认图标
- 标题文案
- 说明文案
- 可选操作按钮
- 少量灰粉 / 紫灰渐变背景装饰

适用场景：

- 无影片
- 无搜索结果
- 无扫描记录
- 无观影数据
- 用户画像数据不足
- 无播放源
- 识别结果为空
- WebDAV 未连接
- 推荐结果为空

要求：

- 文案清楚
- 操作入口明确
- 不做复杂插画
- 不使用带文字的图片
- 适配深色和浅色主题

## 16. Player UI

播放器区域：

- 永远深色沉浸式
- 控制菜单使用半透明深灰浮层
- 播放器不跟随普通页面浅色风格
- 以不遮挡影片内容为原则

播放器背景：

- 不准备播放器背景图片
- 由 WPF / XAML 生成
- 基础深灰 / 黑灰背景
- 左上角低透明灰粉光晕
- 右下角低透明紫灰光晕
- 轻微暗色渐变
- 中心区域保持干净

播放器控制层：

- 底部控制栏：半透明深灰 / 黑灰
- 播放进度条：灰粉主色
- 播放进度悬浮 / 拖动时：略微提亮
- 缓存进度：半透明浅灰
- 普通音量条 0-100%：与亮度条一致，使用浅粉 / 灰粉主色
- 音量增强 100-200%：低饱和橙色
- 音量超过 150%：橙色加深
- 亮度条：与普通音量条一致，使用浅粉 / 灰粉主色
- 播放源 / 字幕 / 音轨菜单：半透明深灰浮层

注意：

- 亮度条颜色不是音量增强颜色
- 普通音量颜色和亮度条颜色一致
- 音量增强颜色只用于超过 100% 的区域

## 17. Key Page Personalities

### 观影洞察

观影洞察包含：

- 观影统计
- 用户画像

风格规则：

- 观影统计：偏数据看板
- 用户画像：偏 AI 口味报告
- 两者保持同一视觉体系
- 可以增加趣味感
- 不要让统计页变成人格页
- 不要让画像页变成纯数据后台

观影统计关键词：

- 数据
- 分布
- 趋势
- 节奏
- 标签组合
- 清晰图表

用户画像关键词：

- AI 总结
- 口味
- 人格
- DNA
- 象限
- 代表影片
- 趣味化表达

### 影片详情页

影片详情页：

- 按无边记设计执行
- 不重新定结构
- 保持海报、影片信息、评分、标签、简介、播放源列表、操作按钮的既定层级
- 信息密度要清楚
- 播放源列表需要易读
- 操作按钮需要区分主次

重点：

- 播放默认播放源是主操作
- 观看状态按钮在“标记已看 / 标记未看”之间切换
- 偏好按钮根据观看状态在“标记想看 / 标记喜爱”之间切换
- 未看影片使用“想看”，已看影片使用“喜爱”
- 不想看 / 想看 / 喜爱三者互斥，任意两个不能共存
- 不想看会覆盖想看和喜爱
- 播放源相关操作要清晰但不过度抢主视觉
- 修正识别是辅助但重要操作

### 扫描任务页

扫描任务页：

- 按无边记设计执行
- 日志清楚
- 进度和状态适度可视化
- 不做过度装饰
- 适合长时间查看任务状态

重点：

- 开始扫描按钮清晰
- 扫描路径明确
- 扫描日志可滚动
- 成功、失败、错误、跳过等状态需要可区分
- 日志区域可使用更紧凑的字体和行高

### 设置页

设置页：

- 按无边记设计执行
- 使用 WPF UI 设置项 / 卡片样式统一视觉
- 设置项要清楚、稳定、可读
- 不做过度装饰

设置页重点：

- 分区清晰
- 操作反馈清楚
- 危险操作需要明确区分
- 关于按钮保持低调

## 19. Implementation Notes for Codex

实现时必须遵守：

- 不要擅自改变无边记页面布局
- 不要擅自删除功能
- 不要重新设计页面结构
- 使用 WPF UI 作为基础 UI 库
- 深色 / 浅色主题都要支持
- 基础控件优先使用 WPF UI 默认
- 业务组件单独封装
- 颜色、字体、间距、样式统一放入 ResourceDictionary
- 不要在各页面硬编码颜色
- 不要混用多套图标风格
- 保留现有后端逻辑
- 新 UI 新建一套 Views / Controls，逐步替换旧 UI
