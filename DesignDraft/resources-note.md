# XFVerse UI Resources Note

本文件用于说明 XFVerse UI 重构中需要准备的资源、图标策略、占位图策略，以及由 WPF / XAML 生成的视觉元素。

项目技术栈：

- WPF + .NET
- 基础 UI 库：WPF UI
- 支持浅色 / 深色双主题
- 页面结构以无边记设计为准
- 视觉规范以 DESIGN.md 为准

---

## 1. Resource Strategy

本项目不准备大量图片资源。

原则：

- 基础图标使用 WPF UI 默认图标
- Logo 使用 SVG 矢量资源
- 默认头像由 WPF / XAML 生成，不准备默认头像图片
- 播放器背景由 WPF / XAML 渐变生成，不准备播放器背景图片
- 空状态由 WPF UI 图标 + 文案 + 少量渐变装饰生成，不准备空状态插画图片
- 仅海报占位图使用 AI 生成图片资源
- AI / 用户画像装饰图后续看 UI 效果再决定
- 不使用带文字的图片资源，Logo 除外
- 不把按钮、输入框、卡片、图表、播放器控件做成图片

需要准备的实际资源：

- app-logo.svg
- app-logo.ico
- poster-placeholder.png

可选准备：

- app-logo.png

不再准备：

- default-avatar.png
- player-placeholder-bg.png
- 空状态插画图片
- 小图标图片

暂不准备：

- AI / 用户画像装饰图

## 2. App Logo

### 文件

必须准备：

- app-logo.svg
- app-logo.ico

可选准备：

- app-logo.png

### 用途

- 应用图标
- 窗口标题栏
- 任务栏图标
- 关于弹窗
- 启动画面，如果后续有
- 设置页 / 用户资料页中的品牌展示，如果后续需要

### 设计方式

Logo 使用 SVG 矢量方式设计。

Logo 最终方向为黑色 / 深色底板 + 白色 XF 艺术字。

Logo 需要保证：

- 小尺寸清晰
- SVG 外层画布可以透明，但 Logo 主体内部应包含黑色 / 深色圆角底板
- 不要求将白色 XF 艺术字单独抠成透明背景
- 导出 PNG 时应保留完整图标底板
- 适配深色 / 浅色主题
- 可转换为 ICO
- 后续可修改颜色和细节

### 设计方向

最终 Logo 方向：

- 黑色 / 深色圆角底板
- 白色 XF 艺术字
- 简洁
- 高级
- 商业化
- 适合作为桌面软件图标
- 不使用复杂 3D
- 不使用强霓虹
- 不使用赛博朋克
- 不使用真实电影元素
- 不使用人物
- 不使用电影海报元素
- 不强行表达播放按钮、胶片、宇宙轨道等元素

### 设计概念

软件名为 XFVerse。

其中：

- XF 是软件名称前缀
- Verse 表达“影音宇宙 / 个人片库宇宙 / Media Verse”的概念

Logo 图形本身以 XF 艺术字作为识别核心。  
Verse 的概念可以由软件名称、界面内容和品牌说明承担，不要求 Logo 图形强行表达所有含义。

### 推荐尺寸

SVG：

- viewBox 建议使用 0 0 1024 1024

如果导出 PNG：

- 1024 x 1024
- 背景按 Logo 方案处理

ICO 建议包含：

- 16 x 16
- 32 x 32
- 48 x 48
- 256 x 256

## 3. Default Avatar

### 是否准备图片

不准备默认头像图片。

不需要：

- default-avatar.png

### 用途

- 用户未设置头像时显示
- 用户个人资料页
- 侧边栏用户区域，如果有
- 评论 / 账号区域，如果后续有

### 实现方式

默认头像由 WPF / XAML 生成。

结构：

- 圆形头像容器
- 偏纯色的灰粉 / 紫灰背景
- 可以有非常轻微的渐变，但渐变不要明显
- 居中显示用户名首字母
- 如果用户名为空，则显示 WPF UI 默认用户图标

### 行为规则

如果用户有头像：

- 显示用户头像

如果用户没有头像但有用户名：

- 显示用户名首字母

如果用户没有头像且没有用户名：

- 显示 WPF UI 默认用户图标

### 示例

用户名：

- XFVerse

默认头像显示：

- X

用户名：

- 夜间观影者

默认头像显示：

- 夜

## 4. Poster Placeholder

### 文件

- poster-placeholder.png

### 是否使用 AI 生成

使用 AI 生成。

### 用途

- 影片没有海报时显示
- 影片卡片
- 影片详情页海报区域
- 搜索结果列表
- 推荐影片卡片
- 识别修正结果列表

### 数量

- 只准备 1 张统一占位图

### 比例

- 2:3

### 推荐尺寸

- 800 x 1200

### 背景

- 正常背景
- 非透明背景
- 不需要透明
- 不留透明区域
- 本身作为完整海报画面使用

### 风格要求

- 极简
- 高级
- 克制
- 深灰 / 黑灰主色
- 少量灰粉和紫灰柔和光影
- 介于电影感和柔和感之间
- 不使用真实电影画面
- 不使用真实电影海报
- 不使用真实人物
- 不使用明星
- 不出现片名
- 不出现任何文字
- 不强烈图标化
- 不要电影幕布
- 不要胶片条
- 不要大播放按钮
- 不要赛博朋克
- 不要强霓虹

### 画面方向

可以包含：

- 柔和几何光影
- 抽象暗色渐变
- 非常低调的小型抽象影片符号
- 低透明度的播放暗示
- 现代软件占位图感觉

整体要像现代软件里的占位图，不要像电影宣传海报，也不要像普通图标占位。

## 5. Player Background

### 是否准备图片

不准备播放器背景图片。

暂时不需要：

- player-placeholder-bg.png

说明：

- 目前使用 WPF / XAML 生成
- 如果后续实际效果不够好，再补充图片资源

### 用途

- 播放器未加载影片时的背景
- 播放失败时的背景
- 播放器空状态背景
- mpv 播放区域未初始化时的占位背景，如果需要

### 实现方式

播放器背景由 WPF / XAML 生成。

### 视觉结构

播放器背景由以下元素组成：

- 基础深灰 / 黑灰背景
- 左上角低透明灰粉光晕
- 右下角低透明紫灰光晕
- 轻微暗色渐变
- 中心区域保持干净

### 控件可读性要求

背景必须保证以下内容清晰可读：

- 底部控制栏
- 播放进度条
- 缓存进度
- 播放源菜单
- 字幕菜单
- 音轨菜单
- 左侧亮度条
- 右侧音量条

## 6. Empty States

### 是否准备独立图片

不准备单独空状态插画资源。

### 实现方式

空状态统一封装为 EmptyState 组件，使用：

- WPF UI 默认图标
- 标题文案
- 说明文案
- 可选操作按钮
- 少量灰粉 / 紫灰渐变背景装饰

### 适用场景

- 无影片
- 无搜索结果
- 无扫描记录
- 无观影数据
- 无播放源
- 识别结果为空
- 画像数据不足
- 缓存为空
- 推荐结果为空
- WebDAV 未连接

### 空状态文案示例

#### 无影片

标题：

暂无影片

说明：

添加 WebDAV 片库后，这里会显示你的影片收藏。

操作：

去添加片库

#### 无搜索结果

标题：

没有搜索结果

说明：

换个关键词试试，或者使用识别修正功能重新搜索。

#### 无观影数据

标题：

暂无观影数据

说明：

观看或标记影片后，这里会生成你的专属观影洞察。

#### 无播放源

标题：

暂无播放源

说明：

当前影片还没有可用播放源，请重新扫描或修正识别。

#### 画像数据不足

标题：

画像仍在学习中

说明：

继续观看、标记喜爱或加入想看后，AI 会更准确地理解你的口味。

## 7. Icons

### 图标策略

普通图标统一使用：

- WPF UI 默认图标

不足部分：

- 由 Codex 判断是否补充 Fluent / IconPack
- 不能混用风格差异明显的图标

### 图标使用规则

- 普通图标使用中性色
- 选中图标使用主色
- 状态图标使用对应语义色
- 危险图标使用低饱和红 / 橙红
- 成功图标使用低饱和绿色
- 不要所有图标都使用浅粉色
- 不要手动准备大量小图标图片

### 不需要准备的图标资源

不需要手动准备：

- 播放图标
- 暂停图标
- 设置图标
- 搜索图标
- 用户图标
- 收藏图标
- 喜爱图标
- 已看图标
- 未看图标
- 扫描图标
- WebDAV 图标
- 字幕图标
- 音轨图标
- 播放源图标
- 修正识别图标

这些优先由 WPF UI 默认图标或后续补充图标库提供。
