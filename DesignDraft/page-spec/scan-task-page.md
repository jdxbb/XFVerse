# 扫描任务页面 UI 实施说明

## 1. 模块名称

扫描任务

建议文件名：

- `scan-task.md`

建议实现对象：

- `ScanTaskPage`
- 或 `ScanTaskPage.xaml`

---

## 2. 页面用途

扫描任务页面用于配置 WebDAV 扫描信息、管理扫描路径、启动或暂停扫描、取消扫描、查看扫描进度和历史扫描记录。

该页面是完整全屏页面，不是弹窗。

用户从首页左下角的头像和账号区域进入：

1. 点击首页左下角头像 / 账号区域
2. 弹出用户菜单
3. 点击菜单中的“扫描任务”
4. 打开扫描任务全屏页面

页面主要功能包括：

- 展示扫描任务标题
- 配置已添加扫描路径
- 配置 WebDAV 连接信息
- 保存 WebDAV 配置
- 测试 WebDAV 连接
- 展示连接反馈
- 展示扫描进度
- 展示当前扫描文件
- 展示扫描统计数据
- 开始 / 暂停扫描
- 取消扫描
- 展示历史扫描记录
- 切换主题

## 8. 已添加路径区域

### 用途

用于选择和管理需要扫描的路径。

### 内容

包含：

- 标题：`已添加路径`
- 文件夹图标
- 文案：`选择要添加的路径`
- 按钮：`选择路径`
- 已添加路径列表
- 每个路径项右侧的 `移除` 按钮

### 文件夹图标

截图中正方形区域表示文件夹图标。

要求：

- 使用 WPF UI 默认图标或等效文件夹图标
- 图标颜色使用中性色或主色轻提示
- 不使用图片文件作为图标

### 选择路径按钮

按钮文案：

- `选择路径`

点击后：

- 按现有后端 / 现有业务逻辑选择扫描路径
- 路径可能是本地路径，也可能是 WebDAV 远程路径
- UI spec 不强行限定路径来源
- 选择成功后添加到已添加路径列表

### 已添加路径列表

已添加路径列表是可滚动区域。

要求：

- 列表区域高度固定或受父容器约束
- 路径数量过多时，列表内部滚动
- 不让整个页面被路径列表撑开
- 每条路径是一行路径项
- 每条路径右侧有 `移除` 按钮
- 路径文本过长时使用省略号
- 鼠标悬浮路径文本时显示完整路径 Tooltip

### 路径项结构

每个路径项包含：

- 路径文本
- 移除按钮

路径显示规则：

- 路径过长时单行省略
- Tooltip 显示完整路径
- 不换行撑高列表项

移除按钮：

- 文案：`移除`
- 点击后移除该路径
- 按现有业务逻辑处理确认或直接移除

## 13. 扫描统计气泡

扫描进度卡片内有六个统计气泡。

字段包括：

- 已扫描数量
- 已识别数量
- 未识别数量
- 更新数量
- 错误数量
- 耗时

每个统计气泡包含：

- 语义图标
- 指标名称
- 指标值

要求：

- 图标使用 WPF UI 默认图标或补充图标库
- 图标需要与语义相关
- 气泡文字左对齐
- 指标名和数值保持清晰层级
- 不同状态可使用低饱和语义色点缀
- 不使用截图中的黑色占位块样式

气泡示例：

```text
[图标] 已扫描数量
       128
```

布局要求：

- 六个气泡分两行或按截图结构排列
- 在可用空间内保持均匀分布
- 不要拥挤
- 不要因为数字变化导致布局跳动

## 14. 扫描操作按钮

扫描进度卡片右侧包含两个操作按钮：

1. 开始 / 暂停扫描按钮
2. 取消扫描按钮

### 开始 / 暂停扫描按钮

初始状态：

- 文案：`开始扫描`

点击后：

- 启动扫描
- 按钮文案变为：`暂停扫描`

再次点击后：

- 暂停扫描
- 按钮文案变回：`开始扫描`

说明：

- 按用户确认，按钮在开始与暂停之间切换
- 实际扫描状态以现有后端逻辑为准
- UI 不擅自改变扫描业务逻辑

### 取消扫描按钮

按钮文案：

- `取消扫描`

用途：

- 取消当前扫描任务
- 取消行为绑定现有 Command 或由后端逻辑处理

视觉建议：

- 开始扫描为主按钮
- 暂停扫描为次级或状态按钮
- 取消扫描可使用危险 / 警告语义，但不要强烈刺眼
- 按钮左侧可带语义图标

## 16. 扫描记录气泡卡片

每条扫描记录是一个长方形气泡卡片。

用户确认：

- 不需要点击查看详情
- 不需要展开
- 不打开详情弹窗
- 只显示截图中的内容

### 卡片内容

每条记录包含：

- BaseUrl
- 用户名
- 扫描状态
- 开始时间
- 结束时间
- 扫描数量
- 新增数量
- 错误数量

### 对齐规则

扫描记录气泡卡片内文字必须左对齐。

要求：

- 标签和值左对齐
- 多行信息保持统一左边界
- 不要居中排版

### 最后一行统计

最后一行统计项包括：

- 扫描
- 新增
- 错误

布局要求：

- 三个统计项水平均匀分布
- 每个统计项内部文本左对齐或在自身区域内保持清晰
- 不要挤在一起
- 不要因为数字长度变化导致布局明显跳动

### 视觉状态

扫描记录可以根据状态轻微变化视觉：

- 成功：低饱和绿色点缀
- 失败：低饱和红点缀
- 扫描中：蓝色或主色点缀
- 已取消：灰色或橙色点缀
- 部分异常：低饱和橙色点缀

但不应使用高饱和背景色。

## 18. 状态处理

### WebDAV 连接测试状态

测试连接可能有以下状态：

- 未测试
- 测试中
- 成功
- 失败

UI 表现：

- 未测试：连接反馈为空或显示轻量提示
- 测试中：显示加载状态
- 成功：低饱和绿色提示
- 失败：低饱和红提示 + 错误信息

### 扫描状态

扫描可能有以下状态：

- 未开始
- 扫描中
- 暂停
- 已取消
- 已完成
- 失败

UI 表现：

- 进度条颜色根据状态变化
- 扫描按钮文案根据状态变化
- 统计气泡实时更新
- 当前扫描文件实时更新
- 扫描记录在扫描完成 / 取消 / 失败后生成或更新

### 空状态

如果没有扫描记录：

- 使用 EmptyState 组件
- 标题可为：`暂无扫描记录`
- 说明可为：`开始扫描后，这里会显示历史扫描结果。`

如果没有已添加路径：

- 路径列表区域可为空
- 不需要额外插画
- 可显示轻量提示，但不要占用过多空间

## 19. 深色 / 浅色主题注意事项

页面视觉遵循 `DESIGN.md`。

### 浅色主题

- 页面背景为淡粉白或统一浅色背景
- 卡片使用白色或轻微渐变
- 进度条底色使用浅灰
- 统计气泡使用浅色表面
- 文字使用深灰 / 深蓝灰
- 主按钮使用灰粉 / 浅粉点缀

### 深色主题

- 页面背景使用深灰
- 卡片使用深灰表面或稍亮层级
- 使用边框和透明度区分卡片层级
- 进度条底色使用深灰 / 中灰
- 统计气泡使用深色表面
- 不使用纯黑大面积背景
- 不使用强烈发光效果
