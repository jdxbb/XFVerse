## 2. 页面用途

设置页面用于管理 XFVerse 的通用设置、行为设置、API 配置和软件缓存入口。该页面是由用户菜单进入的全屏功能页，不是弹窗。

本规格已按 Phase 6.0g-B 用户确认结果更新。与旧截图或旧文字说明冲突时，以本文件当前规则为准。

---

## 3. 设计来源与保留范围

参考截图：

- `screenshots/账号/设置/01-设置-通用-全屏.png`
- `screenshots/账号/设置/02-设置-API配置-全屏.png`

保留旧设计的主要结构：

- 设置页标题与副标题风格
- 页面内独立主题切换入口
- `通用` 与 `API 配置` 两个基础 Tab
- 行为设置卡片
- 关于 XFVerse 区域
- API 配置卡片的整齐分组方式

更新后的关键规则：

- 全局壳层可以同时保留主题入口；设置页内的主题切换控件不取消。
- 设置页允许纵向滚动，不再要求全部内容固定在单屏中。
- 缓存管理仅按当前真实类别设计，不恢复视频缓存入口。
- API 配置补入 OpenSubtitles 与 AI 高级路由设置。
- 密码、token、API key 不得常显明文，统一采用显示 / 隐藏眼睛按钮。
- 保存和测试是两个独立动作；保存不自动执行连接测试。

## 4. 页面整体结构

```text
设置页面
├─ 顶部标题区
│  ├─ 返回按钮
│  ├─ 标题：设置
│  ├─ 副标题
│  └─ 页面内主题切换按钮
├─ Tab 区
│  ├─ 通用
│  └─ API 配置
└─ 可纵向滚动内容区
   ├─ 通用 Tab
   │  ├─ 缓存管理入口 / 摘要
   │  ├─ 行为设置卡片
   │  └─ 关于 XFVerse
   └─ API 配置 Tab
      ├─ TMDB 配置卡片
      ├─ OMDb 配置卡片
      ├─ OpenSubtitles 配置卡片
      └─ AI 配置卡片
         └─ 高级路由设置
```

页面级规则：

- 顶部不把 App Logo 当作返回按钮；左上方返回区域使用返回图标。
- 主题切换为图标按钮，位于设置页标题区域；它与全局入口可以并存。
- 页面内容区允许纵向滚动；Tab、标题区可按实现需要保持可见或随内容滚动。
- 卡片内部不因普通字段数量产生横向滚动。

## 5. 标题、Tab 与页面状态

### 标题区

- 标题：`设置`
- 副标题使用简短设置语义，不包含实现说明或服务内部名称堆叠。
- 右上角保留旧设计的主题图标按钮。

### Tab

- 基础分区仍为 `通用` 与 `API 配置`。
- 每次进入设置页默认打开 `通用`；不要求记忆上次 Tab。
- Tab 选中态使用全局控件基线，切换时页面宽度不得跳动。

### 通用状态

页面和卡片需要设计以下状态：

- `loading`：读取配置或缓存统计中。
- `saving`：保存按钮 disabled 并显示执行中反馈。
- `testing`：测试连接进行中。
- `success`：保存或测试成功。
- `error`：保存或测试失败，显示安全错误提示。
- `config missing`：必填配置缺失。
- `disabled`：因依赖条件不足不可操作，并说明原因。

## 6. 通用 Tab

### 6.1 缓存管理

通用 Tab 展示软件缓存管理区。完整规格见 `cache-management-page.md`。

最终缓存类别仅包含：

- 海报缓存
- metadata / 其它缓存
- 在线字幕缓存

明确不纳入：

- 视频缓存管理入口
- 视频缓存容量上限设置
- 将 mpv 缓冲描述为可清理的离线视频缓存

### 6.2 行为设置

行为设置沿用旧设计结构，并保留以下最终设计项：

| 设置项 | 控件形态 | 设计说明 |
| --- | --- | --- |
| 关闭窗口时的行为 | 单选 / 分段选择 | 例如退出应用或最小化到托盘，具体值以功能阶段确认 |
| 点击播放后自动全屏 | 开关 | 描述播放器启动后的窗口行为 |
| 主题模式 | 单选 / 下拉选择 | 与页面内主题快捷按钮共同工作 |
| 自动扫描 WebDAV | 单选 / 开关 | 描述启动时扫描策略 |

当前实现若尚未提供某项能力：

- 在 Phase 7 或后续功能对齐阶段处理。
- 不在最终 UI 中展示“后续阶段接入”之类占位文案。
- 不在 Phase 6 文档更新中声称功能已经实现。

### 6.3 关于 XFVerse

- 保留低调的关于行：产品名称、版本信息、进入关于内容的箭头按钮。
- 该区域不是营销卡片，不抢占设置主流程。

## 7. 敏感字段统一规则

以下字段属于敏感字段：

- password
- token
- API key
- access token
- 其它可用于访问远程服务的凭据

控件规则：

- 默认隐藏内容。
- 输入框末尾提供小眼睛图标按钮。
- 点击眼睛后显示当前内容；图标切换为闭眼状态。
- 再次点击后恢复隐藏。
- 显示 / 隐藏只影响界面呈现，不改变保存语义。
- 状态提示、错误消息、日志和示例文案不得回显凭据内容。

非敏感字段，例如 endpoint、模型名称、默认语言，可以使用普通文本或选择控件；完整 WebDAV URL 在日志和历史记录区仍不得裸露。

## 8. API 配置 Tab

API 配置区包含四类服务卡片：

```text
API 配置
├─ TMDB 配置
├─ OMDb 配置
├─ OpenSubtitles 配置
└─ AI 配置
   ├─ 基础连接配置
   └─ 高级路由设置
```

### 8.1 通用交互

每张服务卡片可以包含：

- 卡片标题与配置状态
- 输入字段
- `保存` 按钮
- `测试连接` 按钮
- 安全的状态文本或状态徽标

保存与测试规则：

- `保存` 只保存当前配置，不自动测试连接。
- `测试连接` 只在用户主动点击后执行。
- 保存成功、保存失败、测试中、测试成功、测试失败分别显示状态反馈。
- 测试失败可以说明配置缺失、鉴权失败、网络错误或服务限制，但不得泄露凭据。

### 8.2 TMDB 配置

字段可包含：

- Read Access Token
- API Key，备用或按当前能力使用

所有 token / API key 按敏感字段规则处理。

### 8.3 OMDb 配置

字段：

- OMDb API Key

API key 按敏感字段规则处理。

### 8.4 OpenSubtitles 配置

OpenSubtitles 是 API 配置区第四张正式卡片。

字段与控件：

| 字段 | 控件 / 状态 |
| --- | --- |
| 启用在线字幕服务 | 开关 |
| Endpoint | 普通文本输入 |
| API key | 敏感字段输入 |
| Username | 普通文本输入，可选 |
| Password | 敏感字段输入，可选 |
| 默认语言 | 下拉选择 |
| 保存 | 独立命令按钮 |
| 测试连接 / 鉴权 | 独立命令按钮 |

语义：

- 允许 API-key-only 使用方式。
- username / password 是可选的增强鉴权配置。
- 测试状态需要覆盖未配置、鉴权失败、网络错误、服务限制等结果。
- 下载额度如不能提前查询，不要求设置页预先展示；额度类反馈由下载返回结果承载。
- 在线字幕搜索发生在播放器链路；设置页只负责配置。

### 8.5 AI 配置

AI 配置包含两层：

1. 基础连接配置：endpoint、API key 等。
2. 高级路由设置：不同业务用途对应的模型和 timeout。

高级路由可覆盖当前真实用途，例如：

- 详情修正
- 批量辅助识别
- 扫描辅助识别
- AI 推荐
- 观影画像

呈现规则：

- 信息密度较高时，高级路由设置可折叠或按用途分组。
- API key 按敏感字段规则处理。
- 不把尚未存在的业务能力作为新配置用途添加。

## 9. 缓存管理入口与边界

- 缓存卡片详细结构、清理状态和在线字幕保护语义见 `cache-management-page.md`。
- 缓存清理确认遵循 `global-dialogs.md`。
- 缓存清理不删除真实本地媒体文件，不删除 WebDAV 远端文件，不清除用户配置。
- 在线字幕缓存清理只清孤立缓存，不删除绑定关系，也不清理仍被绑定引用的缓存。

## 11. Phase 7 实施注意事项

- 当前文档补全设计，不代表旧设计行为设置已经由当前代码实现。
- Phase 7 应基于全局表单、敏感输入、卡片、状态反馈和危险按钮 token 实施 UI。
- 不为满足页面外观擅自新增设置保存逻辑、测试逻辑或服务能力。
- 不恢复视频缓存管理入口。
- 不将设置页扩展成字幕下载或字幕绑定管理页。

## 12. 验收标准

- 设置页保持旧设计标题 / 副标题风格，并保留页面内主题切换图标按钮。
- 设置页允许纵向滚动，内容增长时仍可访问全部配置。
- 通用 Tab 包含缓存管理、行为设置和关于区域。
- 行为设置包含关闭行为、自动全屏、主题模式和自动扫描 WebDAV 的最终设计说明；未实现项不会伪装为可用功能。
- 缓存管理只显示海报、metadata / 其它、在线字幕三类缓存，不显示视频缓存管理。
- API 配置包含 TMDB、OMDb、OpenSubtitles、AI 四类卡片。
- OpenSubtitles 支持 API-key-only 和可选账号密码的设计表达。
- AI 配置包含基础连接与高级路由 / timeout 的信息架构。
- 所有 password、token、API key 使用眼睛图标显示 / 隐藏规则，默认不明文展示。
- 保存与测试独立；保存不会自动触发测试。
- loading、saving、testing、success、error、config missing、disabled 状态均有设计承载。
- 在线字幕配置、在线字幕搜索、缓存清理三者边界清楚。
