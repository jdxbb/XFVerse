# 媒体库特殊媒体项 UI 实施说明

## 1. 模块名称

媒体库特殊媒体项与人工聚合

建议文件名：

- `media-library-special-items.md`

建议实现对象：

- `MediaItemCard`
- `MediaItemRow`
- `RemovedLibraryPanel`
- `ManualUnknownSeasonAggregationDialog`

关联规格：

- `media-library-page.md`
- `global-dialogs.md`

---

## 2. 模块用途

本规格承载媒体库中无法仅以已识别电影或已识别电视剧表达的项目类型，以及批量人工聚合为未识别季的交互。

覆盖：

- 未识别电影
- orphan 媒体文件项
- failed placeholder
- grouped placeholder
- unknown season / 未识别季
- Other
- no-source / metadata-only 状态
- 人工聚合为未识别季弹窗

本规格的核心原则：

- 未识别不等于无详情
- 所有媒体库项目都必须有详情入口
- 修正属于详情页内部操作
- 人工聚合属于批量模式操作
- 普通卡片不新增 `修正` 或 `聚合` 主入口

---

## 3. 类型与语义

| 类型 | 含义 | 在媒体库中的角色 | 详情目标 |
|---|---|---|---|
| Unknown Movie | 已形成电影占位但未识别完成的项目 | 单片待补 metadata 或待修正项目 | 未识别电影详情页 / 电影详情占位态 |
| Orphan | 有媒体文件但尚未稳定归属 Movie 或 Season | 待确认归属的媒体文件项 | Orphan / 未识别媒体详情页 |
| Failed Placeholder | 识别失败后保留的占位项目 | 显示失败和继续处理入口的承载项 | 对应电影或季详情占位态 |
| Grouped Placeholder | 多个疑似同剧集文件的分组候选 | 尚未成为正式 Season 的集合项 | 未识别季详情页 |
| Unknown Season | 已形成季结构但尚未识别到正式剧集 metadata | 可播放或可修正的电视剧季 | 未识别季详情页 |
| Other | 无法归入当前 Movie / TV 展示模型的项目 | 其它待处理内容集合 | 对应 Other 详情页 |

说明：

- Grouped placeholder 不设计为“无详情”项目
- 如果当前实现尚未能为某类项目打开详情，Phase 7 需要补齐入口和对应占位详情态
- 详情页是否最终复用 Movie / Season 页面结构，由详情链路审计阶段确定

---

## 4. 状态标签

特殊项可组合展示以下数据状态：

- `未识别电影`
- `未识别季`
- `待修正`
- `其他`
- `无播放源`
- `仅 metadata`
- `识别失败`

状态规则：

- `未识别` 表示 metadata / 归属未完成，不表示不能进入详情
- `无播放源` 表示无活动可播放源，与是否识别完成分开表达
- `仅 metadata` 表示存在可展示信息但当前没有可播放源
- `识别失败` 只用于反馈历史或当前识别结果，不取代详情入口
- 如果同一项同时为未识别和无播放源，应分别显示识别状态和来源状态，或使用明确组合文案

视觉要求：

- 状态标签沿用媒体库卡片的语义色和密度
- `待修正` 使用警示但不刺眼的低饱和颜色
- `无播放源` 使用中性或警示态，不使用危险删除颜色

---

## 5. 海报卡片与列表项

### 共通结构

每个特殊项至少包含：

- 海报占位或现有海报
- 显示标题
- 内容类型 / 状态标签
- 来源或播放源状态
- 详情入口

普通模式：

- 点击卡片主体或详情动作进入对应详情页
- 卡片不显示修正和人工聚合主按钮

批量模式：

- 点击卡片主体切换选中
- 不直接进入详情
- 可由批量工具栏提供人工聚合动作

### 字段矩阵

| 特殊类型 | 海报卡片字段 | 列表项可追加字段 | 状态标签 | 详情入口 |
|---|---|---|---|---|
| Unknown Movie | 文件名或推测标题、占位海报、来源摘要 | 来源类型、文件摘要、年份候选（如有） | 未识别电影 / 待修正 / 无播放源 | 未识别电影详情 |
| Orphan | 文件名、占位海报、来源摘要 | 文件来源、可播放源情况 | 未识别 / 待修正 | Orphan 详情 |
| Failed Placeholder | 标题或文件摘要、占位海报、来源 | 失败状态说明 | 识别失败 / 待修正 | 对应占位详情 |
| Grouped Placeholder | 分组名或剧名候选、季号（如有）、文件数 | 集号覆盖范围、来源数量 | 未识别季 / 待修正 / 无播放源 | 未识别季详情 |
| Unknown Season | 剧名、季号、季名、集数、来源状态 | 已看集数、来源数量 | 未识别季 / 待修正 | 未识别季详情 |
| Other | 标题 / 摘要、占位海报、来源 | 当前可解释的类型原因 | 其他 / placeholder / 无播放源 | Other 详情 |

---

## 6. 详情入口与修正边界

### 详情入口

以下项目均必须从媒体库进入详情：

- 未识别电影
- orphan
- failed placeholder
- grouped placeholder
- unknown season
- Other

详情页职责：

- 展示可用 metadata、来源、文件或集号信息
- 提供该类型适用的修正入口
- 在没有播放源时呈现无播放源状态
- 在资料不完整时呈现占位 / 待修正状态

### 不放在卡片上的动作

媒体库普通卡片不增加：

- `修正`
- `识别修正`
- `聚合为季`
- `加入已有季`

原因：

- 卡片用于浏览与进入详情，避免操作层级拥挤
- 修正需要详情上下文
- 聚合涉及多选项目，必须由批量模式承载

---

## 7. 无播放源与 metadata-only

### 无播放源

适用场景：

- 已识别但没有活动播放源
- 未识别但没有活动播放源
- 已移出后恢复信息可见但当前没有活动源

展示要求：

- 卡片和列表项显示 `无播放源`
- 允许进入详情
- 可被媒体库的 `无播放源` 筛选命中
- 纳入结果区 `无播放源` 统计

### Metadata-only

metadata-only 是一种可展示但不可播放的数据状态：

- 可显示标题、海报、年份和标签等 metadata
- 同时明确显示 `无播放源` 或 `仅 metadata`
- 详情页可展示完整信息，但播放动作应 disabled 并说明原因

---

## 8. 已移出媒体库中的特殊项

特殊项被移出后仍可在独立的 `已移出媒体库` 面板中管理。

面板内应支持：

- 展示类型和状态标签
- 单项恢复
- 详情入口
- 删除记录
- 对可以形成 TV 分组的项目提供分组恢复或删除记录动作

语义：

- 移出仅隐藏，不清用户状态、不清 metadata、不删除真实文件
- 删除记录只清软件记录，不删除真实本地文件或 WebDAV 文件

---

## 9. 人工聚合为未识别季

### 入口边界

人工聚合为未识别季只属于批量模式：

- 普通卡片没有人工聚合按钮
- 用户在批量模式选择适用项目后，通过批量工具栏进入

适用项目：

- 有活动播放源的未识别媒体项
- grouped placeholder
- 可被合并为未识别 Season 的 failed placeholder 或 orphan

不适用项目：

- 没有活动播放源的项目
- 已确认归属且不需要聚合的内容
- 普通 Series 总览

### 弹窗结构

```text
ManualUnknownSeasonAggregationDialog
├─ Title: 人工聚合为未识别季
├─ Description
├─ SeriesTitleInput
├─ SeasonNumberInput
├─ SeasonTitleInput
├─ SourceAssignmentList
│  ├─ FileName / SourceSummary
│  ├─ CurrentBinding
│  └─ EpisodeNumberInput
├─ StatusMessage
└─ Actions
   ├─ Cancel
   ├─ AggregateAndIdentify
   └─ Aggregate
```

### 交互与校验

- 必须填写剧名和季名称
- 季号为 `0` 或正整数
- 每个来源需要有效集号
- 相同集号可以表示同一集的多播放源
- 没有可聚合播放源时不允许提交

### 状态

| 状态 | 展示要求 |
|---|---|
| 准备来源中 | 显示正在展开已选未识别播放源 |
| 输入不可提交 | 禁用确认动作，并说明缺少的字段或无效集号 |
| 聚合中 | 锁定字段与按钮，显示进行中 |
| 聚合成功 | 显示移动来源数、创建集数和多源补充数量 |
| 聚合失败 | 显示失败原因，并说明未完成变更 |
| 聚合后识别中 | 显示识别进度 |
| 聚合成功但识别失败 | 保留未识别季，并提示可在详情页继续修正 |
| 取消 | 关闭弹窗，不修改当前项目 |

---

## 10. 批量 AI 与特殊项

特殊项可以参与现有批量 AI 辅助识别流程，但 UI 必须显示结果类别：

- 成功
- 跳过 / 无结果
- 失败
- 已取消
- 部分失败

要求：

- AI 处理中不得把卡片主入口改成识别操作
- 未成功项应保持可再次处理的选择反馈
- 识别后的详情路由遵循项目最终类型

---

## 11. Phase 7 实施注意事项

- 补齐 grouped placeholder 等当前缺少的详情入口
- 详情页承载修正动作，媒体库卡片不承载修正主入口
- 人工聚合只通过批量模式打开
- 特殊项的卡片和列表项沿用媒体库统一样式，不另造视觉体系
- 保持移出和删除记录语义，不通过文案弱化安全边界

---

## 12. 验收标准

- Unknown Movie、Orphan、Failed Placeholder、Grouped Placeholder、Unknown Season 和 Other 均有设计规格
- 所有特殊项目在普通模式下均可进入对应详情
- Grouped Placeholder 不被设计为无详情状态
- 卡片不存在修正或人工聚合主入口
- 修正入口明确归属详情页
- 人工聚合明确归属批量模式
- 无播放源和 metadata-only 均可在卡片、筛选、统计和详情状态中表达
- 已移出面板可管理特殊项并保持安全语义
- 人工聚合弹窗覆盖校验、进行中、成功、失败和取消状态
