# 首页 UI 实施说明

## 1. 模块名称

首页

建议文件名：

- `home-page.md`

建议实现对象：

- `HomePage`
- 或 `HomePage.xaml`

---

## 2. 页面用途

首页是 XFVerse 打开软件后的默认页面。

打开软件时固定进入首页。

首页用于展示：

- 用户欢迎信息
- 片库状态统计
- 继续观看影片
- 最近新增影片
- AI 推荐影片
- 主导航入口
- 用户账号菜单入口

首页是完整页面，不是弹窗。

---

## 3. 截图说明

本模块截图包含 3 张：

1. 展开导航栏状态
2. 展开导航栏时点击头像账号区域后出现用户菜单
3. 收起导航栏状态

截图中的颜色不参与最终 UI 设计。

截图颜色只用于区分区域和信息结构。  
最终视觉必须遵循：

- `DESIGN.md`
- `resources-note.md`
- `codex-ui-rules.md`

实际 UI 应使用：

- WPF UI
- 深色 / 浅色双主题
- 灰粉 / 中性色视觉体系
- 低饱和语义色
- WPF UI 默认图标优先

---

## 4. 页面整体结构

首页主要由三部分组成：

1. 左侧导航栏
2. 中间主内容区
3. 右侧 AI 推荐区

展开导航栏状态结构：

```text
首页
├─ 左侧导航栏
│  ├─ Logo + XFVerse
│  ├─ 导航栏收起按钮
│  ├─ 导航菜单
│  └─ 底部账号区域
│
├─ 中间主内容区
│  ├─ 欢迎区
│  ├─ 片库预览
│  ├─ 继续观看
│  └─ 最近新增
│
└─ 右侧 AI 推荐区
   ├─ 标题区
   ├─ 推荐列表
   └─ 发现更多影片按钮
```

收起导航栏状态结构：

```text
首页
├─ 顶部左侧导航栏展开按钮 + 首页标题
├─ 中间主内容区
│  ├─ 片库预览
│  ├─ 继续观看
│  └─ 最近新增
└─ 右侧 AI 推荐区
```

说明：

- 展开导航栏状态下，首页使用旧设计的欢迎语结构。
- 全局壳层的 `PageTitle / PageSubtitle` 不替代展开态首页欢迎区。
- 收起导航栏状态下，按旧设计保留紧凑的导航栏展开按钮 + `首页` 标题，不重复显示展开态欢迎文案。

---

## 5. 默认进入规则

软件启动后：

- 默认进入首页
- 首页导航项处于选中状态
- 如果导航栏状态需要记忆，可按现有逻辑处理
- 如果没有现有逻辑，默认展开导航栏

---

## 6. 左侧导航栏

### 展开状态内容

展开状态下，左侧导航栏包含：

- Logo 区
- XFVerse 名称
- 导航栏收起按钮
- 导航菜单
- 底部账号区域

导航菜单项包括：

- 首页
- 媒体库
- 影片发现
- 观影历史
- 收藏夹
- 观影洞察

每个菜单项左侧有语义相关图标。

要求：

- 图标使用 WPF UI 默认图标或补充图标库
- 当前选中项为“首页”
- 选中态使用主色灰粉 / 浅粉点缀
- 未选中项使用中性色
- 不使用截图中的高饱和颜色

### 导航菜单点击行为

导航菜单只跳转到对应名字的页面，不额外推断页面功能。

跳转规则：

- 首页 → 首页
- 媒体库 → 媒体库页面
- 影片发现 → 影片发现页面
- 观影历史 → 观影历史页面
- 收藏夹 → 收藏夹页面
- 观影洞察 → 观影洞察页面

说明：

- 不要把“影片发现”额外理解成 AI 推荐页面，除非后续页面规格另有说明
- 路由命名和页面对象以现有项目为准

---

## 7. 导航栏展开 / 收起

### 展开状态

导航栏展开时：

- 导航栏显示 Logo、软件名、菜单文字、账号区域
- 导航栏右上角显示收起按钮
- 点击收起按钮后关闭导航栏
- 收起后按钮图标需要变化

### 收起状态

导航栏收起时：

- 左侧完整导航栏隐藏或收缩
- 导航栏切换按钮移动到首页左上区域
- 首页左上显示按钮 + 页面标题“首页”
- 点击按钮后重新展开导航栏
- 主内容区向左扩展

要求：

- 展开 / 收起切换应平滑
- 不应导致页面内容错位
- 不应遮挡主要内容
- 收起状态下仍然可以进入导航栏展开状态

---

## 8. 底部账号区域

展开导航栏底部为账号区域。

内容包括：

- 圆形头像
- 账号文本
- 右侧箭头

头像规则遵循 `DESIGN.md` 和 `resources-note.md`：

- 如果用户有头像，显示用户头像
- 如果用户没有头像但有用户名，显示首字母头像
- 如果用户没有头像且没有用户名，显示 WPF UI 默认用户图标

右侧箭头点击后显示用户菜单。

整个账号区域也可以作为点击区域，点击后打开用户菜单。

---

## 9. 用户菜单

用户菜单在点击账号区域后出现。

用户菜单不是全屏页面，是账号区域附近的浮层菜单。

菜单内容包括：

- 用户头像
- 用户名
- 个人资料
- 扫描任务
- 设置
- 退出登录

结构：

```text
用户菜单
├─ 用户头像
├─ 用户名
├─ 个人资料
├─ 扫描任务
├─ 设置
└─ 退出登录
```

### 菜单点击行为

- 个人资料 → 打开个人资料弹窗
- 扫描任务 → 打开扫描任务全屏页面
- 设置 → 打开设置全屏页面
- 退出登录 → 保留旧设计入口，但只作为视觉入口 / 假按钮

要求：

- 菜单应浮在首页内容之上
- 菜单不应被导航栏或内容卡片裁切
- 菜单项高度应适合点击
- 菜单项之间层级清晰
- 当前截图颜色不作为视觉参考
- 视觉遵循 WPF UI 菜单 / Flyout 风格
- 退出登录不接入后端登录、退出登录或云账号系统
- 如果需要提示，可在菜单底部显示轻量状态消息，例如“退出登录功能尚未接入”

---

## 10. 中间主内容区

中间主内容区包含：

- 欢迎区
- 片库预览
- 继续观看
- 最近新增

展开导航栏状态下，中间主内容区位于导航栏右侧、AI 推荐区左侧。

收起导航栏状态下，中间主内容区向左扩展。

---

## 11. 欢迎区

欢迎区包含：

- 欢迎标题
- 副标题
- 主题切换按钮

### 欢迎标题

文本：

- `欢迎回来，用户名 👋`

其中：

- `用户名` 来自当前用户数据
- 如果用户名为空，按现有用户逻辑处理

### 副标题

文本：

- `享受你的私人 AI 影音库`

### 主题切换按钮

主内容区右上角有主题切换按钮。

说明：

- 大部分页面有主题切换按钮，但不是必须全局固定
- 首页保留主题切换按钮
- 使用 WPF UI 图标按钮样式
- 图标可根据当前主题显示日间 / 夜间图标
- 按钮不应抢占页面主视觉

---

## 12. 片库预览

片库预览标题：

- `片库预览`

片库预览包含 4 个统计卡片：

- 已看
- 喜爱
- 想看
- 不想看

统计口径：

- 首页片库预览只展示本月统计，不展示全部累计统计
- 统计口径参考观影统计页已有“全部 / 本月”逻辑
- 首页只取本月概览
- 不保留“未看”统计

每个统计卡片包含：

- 语义图标
- 状态名称
- 数量
- 单位：`部`
- 相比上个月的变化值
- 趋势箭头

示例：

```text
已看
XXX 部
相比上个月 +10 ↑
```

趋势说明：

- 趋势展示形式沿用旧设计
- 趋势对比口径改为相比上个月
- 不再使用“较上周”
- 如果上个月无可比数据，可显示 0、留空或使用统一空状态文案，按后续数据规则决定

### 图标

截图中的黑色正方形是语义相关图标。

要求：

- 使用 WPF UI 默认图标或补充图标库
- 不使用图片资源
- 图标颜色可按对应状态使用低饱和语义色
- 图标不应全部使用主色

### 交互

片库预览统计卡不可点击。

要求：

- 不跳转页面
- 不触发筛选
- 不显示点击态
- 只作为统计展示

---

## 13. 继续观看

继续观看标题：

- `继续观看`

继续观看区域展示一组影片卡片。

每张继续观看影片卡片包含：

- 影片海报
- 电影名
- 观看至时间
- 百分比
- 底部观看进度条
- 播放源可用状态

示例字段：

```text
电影名
观看至 hh:mm:ss
百分比
```

### 海报点击行为

点击继续观看影片海报：

- 进入影片详情页

如果该影片暂无播放源：

- 卡片仍可以点击进入详情页
- 不直接触发播放
- 需要在卡片中显示“暂无播放源”或等效状态
- 进入详情页后由详情页展示播放源缺失原因或可修正入口

### 右侧箭头

继续观看区域右侧有箭头按钮。

点击箭头：

- 跳转到观影历史页面

说明：

- 该箭头不做横向切换
- 不作为横向查看更多按钮
- 只负责跳转到观影历史
- 当前成品中的“查看更多”入口在新版设计中合并为该右侧箭头，不单独保留文字按钮

---

## 14. 最近新增

最近新增标题：

- `最近新增`

最近新增区域展示一组影片卡片。

每张最近新增影片卡片包含：

- 影片海报
- 电影名
- 电影年份

### 海报点击行为

点击最近新增影片海报：

- 进入影片详情页

### 查看全部入口

如果当前成品保留“查看全部”入口：

- 入口放在最近新增标题区右侧
- 作为低权重文字按钮或图标 + 文案按钮
- 点击后进入媒体库，并优先呈现最近新增相关排序或筛选
- 不改变最近新增卡片本身结构

### 电影名显示规则

最近新增的电影名：

- 最多显示两行
- 超过两行省略
- 即使电影名不足两行，也需要预留两行高度
- 避免不同卡片因为标题行数不同导致底部年份上下跳动

## 15. 右侧 AI 推荐区

右侧 AI 推荐区包含：

- 标题区
- 推荐列表
- 底部“发现更多影片”按钮

标题区包含：

- `AI 推荐`
- 推荐来源说明
- `换一批` 按钮

推荐来源说明：

- `默认推荐库外 + 未看影片`

说明：

- 这里的“未看影片”描述 AI 推荐候选来源，不是片库预览统计卡
- 片库预览仍不保留“未看”统计

### 换一批

按钮文案：

- `换一批`

点击行为：

- 按现有后端逻辑获取或切换推荐列表
- 不在 UI spec 中重新定义推荐算法

## 16. AI 推荐卡片：展开导航栏状态

展开导航栏状态下，AI 推荐区宽度较窄。

推荐卡片布局参考第一张和第二张截图。

每条推荐包含：

- 海报
- 电影名，最多两行
- 年份
- 分类标签
- 情绪标签
- 推荐场景
- 平均评分
- `+想看` 按钮

展开导航栏状态下：

- 不显示推荐理由
- 推荐信息更紧凑
- 标签和评分字段按截图结构展示
- 海报位于左侧
- 电影名和标签信息位于中间或右侧
- 平均评分位于右侧区域
- `+想看` 按钮位于评分附近或右侧操作区域

字段说明：

- 年份是具体字段值
- 分类标签是具体字段值
- 情绪标签是具体字段值
- 推荐场景是具体字段值
- 平均评分是具体字段值

截图中的文字只是占位符，最终应绑定真实数据。

---

## 17. AI 推荐卡片：收起导航栏状态

收起导航栏状态下，AI 推荐区布局和展开导航栏状态不同。

收起导航栏状态下推荐卡片包含：

- 海报
- 电影名
- 年份 + 分类标签
- 平均评分
- 推荐理由
- `+想看` 按钮

与展开导航栏状态的区别：

- 收起导航栏状态下显示推荐理由
- 收起导航栏状态下标签只显示分类标签
- 不显示情绪标签
- 不显示推荐场景
- 布局更宽，推荐理由区域更明显

### 推荐理由

推荐理由区域：

- 显示 AI 推荐理由文本
- 文本区域应有稳定高度
- 推荐理由过长时按布局截断或省略
- 不应撑破推荐卡片
- 不应导致列表高度不可控

---

## 18. AI 推荐电影名显示规则

AI 推荐中的电影名：

- 最多显示两行
- 超过两行省略
- 即使电影名不足两行，也需要预留两行高度
- 避免卡片内部布局上下跳动

该规则适用于：

- 展开导航栏状态
- 收起导航栏状态

---

## 19. AI 推荐点击行为

### 海报

点击 AI 推荐影片海报：

- 进入影片详情页

### 电影名

点击 AI 推荐电影名：

- 进入影片详情页

### +想看

点击 `+想看`：

- 将该影片标记为想看
- 想看状态规则按现有业务逻辑处理
- 不在 UI spec 中重新定义业务限制

### 推荐卡片整体

除非现有项目已有整卡点击逻辑，否则不要求整张推荐卡片都可点击。  
至少需要保证海报和电影名可点击进入详情页。

## 20. 发现更多影片

右侧 AI 推荐区底部按钮：

- `发现更多影片 »`

点击后：

- 跳转到影片发现页面，并选中其中的 `AI 推荐` Tab

说明：

- `AI 推荐` 是影片发现页面中的推荐语义，不作为新版可见的独立页面
- 不以影片搜索 Tab 作为该入口的最终落点
- 当前成品如果仍进入默认搜索 Tab，只作为 Phase 7 需要对齐的 UI 行为差异
- 不要把该按钮解释为展开当前首页推荐列表

## 20.1 当前成品入口合并方案

当前成品已有的真实入口在新版设计中按以下方式设计化：

- `进入媒体库`：可放在片库预览标题区右侧，作为低权重次级入口
- `查看更多`：合并为继续观看旧设计右侧箭头，不再单独作为文字按钮
- `查看全部`：可放在最近新增标题区右侧，进入媒体库最近新增视图或排序
- `换一批`：保留在 AI 推荐标题区
- `发现更多影片`：保留在 AI 推荐区底部

这些入口只定义 UI 位置和交互语义，不新增后端逻辑。

---

## 21. 可滚动区域

首页无可滚动区域。

要求：

- 左侧导航栏不滚动
- 中间主内容区不滚动
- 右侧 AI 推荐区不滚动
- 继续观看区域不做横向滚动
- 最近新增区域不做横向滚动
- AI 推荐列表不滚动

窗口变小时：

- 允许整体缩放或压缩间距
- 需要设置合理最小窗口尺寸
- 低于最小尺寸时不保证完整展示
- 不应随意添加滚动条破坏无边记布局

---

## 22. 响应式 / 导航栏状态差异

首页需要支持两种导航栏状态：

1. 展开导航栏
2. 收起导航栏

### 展开导航栏状态

- 显示完整左侧导航栏
- 中间主内容区较窄
- 右侧 AI 推荐区较窄
- AI 推荐卡片不显示推荐理由
- AI 推荐卡片显示分类标签、情绪标签、推荐场景、平均评分

### 收起导航栏状态

- 左侧导航栏收起
- 导航栏切换按钮移动到首页左侧
- 中间主内容区向左扩展

- AI 推荐卡片显示推荐理由
- AI 推荐卡片标签只显示分类标签
- 不显示情绪标签和推荐场景

## 23. 状态处理

### 无用户信息

如果用户名为空：

- 欢迎区按现有用户逻辑处理
- 不要显示异常占位
- 可显示默认用户名称或留空，按现有逻辑决定

### 无继续观看

如果没有继续观看数据：

- 可使用 EmptyState 组件
- 不要空白大片区域
- 不使用独立插画图片

### 暂无播放源

如果继续观看影片暂无播放源：

- 卡片显示“暂无播放源”或等效状态
- 卡片不触发播放
- 海报、电影名或详情入口仍可点击进入详情页
- 不应把暂无播放源影片从继续观看中静默隐藏

### 无最近新增

如果没有最近新增数据：

- 可使用 EmptyState 组件
- 不使用独立插画图片

### 无 AI 推荐

如果没有 AI 推荐数据：

- 右侧 AI 推荐区可显示 EmptyState 组件
- 保留“换一批”或“发现更多影片”入口按现有逻辑处理

### 刷新中

首页刷新中：

- 可在片库预览标题区或页面顶部显示轻量 loading 状态
- 不使用全屏遮挡
- 不应导致首页布局大幅跳动

### 刷新失败

首页整体刷新失败：

- 显示错误文案
- 保留已有数据时不强制清空页面
- 如有重试命令，可显示低权重重试入口

### 局部刷新失败

某个模块局部刷新失败：

- 在对应模块显示局部错误状态
- 其它模块继续展示可用内容
- 不用一个模块失败阻断整个首页

### 图片缺失

影片海报缺失时：

- 使用 `poster-placeholder.png`

## 24. 深色 / 浅色主题注意事项

页面视觉遵循 `DESIGN.md`。

### 浅色主题

- 页面背景为淡粉白或统一浅色背景
- 卡片使用白色或轻微渐变
- 影片卡片保持清晰层级
- 统计卡片使用低饱和状态色
- 右侧 AI 推荐区保持清爽

- 卡片使用深灰表面或稍亮层级
- 使用边框和透明度区分内容
- AI 推荐区不使用强发光效果
- 不使用纯黑大面积背景，播放器除外
