# 观影洞察页面 UI 实施说明

## 1. 模块名称

观影洞察

建议文件名：

- `watch-insights-page.md`

建议实现对象：

- `WatchInsightsPage`
- 或 `WatchInsightsPage.xaml`

---

## 2. 页面用途

观影洞察页面用于展示用户的观影统计、偏好分析和用户画像。

页面包含两个 Tab：

- `画像分析`
- `观影统计`

默认打开：

- `画像分析`

用户从左侧导航栏点击“观影洞察”进入该页面。

该页面是完整页面，不是弹窗。

## 6. 导航栏状态

观影洞察页面支持：

- 展开导航栏状态
- 关闭导航栏状态

### 展开导航栏状态

展开状态下：

- 左侧显示完整导航栏
- 当前选中菜单为“观影洞察”
- 主内容区位于导航栏右侧
- 导航栏右上角显示关闭导航栏按钮
- 点击关闭导航栏按钮后，导航栏收起，页面切换为关闭导航栏状态

### 关闭导航栏状态

关闭状态下：

- 左侧导航栏隐藏或收缩
- 顶部左侧显示导航栏展开按钮
- 点击导航栏展开按钮后，恢复展开导航栏状态
- 页面标题显示为“观影洞察”
- 副标题显示为“让你更懂你”
- 主内容区向左扩展

说明：

- 展开导航栏右上角的按钮不是 Logo
- 关闭导航栏状态下页面左上角的按钮也不是 Logo
- 这两个按钮分别用于关闭导航栏和展开导航栏

## 7. Tab 区

观影洞察页面包含两个 Tab：

- `画像分析`
- `观影统计`

## 13. 口味象限

组件标题：

- `口味象限`

副标题：

- `你的观影风格更接近哪种？`

内容：

- AI 解读内容
- 十字坐标轴
- 用户点位

说明：

- 口味象限右边的正方形是一个十字坐标轴
- 十字坐标轴代表四个不同方向
- 四个方向固定写死
- 有四类具有代表性的类型
- 使用一个圆点标记用户所在的位置
- 圆点位置根据用户在四个方向的得分决定
- 具体决定规则待定，按后端或后续算法接入

要求：

- 十字坐标轴固定显示四个方向
- 四个方向文案固定写死
- 四类代表型固定显示
- 用户位置用圆点标记
- 圆点位置由用户四个方向得分决定
- AI 解读内容显示在左侧或卡片内

建议方向文案可沿用：

- 横轴：`熟悉安全 ←→ 新鲜探索`
- 纵轴：`轻松消遣 ←→ 情绪沉浸`

具体最终文案以项目实际设计为准。

## 16. 洞察总览

组件标题：

- `洞察总览`

副标题：

- `你的观影世界，一目了然`

包含 4 个状态统计卡：

- 已看
- 喜爱
- 想看
- 不想看

每个统计卡包含：

- 语义图标
- 状态名称
- 数量
- 单位：`部`
- 相比上个月的变化

下方扩展信息包含：

- 观影总时长
- 本月观影数

- 观影统计支持 `全部` / `本月` 切换。
- 趋势统一使用“相比上个月”，不使用“较上周”。
- 观影统计数据变化时自动刷新；刷新按钮不是最终必备控件。

### 本月高频标签

使用胶囊标签展示。

要求：

- 黑色正方形是语义相关图标
- 状态统计卡不可点击，只展示

## 22. 数据不足状态

观影洞察页面的数据不足处理规则：

- 能显示多少显示多少
- 缺数据的组件显示空状态
- 不因为某个组件缺数据而隐藏整个页面
- 不因为某个图表缺数据而影响其他组件显示

### 组件级空状态

适用组件包括：

- 观影口味总结
- 观影 DNA
- 观影人格
- 口味象限
- 看得多 vs 真喜欢
- 洞察总览
- 观影日历
- 偏好图谱
- 标签排行月榜
- 观影节奏
- 口味组合地图

空状态建议：

- 使用 EmptyState 组件
- 不使用独立插画图片
- 使用 WPF UI 图标 + 标题 + 说明文案
- 保持组件原本区域结构稳定

示例文案：

标题：

- `暂无足够数据`

说明：

- `继续观看、标记影片或加入想看后，这里会生成更完整的观影洞察。`

## 22.1 画像与统计状态矩阵

| 状态 | 展示方式 | 操作规则 |
|---|---|---|
| Loading | 保持卡片 / 图表区域结构稳定，展示轻量 loading | 禁用重复生成或冲突操作 |
| Data insufficient | 展示组件级数据不足状态 | 已有可用统计仍正常展示 |
| AI config missing | 在画像分析区域显示配置缺失状态 | 可选提供前往设置入口，不作为必备入口 |
| Generation failed | 显示生成失败信息 | 当前能力支持时可重试 |
| Cached result / fallback | 明确标识当前展示的是已缓存或上一次可用结果 | 不清空已有可读内容 |
| Error / Disabled | 使用统一错误或禁用状态 | 说明不可用原因 |

要求：

- 状态 UI 是最终规格的一部分，不能用空白区域或占位文案代替。
- 配置缺失的重点是清晰呈现状态；进入设置页仅是可选增强。

## 22.2 Movie-only 洞察边界

本页面的观影统计、Watch Profile、fingerprint、persona 与 AI 推荐画像均只使用 Movie 数据。

- TV、Episode、Season、Series 不进入电影洞察统计或画像。
- Episode 可以显示在观影历史中并导航到相关详情，但不纳入本页面统计与画像。
- Season 可以显示在收藏夹中并导航到季详情，但不纳入本页面统计与画像。
- 页面最终 UI 不额外展示“TV 已排除”提示；该边界作为设计和实现规则维护。
