# 全局壳层 UI 实施说明

## 1. 模块名称

全局壳层

建议文件名：

- `global-shell.md`

建议实现对象：

- `MainWindow`
- `AppShell`
- `AppSidebar`
- `UserAccountMenu`

---

## 2. 模块用途

全局壳层用于承载 XFVerse 主窗口的共同结构。

该规格覆盖：

- 品牌区
- 侧边栏展开 / 收起
- 主导航
- 页面顶部区域
- 底部账号入口
- 用户菜单
- 主题切换入口
- 内部隐藏路由记录

该规格只定义 UI 结构和交互语义，不新增业务功能。

---

## 3. 品牌区

最终品牌区使用旧设计方案：

- 左侧图标
- `XFVerse` 文字

不要把当前成品中的以下文字作为最终品牌展示：

- `智能影音库`
- `WebDAV 路径 · Metadata 识别 · AI`

要求：

- 左侧图标只作为品牌图标，不作为导航栏展开 / 收起按钮
- `XFVerse` 是主品牌文本
- 品牌区只出现在侧边栏品牌位置、关于 XFVerse、应用图标等品牌语境
- 品牌区不承载页面标题

---

## 4. 主窗口整体结构

主窗口由三部分组成：

1. 左侧侧边栏
2. 顶部 / 页面标题区域
3. 当前页面内容区域

结构：

```text
MainWindow
├─ CustomWindowChrome
│  ├─ DragRegion
│  └─ Minimize / Maximize or Restore / Close
├─ Sidebar
│  ├─ Brand: Icon + XFVerse
│  ├─ CollapseButton
│  ├─ NavigationMenu
│  └─ AccountEntry
│
├─ TopArea
│  ├─ SidebarExpandButton
│  ├─ PageTitle / PageSubtitle
│  └─ ThemeToggle
│
└─ PageContent
```

说明：

- 顶部 `PageTitle / PageSubtitle` 是普通页面的壳层标题区
- 展开导航栏状态下，首页使用旧设计欢迎语，不用壳层标题 / 副标题取代首页欢迎区
- 收起导航栏状态下，首页按 `home-page.md` 的旧设计显示紧凑标题

---

## 4.1 自定义窗口标题栏

主软件窗口的最终 UI 不使用原生系统窗口标题栏。Phase 7 应先建立全局窗口基线，并将以下控件纳入壳层：

- 可拖动标题栏区域。
- 最小化按钮。
- 最大化 / 还原按钮。
- 关闭按钮。

要求：

- 窗口按钮使用现代、克制的图标化设计，与当前主题 token 匹配。
- 自定义窗口控制不承担页面导航功能，也不取代 `PageTitle / PageSubtitle` 或首页欢迎区。
- 主窗口与播放器窗口复用窗口交互基线，但播放器使用固定深色变体。
- 本规则只定义最终视觉和交互位置，不在 Phase 6 修改现有窗口实现。

---

## 5. 侧边栏展开 / 收起

### 展开状态

展开状态下：

- 显示完整品牌区
- 显示主导航文字和图标
- 显示导航栏收起按钮
- 显示底部账号入口

导航栏右上角是收起按钮。

### 收起状态

收起状态下：

- 左侧完整侧边栏隐藏或收缩
- 页面左上角显示导航栏展开按钮
- 主内容区向左扩展

要求：

- 展开 / 收起按钮不是 App Logo
- 展开 / 收起切换不应遮挡页面主要内容
- 收起状态仍要能恢复展开状态

---

## 6. 主导航项

主导航项包括：

- 首页
- 媒体库
- 影片发现
- 观影历史
- 收藏夹
- 观影洞察

要求：

- 每个导航项有语义相关图标
- 当前选中项需要明确高亮
- 未选中项使用中性色
- 导航项只跳转对应页面，不额外推断业务功能

隐藏路由只作为内部路由记录，不作为主导航新增项。

当前内部路由可包括：

- 电视剧
- 电视剧季
- 剧集详情
- 影片详情
- 扫描任务
- AI 推荐（旧 UI 遗留 / 内部兼容路由）
- 设置

这些路由可以由菜单、卡片、详情入口或业务流程进入，但不因此加入主导航。

AI 推荐最终可见入口规则：

- 新版可见入口为 `影片发现` 页中的 `AI 推荐` Tab
- 首页 `发现更多影片` 进入影片发现页的 `AI 推荐` Tab
- 隐藏 Recommendations 路由不作为独立可见页面，不加入主导航

---

## 7. 当前选中态

主导航当前选中态使用主色点缀。

要求：

- 选中项背景、左侧标记或文字颜色需要和未选中项明显区分
- 不使用截图中的高饱和蓝作为最终颜色依据
- 选中态颜色由全局 token / ResourceDictionary 定义

---

## 8. 主题切换入口

主题切换使用旧设计的图标化方案。

要求：

- 使用图标按钮
- 图标根据当前主题显示日间 / 夜间语义
- 可放在首页欢迎区右上或全局顶部工具区，具体位置按页面规格决定
- 设置页可以同时保留自身标题区内的主题图标按钮；全局入口不会替代该页面内入口
- 不使用当前成品中的长文字按钮作为最终形态
- 需要有可访问名称或 tooltip

---

## 9. 底部账号入口

侧边栏底部为账号入口。

内容包括：

- 圆形头像
- 账号 / 用户名文本
- 右侧箭头

交互：

- 点击头像、账号文本或右侧箭头均可打开用户菜单
- 账号入口使用旧设计风格
- 头像规则遵循 `DESIGN.md` 和 `resources-note.md`

---

## 10. 用户菜单

用户菜单是账号入口附近的浮层菜单。

结构：

```text
用户菜单
├─ 用户头像
├─ 用户名
├─ 个人资料
├─ 扫描任务
├─ 设置
└─ 退出登录
```

菜单点击行为：

- 个人资料：打开个人资料弹窗
- 扫描任务：打开扫描任务页面
- 设置：打开设置页面
- 退出登录：使用旧设计入口，但只作为视觉入口 / 假按钮

退出登录说明：

- 不接入后端登录系统
- 不执行真实退出登录
- 不做云账号状态切换
- 可以显示轻量提示，例如“退出登录功能尚未接入”
- Phase 7 只实现视觉和提示状态，不新增账号后端逻辑

如果当前成品存在菜单状态消息区域，应设计为菜单底部提示区：

- 默认隐藏
- 点击假按钮或 disabled 操作时显示
- 不占用主要菜单项层级

---

## 11. 页面顶部 title / subtitle 关系

全局壳层可以为普通页面提供页面标题和副标题。

首页例外：

- 展开导航栏状态下，首页恢复旧设计欢迎语结构
- 展开态首页欢迎语是首页首屏内容的一部分
- 不用壳层 `PageTitle / PageSubtitle` 替代展开态首页欢迎区
- 收起导航栏状态下，按旧设计显示紧凑的 `首页` 标题

普通页面：

- 可以使用壳层 title / subtitle 展示页面名称和说明
- 具体标题文案由对应 page-spec 决定

---

## 12. 状态处理

壳层级状态只处理全局可见状态。

包括：

- 当前页面 loading
- 当前页面 error
- 页面无法进入
- 配置缺失入口提示
- 菜单项 disabled

要求：

- 不在壳层吞掉页面内部状态
- 页面内部 empty / loading / error 仍由对应 page-spec 说明
- 配置缺失类入口应给出明确提示，不静默失败

---

## 13. Codex 实现注意事项

Codex 实现时必须遵守：

- 不修改业务路由语义
- 不把隐藏路由加入主导航
- 不把当前品牌文案 `智能影音库` 作为最终设计品牌
- 不使用文字按钮替代旧设计主题图标方案，除非用户另行确认
- 退出登录是假按钮，不接入后端逻辑
- 首页欢迎语由 `home-page.md` 控制
- 全局 token / 控件基线应在 Phase 7 第一批完成
- 主软件窗口最终使用自定义窗口标题栏和现代窗口按钮，不保留原生系统标题栏

---

## 14. 验收标准

该壳层完成后，需要满足：

- 品牌区显示左侧图标和 `XFVerse`
- 侧边栏支持展开 / 收起
- 主导航只包含首页、媒体库、影片发现、观影历史、收藏夹、观影洞察
- 当前导航项有明确选中态
- 隐藏路由不进入主导航
- 隐藏 AI 推荐路由只作为内部兼容记录，最终可见推荐入口属于影片发现页
- 底部账号入口可以打开用户菜单
- 用户菜单包含个人资料、扫描任务、设置、退出登录
- 退出登录入口是视觉入口 / 假按钮
- 主题切换为图标化入口
- 展开态首页欢迎语不被壳层 title / subtitle 替代
- 主窗口拥有设计化的最小化、最大化 / 还原、关闭按钮，并不使用原生系统标题栏作为最终 UI
