# 影片发现页面 UI 实施说明

## 1. 模块名称

影片发现

建议文件名：

- `movie-discovery-page.md`

建议实现对象：

- `MovieDiscoveryPage`
- 或 `MovieDiscoveryPage.xaml`

---

## 2. 页面用途

影片发现页面用于搜索影片、浏览榜单影片、查看 AI 推荐影片。

用户可以从以下入口进入：

- 左侧导航栏点击“影片发现”
- 首页 AI 推荐区域点击“发现更多影片”

进入规则：

- 从左侧导航栏进入时，默认打开 `影片搜索` Tab
- 从首页“发现更多影片”进入时，默认打开 `AI 推荐` Tab

该页面是完整页面，不是弹窗。

顶部标题区包含：

- 页面标题
- 页面副标题
- 主题切换按钮

### 页面标题

标题文本：

- `影片发现`

### 页面副标题

副标题文本：

- `遇见更多适合你的影片`

说明：

- 展开导航栏状态下，截图中主要显示标题

### 主题切换按钮

页面右上角显示主题切换按钮。

要求：

- 图标可根据当前主题显示日间 / 夜间图标

## 6. 导航栏状态

影片发现页面支持：

- 展开导航栏状态
- 关闭导航栏状态

### 展开导航栏状态

展开状态下：

- 左侧显示完整导航栏
- 当前选中菜单为“影片发现”
- 主内容区位于导航栏右侧
- 当前 Tab 内容根据截图较紧凑展示

### 关闭导航栏状态

关闭状态下：

- 左侧导航栏隐藏或收缩
- 顶部左侧显示导航栏展开按钮
- 页面标题显示为“影片发现”
- 副标题显示为“遇见更多适合你的影片”
- 主内容区向左扩展
- 当前 Tab 内容根据可用宽度调整布局

---

## 7. Tab 区

影片发现页面包含三个 Tab：

- `影片搜索`
- `榜单`
- `AI 推荐`

要求：

- 当前选中的 Tab 需要有明显高亮
- 选中态使用主色灰粉 / 浅粉点缀
- 可使用下划线或其他轻量选中标记
- Tab 切换不应影响导航栏状态
- Tab 切换不应重置主题状态

### 默认 Tab 规则

- 从左侧导航栏进入影片发现页时，默认打开 `影片搜索`
- 从首页“发现更多影片”进入影片发现页时，默认打开 `AI 推荐`

# 影片搜索 Tab

## 8. 影片搜索 Tab 用途

影片搜索 Tab 用于搜索和筛选影片发现结果。

支持：

- 搜索影片、导演、演员
- 按筛选条件过滤
- 切换海报布局 / 列表布局
- 标记想看
- 进入影片详情页

---

## 9. 影片搜索筛选区

影片搜索筛选区包含搜索框、搜索按钮、筛选项、排序项和清除筛选按钮。

### 展开导航栏状态筛选项

展开导航栏状态下，按截图显示以下筛选项：

- 类型
- 地区
- 观看状态
- 顺序
- 排序

同时包含：

- 搜索框
- 搜索按钮
- 清除筛选
- 切布局

### 关闭导航栏状态筛选项

关闭导航栏状态下，按截图显示以下筛选项：

- 类型
- 年代
- 地区
- 语言
- 观看状态
- 顺序
- 排序

同时包含：

- 搜索框
- 搜索按钮
- 清除筛选
- 切布局

说明：

- 展开 / 关闭导航栏状态的筛选项按截图区分
- 不由 Codex 自行增加其他筛选项
- 筛选项具体选项按现有后端字段处理
- 未明确的筛选选项暂时待定

### 搜索框

提示文案：

- `搜索影片、导演、演员`

### 搜索按钮

按钮文案：

- `搜索`

点击后：

- 根据搜索词和筛选条件刷新影片结果

### 清除筛选

按钮文案：

- `清除筛选`

点击后：

- 清空搜索词
- 重置筛选条件
- 刷新影片结果

## 10. 影片搜索结果摘要

结果摘要显示：

- `共找到 xx 部影片`

关闭导航栏状态下可显示补充摘要，例如：

- `已看 xx · 未看 xx`

要求：

- 摘要根据当前搜索和筛选结果更新
- 不显示截图中的固定占位值
- 文案简洁，不挤压影片区域

---

## 11. 影片搜索布局切换

影片搜索支持两种布局：

- 海报布局
- 列表布局

按钮文案：

- `切布局`

点击后：

- 海报布局切换为列表布局
- 列表布局切换为海报布局

要求：

- 需要记住用户上次选择的布局
- 下次进入影片搜索 Tab 时保持上次布局
- 如果没有历史记录，默认海报布局
- 切换布局不清空搜索词
- 切换布局不清空筛选条件

---

## 12. 影片搜索海报布局

海报布局以网格卡片展示影片。

每张影片卡片包含：

- 海报
- 电影名
- 日期
- 类别标签
- 情绪标签
- 右下角评分
- 右上角想看状态按钮

结构示例：

```text
影片搜索海报卡片
├─ 右上：+想看 / 想看 / 已看
├─ 海报
├─ 电影名
├─ 日期 yyyy-mm-dd
├─ 类别标签 | 情绪标签
└─ 右下：评分
```

### 右下角评分

截图中海报右下角的蓝色方块表示评分。

要求：

- 显示影片评分
- 不显示状态标签
- 评分缺失时可留空或按现有逻辑处理

### 点击行为

普通状态下：

- 点击整张海报卡片进入影片详情页

### 海报缺失

如果影片没有海报：

- 使用 `poster-placeholder.png`

## 23. 想看状态按钮规则

影片搜索、榜单、AI 推荐中都有想看相关按钮。

### 未看且未想看

显示：

- `+想看`

点击后：

- 标记想看
- 按钮变为带图标的 `想看`

### 未看且已想看

显示：

- 带图标的 `想看`

点击后：

- 取消想看
- 按钮变回 `+想看`

### 已看影片

已看影片不显示 `+想看`。

显示：

- `已看`

要求：

- 已看状态不可作为想看按钮操作
- 不触发想看切换
- 不显示 `标记喜爱`
- 只显示已看状态

### 偏好互斥规则

遵循全局偏好状态规则：

- 不想看、想看、喜爱三者互斥
- 任意两个不能共存
- 不想看会覆盖想看和喜爱
- 想看只适用于未看影片
- 喜爱只适用于已看影片

- 标题：`暂无推荐结果`
- 说明：`换一批或调整推荐偏好后再试试。`

要求：

- 使用 EmptyState 组件
- 不使用独立插画图片
- 使用 WPF UI 图标 + 文案 + 可选操作按钮

- 使用边框和透明度区分层级
- 推荐理由区域在深色主题下保持可读
- 不使用强烈发光效果
- 不使用纯黑大面积背景，除非是海报占位或播放器区域

- 页面包含影片搜索、榜单、AI 推荐三个 Tab
