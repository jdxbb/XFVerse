# 影片发现页面 UI 实施说明

## 1. 模块名称

影片发现

建议文件名：

- `movie-discovery-page.md`

建议实现对象：

- `MovieDiscoveryPage`
- 或 `MovieDiscoveryPage.xaml`

---

## 2. 页面用途

影片发现页面用于搜索影片、浏览榜单影片、查看 AI 推荐影片。

用户可以从以下入口进入：

- 左侧导航栏点击“影片发现”
- 首页 AI 推荐区域点击“发现更多影片”

进入规则：

- 从左侧导航栏进入时，默认打开 `影片搜索` Tab
- 从首页“发现更多影片”进入时，默认打开 `AI 推荐` Tab

该页面是完整页面，不是弹窗。

## 2.1 Phase 6.0e-B 新版覆盖规则

本节根据影片发现与 AI 推荐组件审计后的确认结果补充，并覆盖本文后续与其冲突的旧假设。原截图仍用于布局参考，不再作为缺失功能组件或旧交互语义的唯一依据。

### 最终页面定位

- 页面标题采用旧设计：`影片发现`
- 页面副标题采用旧设计：`遇见更多适合你的影片`
- 不把 TMDB、TV 支持范围或当前推荐流程直接写成最终副标题
- 页面保留三个 Tab：`影片搜索`、`榜单`、`AI 推荐`
- Movie 与 TV 共用一个影片发现页面，不新增独立 TV Discovery 页面
- Movie / TV 切换只出现在 `影片搜索` 与 `榜单` 的媒介切换控件中
- `AI 推荐` 是影片发现页面内的 Tab，不作为新版主导航中的独立可见页面
- 旧实现中存在的隐藏 Recommendations 路由只作为内部兼容记录，不作为最终可见入口

### 进入规则

| 来源 | 最终目标 | 说明 |
|---|---|---|
| 主导航 `影片发现` | `影片搜索` Tab | 默认用于主动发现内容 |
| 首页 `发现更多影片` | `AI 推荐` Tab | 保留旧设计的推荐语义，不回落到搜索 Tab |
| Movie 搜索或榜单卡片 | `movie-detail-page.md` | 进入电影详情 |
| TV 搜索或榜单卡片 | `tv-detail-page.md` | 进入剧详情；metadata-only 仍可浏览详情 |

当前成品如果仍将首页入口导航到默认搜索 Tab，只记录为 Phase 7 需要对齐的 UI 行为差异，本阶段不修改代码。

### 页面级状态基线

影片发现页面所有 Tab 均需要覆盖以下通用状态：

| 状态 | 展示方式 | 行为 |
|---|---|---|
| Loading | 保持工具栏和结果容器稳定，使用轻量加载提示 | 禁用重复提交或冲突操作 |
| Empty | 使用 EmptyState，说明当前筛选或来源无结果 | 保留清空筛选、换一批或重试入口 |
| Error | 页面内错误提示和重试入口 | 不使用无说明空白区域 |
| Disabled | 显示不可操作原因 | 不仅使用颜色表示 |
| ConfigMissing / API key missing | 页面内提示 TMDB 或相关配置缺失，可提供进入设置入口 | 不以无结果替代配置问题 |
| RateLimited / Retry | 页面内提示服务繁忙或请求受限，并提供重试 | 不循环自动请求打扰用户 |
| PaginationLoading | 保留已显示结果并标记正在加载下一页 | 禁用重复翻页 |
| TvMetadataEnriching | TV 卡片保持稳定，并提示正在补充剧资料 | 不伪造可播放状态 |
| RequestCancelled | 新查询替代旧查询时不弹出打断提示 | 以最新结果和状态为准 |

## 2.2 搜索 Tab 新版组件补全

### 媒介与搜索模式

搜索 Tab 在旧设计筛选结构中增加媒介切换：

| 媒介 | 搜索模式 | 说明 |
|---|---|---|
| Movie | 影片 / 人物 | 人物搜索沿用当前模式切换能力，结果最终落到相关电影或电影详情 |
| TV | 电视剧 | TV 不提供人物搜索模式 |

### 布局切换

- 海报布局与列表布局均采用旧设计
- 布局切换必须是真实切换，不保留“切布局将在后续视觉阶段接入”之类占位提示作为最终状态
- 用户选择的布局应被记忆，再次进入搜索 Tab 时恢复上次选择
- 切换布局不清空搜索关键词、筛选条件、页码上下文或当前媒介
- Phase 7 负责实现真实切换和布局记忆，本阶段只定义设计语义

### 搜索卡片矩阵

| 卡片类型 | 基础字段 | 类型特有字段 | 操作 | 详情目标 |
|---|---|---|---|---|
| Movie 海报卡 / 列表项 | 海报、片名、原名、年份、评分、标签、播放源状态、用户状态 | 列表视图可展示时长、导演、演员 | 想看、加入媒体库；已隐藏项可恢复 | 电影详情 |
| TV 海报卡 / 列表项 | 海报、剧名、原名、年份、评分、标签、播放源状态、用户状态 | 总季数、有播放源季数、Season 状态摘要；metadata-only 标记 | 想看、加入媒体库；已隐藏项可恢复 | 剧详情 / metadata-only 剧详情 |

要求：

- TV 卡片视觉层级与 Movie 卡片一致，不设计为另一个页面体系
- `加入媒体库` 是 Discovery 卡片的明确动作
- `恢复` 只在当前项目已有隐藏 / 恢复能力适用时出现
- `已入库`、`已隐藏`、`无播放源`、`metadata-only` 作为状态呈现，不伪装成可播放结果
- Movie 点击进入电影详情；TV 点击进入剧详情，不因缺少播放源阻止浏览 metadata-only 详情

### TV metadata 不可用

- TV metadata 不可用时，使用搜索结果区域或页面状态区域内的提示
- 提示可以包含原因、重试或返回当前结果的动作
- 不使用打断用户浏览流程的信息弹窗作为默认呈现
- 该状态不表示 TV 可以进入 AI 推荐；TV 仍只属于搜索、榜单和详情链路

## 2.3 榜单 Tab 新版组件补全

榜单功能按当前真实范围设计，旧稿中的日榜 / 周榜 / 月榜 / 年榜示例入口由以下最终结构替代：

| 控件 | 选项 |
|---|---|
| 媒介切换 | Movie / TV |
| 榜单类型 | 热门 / 高分 / 趋势 |
| 趋势时间 | 今日 / 本周，仅在趋势榜适用 |

要求：

- 不新增月榜或年榜入口
- 榜单保留旧设计的 Top card 与列表卡片视觉层级，并补 Movie / TV 字段矩阵
- Movie 榜单项进入电影详情
- TV 榜单项进入剧详情或 metadata-only 剧详情
- Movie / TV 榜单卡片均可展示想看、加入媒体库以及当前已有的恢复状态
- 榜单覆盖 Loading、Empty、Error、ConfigMissing、RateLimited / Retry 与分页加载状态

| 榜单卡片类型 | 必要字段 | 补充字段 |
|---|---|---|
| Movie Top card / 列表卡片 | 排名、海报、片名、原名、年份、评分、简介、用户状态 | 播放源状态、加入媒体库 / 恢复 |
| TV Top card / 列表卡片 | 排名、海报、剧名、原名、年份、评分、简介、用户状态 | 总季数、有播放源季数、Season 状态摘要、metadata-only、加入媒体库 / 恢复 |

## 2.4 AI 推荐 Tab 引用

AI 推荐的最终组件、卡片、偏好弹窗、异步状态与 Movie-only 边界由 `recommendation-page.md` 定义。

# 影片搜索 Tab

## 8. 影片搜索 Tab 用途

影片搜索 Tab 用于搜索和筛选影片发现结果。

## 23. 想看状态按钮规则

影片搜索、榜单、AI 推荐中都有想看相关按钮。

### 未看且未想看

显示：

- `+想看`

点击后：

- 标记想看
- 按钮变为带图标的 `想看`

### 未看且已想看

显示：

- 带图标的 `想看`

点击后：

- 取消想看
- 按钮变回 `+想看`

### 已看影片

已看影片不显示 `+想看`。

显示：

- `已看`

要求：

- 已看状态不可作为想看按钮操作
- 不触发想看切换
- 不显示 `标记喜爱`
- 只显示已看状态

### 偏好互斥规则

遵循全局偏好状态规则：

- 不想看、想看、喜爱三者互斥
- 任意两个不能共存
- 不想看会覆盖想看和喜爱
- 想看只适用于未看影片
- 喜爱只适用于已看影片

### AI 推荐验收

- AI 推荐 Tab 显示提示区
- AI 推荐 Tab 显示范围选择：不区分来源、仅已有播放源、仅外部候选
- AI 推荐 Tab 显示观看状态选择：全部、已看、未看
- AI 推荐 Tab 显示自定义偏好开关
- 自定义偏好右侧显示设置图标按钮
- 设置图标按钮不显示“弹窗”文字
- 点击设置图标按钮打开自定义推荐偏好弹窗
- AI 推荐 Tab 显示换一批按钮
- 点击换一批刷新推荐结果
- AI 推荐影片区域不可滚动
- AI 推荐只展示固定数量影片
- AI 推荐卡片显示海报
- AI 推荐卡片显示电影名
- AI 推荐卡片显示日期和时长
- AI 推荐卡片显示类别标签、情绪标签、场景标签
- AI 推荐卡片显示评分
- AI 推荐卡片显示推荐理由
- 推荐理由区域内部可滚动

- 点击 AI 推荐电影名进入详情页
- AI 推荐只呈现 Movie，不显示 TV 推荐候选
- AI 推荐覆盖 MissingSeed、候选补充、retry、结果保留、loading、empty、error、disabled 和 config missing 状态
- 未看影片显示 +想看
- 已想看影片显示带图标的想看
- 已看影片显示已看，不显示 +想看

### 自定义推荐偏好弹窗验收

- 弹窗标题为“自定义推荐偏好”
- 弹窗显示副标题“输入你的观影偏好，推荐结果会更符合你的口味噢～”
- 弹窗包含自定义 prompt 输入框
- 弹窗包含清空按钮
- 弹窗包含确认按钮
- 点击清空后清空输入框，不关闭弹窗
- 点击确认后保存偏好
- 保存成功后关闭弹窗
- 保存成功后在下一次刷新或换一批时应用偏好，不强制立即刷新
- 保存失败时提示错误并保持弹窗打开
