# 全局弹窗 UI 实施说明

## 1. 模块名称

全局弹窗

建议文件名：

- `global-dialogs.md`

建议实现对象：

- `ConfirmationDialog`
- `ConfirmationDialogWindow`
- `IConfirmationDialogService`

---

## 2. 模块用途

全局弹窗用于承载项目中可复用的确认类交互。

本阶段新增全局确认弹窗规格。

当前项目已有全局 `ConfirmationDialog` 组件语义，后续 UI 重构应将以下确认归入同一体系：

- 普通确认
- 警示确认
- 危险确认
- 删除记录确认
- 移出媒体库确认
- 来源拆分确认
- 缓存清理确认
- 在线字幕删除绑定轻量确认 Popover
- 删除本地扫描路径轻量确认 Popover
- 其它不可撤销或高风险操作确认

该文档只定义 UI 结构、状态和文案规则，不修改现有业务逻辑。

---

## 3. 通用确认弹窗结构

确认弹窗包含：

- 标题
- 正文说明
- 取消按钮
- 确认按钮

结构：

```text
ConfirmationDialog
├─ Title
├─ Message
└─ Actions
   ├─ CancelButton
   └─ ConfirmButton
```

要求：

- 弹窗居中显示
- 弹窗不是全屏
- 文案可换行
- 按钮区始终在底部
- 默认焦点优先给取消按钮或安全操作，具体按 WPF UI 行为和风险等级处理

---

## 4. 普通确认

普通确认用于低风险操作。

示例：

- 离开未保存页面
- 清空临时筛选
- 重新加载当前数据

按钮：

- 取消按钮：次要按钮
- 确认按钮：主按钮

文案要求：

- 标题明确动作
- 正文说明影响
- 不使用恐吓式文案

---

## 4.1 轻量确认 Popover：删除在线字幕绑定

删除在线字幕绑定属于播放器菜单中的局部轻量确认，不使用居中大弹窗。Popover 锚定在对应字幕行的删除绑定按钮旁，并遵守播放器固定深色视觉。

建议结构：

```text
SubtitleBindingDeletePopover
├─ Message
└─ Actions
   ├─ CancelButton
   └─ RemoveBindingButton
```

建议文案：

```text
提示：仅解除该字幕与当前播放对象的绑定，不删除已下载的字幕缓存文件。
取消按钮：取消
确认按钮：删除绑定
```

要求：

- 用户必须显式确认或取消；点击外部或按 Esc 等同取消。
- 删除绑定只删除软件内绑定关系，不物理删除字幕缓存文件。
- 字幕缓存物理清理归软件缓存管理，不在播放器菜单中提供。
- Popover 不升级为删除记录或删除真实媒体文件能力。

---

## 5. 警示确认与移出媒体库

警示确认用于会改变当前可见结果、但操作本身可恢复且不会清除软件记录的动作。

媒体库中的 `移出媒体库` 归入警示确认，不归入删除记录的危险确认。

建议文案：

```text
标题：确认移出媒体库？
正文：移出后所选项目将从媒体库列表隐藏，可在“已移出媒体库”中恢复。该操作不会清除用户状态或 metadata，也不会删除本地文件或 WebDAV 文件。
取消按钮：取消
确认按钮：移出
```

要求：

- 单项和批量移出使用相同语义，仅按对象数量调整正文
- 确认按钮可使用主按钮或警示按钮，不使用删除语义
- 正文必须说明 hide-only、可恢复、保留状态和 metadata
- 关闭或取消不执行移出操作

---

## 5.1 轻量确认 Popover：删除本地扫描路径

删除本地扫描路径是扫描页路径行内的局部警示操作，使用删除按钮旁的小型 Popover，不默认使用居中大弹窗。

建议文案：

```text
标题：移除此扫描路径？
正文：将删除该扫描路径配置，并可能影响对应软件记录的可见性。不会删除真实本地文件。
取消按钮：取消
确认按钮：移除
```

要求：

- Popover 锚定在本地扫描路径的删除按钮旁。
- 用户显式确认后才移除配置；点击外部或按 Esc 等同取消。
- 确认按钮使用警示语义，不使用“删除文件”文案。
- 该操作不提供删除真实本地媒体文件的能力。

---

## 6. 危险确认

危险确认用于高风险或不可撤销操作。

包括：

- 删除记录确认
- 缓存清理确认
- 批量删除记录确认
- 其它会清除软件记录或本地缓存的操作

按钮：

- 取消按钮：次要按钮
- 确认按钮：危险按钮

危险按钮要求：

- 使用全局 danger token
- 颜色只用于确认危险动作，不用于普通主按钮
- 文案必须是动作词，例如 `删除`、`清理`

危险确认正文必须说明：

- 操作对象
- 操作影响
- 是否不可撤销
- 是否会删除本地文件 / WebDAV 文件

媒体库语义必须保持：

- 删除记录只清软件记录，不删除本地文件 / WebDAV 文件
- 移出媒体库只影响可见性 / hide
- 缓存清理只清理软件缓存，不删除媒体源文件

---

## 6.1 来源拆分警示确认

电影详情和单集详情中，如果已有 `从当前对象拆分来源` 动作，应统一归入警示确认，不视为删除文件。

操作分级：

| 来源操作 | 确认等级 | 说明 |
|---|---|---|
| 立即探测 | 普通操作，无确认弹窗 | 只读取或刷新来源信息 |
| 设为默认 | 普通操作，无确认弹窗 | 只修改默认播放源选择 |
| 从当前对象拆分来源 | 警示确认 | 改变来源归属，可能进入待修正或特殊项承载 |
| 打开识别修正 | 修正弹窗内部确认 | 使用 `correction-flow.md`，不作为删除类危险确认 |

来源拆分建议文案：

```text
标题：确认拆分此播放源？
正文：拆分后，该播放源将不再归属于当前详情对象，可能需要在未识别项目中继续修正。该操作不会删除本地文件或 WebDAV 文件。
取消按钮：取消
确认按钮：拆分来源
```

要求：

- Movie 和 Episode 的来源拆分使用同一警示语义
- 正文必须说明只改变软件内来源归属，不删除真实本地文件或 WebDAV 文件
- 关闭或取消不得执行拆分
- 本规格不为详情页新增删除记录或移出媒体库动作；这两类入口仍以媒体库规格为主

---

## 7. 删除记录确认

媒体库删除确认合并为全局确认弹窗的危险变体。

建议文案：

```text
标题：确认删除记录？
正文：该操作会删除所选媒体在软件中的记录和相关软件状态，不会删除本地文件或 WebDAV 文件。删除后不可从当前列表直接恢复。
取消按钮：取消
确认按钮：删除
```

要求：

- 必须二次确认
- 必须说明不删除本地文件 / WebDAV 文件
- 确认按钮使用危险按钮
- 批量删除记录和已移出面板中的删除记录 / 删除全部记录均复用该危险变体

---

## 8. 缓存清理确认

缓存清理确认合并为全局确认弹窗的危险变体。

建议文案：

```text
标题：确认清理缓存？
正文：该操作会清理软件缓存数据，不会删除媒体库记录、本地文件或 WebDAV 文件。清理后部分页面可能需要重新加载数据。
取消按钮：取消
确认按钮：清理
```

要求：

- 必须说明清理范围
- 必须说明不会删除媒体源文件
- 确认按钮使用危险按钮

### 在线字幕孤立缓存清理

在线字幕缓存清理属于缓存清理的专用危险变体，仅在能够可靠识别孤立缓存时可执行。

建议文案：

```text
标题：确认清理在线字幕缓存？
正文：该操作仅清理未被绑定引用的在线字幕缓存，不会解除字幕绑定，也不会删除本地媒体文件或 WebDAV 远端文件。
取消按钮：取消
确认按钮：清理孤立缓存
```

要求：

- 仍被绑定引用的缓存不可被清理。
- 绑定引用读取失败或无法确认时，清理按钮 disabled，并在卡片内说明停止清理的原因。
- 缓存管理的卡片、数量和禁用状态详见 `cache-management-page.md`。

---

## 9. 长文案处理

当正文较长时：

- 正文区域允许多行换行
- 不应撑破弹窗宽度
- 弹窗可设置最大宽度
- 正文过长时优先压缩文本宽度，不优先添加整体滚动条

如果确实需要滚动：

- 只允许正文区域滚动
- 按钮区保持固定可见

---

## 10. 关闭行为

关闭行为：

- 点击取消按钮返回取消
- 点击右上角关闭按钮或 Esc 等同取消
- 点击确认按钮返回确认
- 点击遮罩是否关闭按 WPF UI Dialog 默认行为处理，但危险确认建议不允许点击遮罩直接确认

要求：

- 关闭弹窗不应执行危险动作
- 只有确认按钮才能执行确认动作

---

## 11. 状态和按钮顺序

按钮顺序：

- 取消按钮在左
- 确认按钮在右

按钮状态：

- 普通确认：确认按钮使用主按钮
- 警示确认：确认按钮使用主按钮或全局警示按钮，文案不得暗示删除文件
- 危险确认：确认按钮使用危险按钮
- 条件不足时确认按钮可以 disabled

disabled 说明：

- disabled 状态需要有明确原因
- 可通过正文、提示文本或 tooltip 说明

---

## 12. Codex 实现注意事项

Codex 实现时必须遵守：

- 不在 Phase 6 修改代码
- 不为每个业务页面重新实现一套确认弹窗
- 媒体库移出确认归入全局警示变体
- 电影详情与单集详情中已存在的来源拆分归入全局警示变体
- 在线字幕删除绑定归入播放器菜单中的轻量确认 Popover，不复用大尺寸危险弹窗
- 删除本地扫描路径归入扫描页路径行旁的轻量警示 Popover，不复用大尺寸危险弹窗
- 媒体库删除确认、缓存清理确认合并为全局危险变体
- 不改变删除记录、移出媒体库、缓存清理的业务语义
- 危险按钮样式必须来自全局 token / ResourceDictionary
- 长文案、不可撤销提示和按钮顺序需要统一
- 删除在线字幕绑定的轻量确认必须说明不删除字幕缓存文件
- 删除本地扫描路径的轻量确认必须说明不删除真实本地文件
- 在线字幕缓存清理必须说明只清孤立缓存，并在引用关系不可确认时禁用清理

---

## 13. 验收标准

该组件完成后，需要满足：

- 普通确认、警示确认和危险确认共用同一确认弹窗体系
- 弹窗包含标题、正文、取消按钮、确认按钮
- 移出媒体库确认说明 hide-only、可恢复、保留状态 / metadata 且不删除真实文件
- 来源拆分确认说明只改变来源归属且不删除真实本地文件或 WebDAV 文件
- 危险确认使用危险按钮
- 删除记录确认说明不删除本地文件 / WebDAV 文件
- 批量删除和已移出项删除记录使用删除记录危险变体
- 缓存清理确认说明不删除媒体源文件
- 取消、关闭、Esc 都不会执行危险动作
- 长文案不撑破弹窗
- 浅色 / 深色主题下均可读
- 详情页不会因确认弹窗规格而新增删除记录或移出媒体库入口
- 在线字幕删除绑定使用按钮旁轻量 Popover，并明确只解除绑定、不删除字幕缓存文件
- 删除本地扫描路径使用按钮旁轻量 Popover，并明确只移除配置、不删除真实本地文件
- 在线字幕缓存清理只清理孤立缓存，受绑定保护或引用关系不明时不可清理
