# 全局弹窗 UI 实施说明

## 1. 模块名称

全局弹窗

建议文件名：

- `global-dialogs.md`

建议实现对象：

- `ConfirmationDialog`
- `ConfirmationDialogWindow`
- `IConfirmationDialogService`

---

## 2. 模块用途

全局弹窗用于承载项目中可复用的确认类交互。

本阶段新增全局确认弹窗规格。

当前项目已有全局 `ConfirmationDialog` 组件语义，后续 UI 重构应将以下确认归入同一体系：

- 普通确认
- 危险确认
- 删除记录确认
- 移出媒体库确认
- 缓存清理确认
- 其它不可撤销或高风险操作确认

该文档只定义 UI 结构、状态和文案规则，不修改现有业务逻辑。

---

## 3. 通用确认弹窗结构

确认弹窗包含：

- 标题
- 正文说明
- 取消按钮
- 确认按钮

结构：

```text
ConfirmationDialog
├─ Title
├─ Message
└─ Actions
   ├─ CancelButton
   └─ ConfirmButton
```

要求：

- 弹窗居中显示
- 弹窗不是全屏
- 文案可换行
- 按钮区始终在底部
- 默认焦点优先给取消按钮或安全操作，具体按 WPF UI 行为和风险等级处理

---

## 4. 普通确认

普通确认用于低风险操作。

示例：

- 离开未保存页面
- 清空临时筛选
- 重新加载当前数据

按钮：

- 取消按钮：次要按钮
- 确认按钮：主按钮

文案要求：

- 标题明确动作
- 正文说明影响
- 不使用恐吓式文案

---

## 5. 危险确认

危险确认用于高风险或不可撤销操作。

包括：

- 删除记录确认
- 缓存清理确认
- 批量删除记录确认
- 其它会清除软件记录或本地缓存的操作

按钮：

- 取消按钮：次要按钮
- 确认按钮：危险按钮

危险按钮要求：

- 使用全局 danger token
- 颜色只用于确认危险动作，不用于普通主按钮
- 文案必须是动作词，例如 `删除`、`清理`

危险确认正文必须说明：

- 操作对象
- 操作影响
- 是否不可撤销
- 是否会删除本地文件 / WebDAV 文件

媒体库语义必须保持：

- 删除记录只清软件记录，不删除本地文件 / WebDAV 文件
- 移出媒体库只影响可见性 / hide
- 缓存清理只清理软件缓存，不删除媒体源文件

---

## 6. 删除记录确认

媒体库删除确认合并为全局确认弹窗的危险变体。

建议文案：

```text
标题：确认删除记录？
正文：该操作会删除所选媒体在软件中的记录和相关软件状态，不会删除本地文件或 WebDAV 文件。删除后不可从当前列表直接恢复。
取消按钮：取消
确认按钮：删除
```

要求：

- 必须二次确认
- 必须说明不删除本地文件 / WebDAV 文件
- 确认按钮使用危险按钮

---

## 7. 缓存清理确认

缓存清理确认合并为全局确认弹窗的危险变体。

建议文案：

```text
标题：确认清理缓存？
正文：该操作会清理软件缓存数据，不会删除媒体库记录、本地文件或 WebDAV 文件。清理后部分页面可能需要重新加载数据。
取消按钮：取消
确认按钮：清理
```

要求：

- 必须说明清理范围
- 必须说明不会删除媒体源文件
- 确认按钮使用危险按钮

---

## 8. 长文案处理

当正文较长时：

- 正文区域允许多行换行
- 不应撑破弹窗宽度
- 弹窗可设置最大宽度
- 正文过长时优先压缩文本宽度，不优先添加整体滚动条

如果确实需要滚动：

- 只允许正文区域滚动
- 按钮区保持固定可见

---

## 9. 关闭行为

关闭行为：

- 点击取消按钮返回取消
- 点击右上角关闭按钮或 Esc 等同取消
- 点击确认按钮返回确认
- 点击遮罩是否关闭按 WPF UI Dialog 默认行为处理，但危险确认建议不允许点击遮罩直接确认

要求：

- 关闭弹窗不应执行危险动作
- 只有确认按钮才能执行确认动作

---

## 10. 状态和按钮顺序

按钮顺序：

- 取消按钮在左
- 确认按钮在右

按钮状态：

- 普通确认：确认按钮使用主按钮
- 危险确认：确认按钮使用危险按钮
- 条件不足时确认按钮可以 disabled

disabled 说明：

- disabled 状态需要有明确原因
- 可通过正文、提示文本或 tooltip 说明

---

## 11. Codex 实现注意事项

Codex 实现时必须遵守：

- 不在 Phase 6 修改代码
- 不为每个业务页面重新实现一套确认弹窗
- 媒体库删除确认、缓存清理确认合并为全局危险变体
- 不改变删除记录、移出媒体库、缓存清理的业务语义
- 危险按钮样式必须来自全局 token / ResourceDictionary
- 长文案、不可撤销提示和按钮顺序需要统一

---

## 12. 验收标准

该组件完成后，需要满足：

- 普通确认和危险确认共用同一确认弹窗体系
- 弹窗包含标题、正文、取消按钮、确认按钮
- 危险确认使用危险按钮
- 删除记录确认说明不删除本地文件 / WebDAV 文件
- 缓存清理确认说明不删除媒体源文件
- 取消、关闭、Esc 都不会执行危险动作
- 长文案不撑破弹窗
- 浅色 / 深色主题下均可读
