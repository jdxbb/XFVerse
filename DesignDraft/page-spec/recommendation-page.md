# AI 推荐 Tab UI 实施说明

## 1. 模块名称

AI 推荐 Tab 与自定义推荐偏好弹窗

建议文件名：

- `recommendation-page.md`

建议实现对象：

- `MovieDiscoveryPage` 中的 `AI 推荐` Tab
- `RecommendationsPage` 或等效 Tab 内容 View
- `RecommendationsViewModel`
- `CustomRecommendationPreferenceDialog`

关联规格：

- `movie-discovery-page.md`
- `home-page.md`
- `global-shell.md`
- `movie-detail-page.md`
- `DESIGN.md`
- `codex-ui-rules.md`

---

## 2. 模块用途与入口

AI 推荐用于根据用户已有的电影偏好生成 Movie 推荐，并提供推荐理由、偏好调整和用户状态动作。

最终入口规则：

| 来源 | 行为 |
|---|---|
| 影片发现页第三个 Tab | 显示 AI 推荐内容 |
| 首页 `发现更多影片` | 进入影片发现页并选中 `AI 推荐` Tab |
| 推荐卡片海报或标题 | 进入电影详情 |

边界：

- AI 推荐采用旧设计的页面内 Tab 结构
- AI 推荐不作为新版主导航中的独立可见页面
- 当前隐藏 Recommendations 路由属于旧 UI 遗留 / 内部兼容记录，不用于最终信息架构

---

## 3. Movie-only 业务边界

AI 推荐只推荐 Movie。

必须明确排除：

- TV 不进入电影 AI 推荐
- TV 不进入 Movie Watch Profile
- TV 不进入 movie fingerprint
- TV 不进入 movie persona
- TV 不作为推荐解释输入候选

TV 可在影片发现页中进行搜索、查看榜单和进入剧详情，但不因此进入 AI 推荐链路。未来若需要 TV 推荐，必须另开独立阶段定义业务语义和设计规格。

---

## 4. 整体布局

推荐区采用旧设计的结构与视觉层级：

```text
AI 推荐 Tab
├─ 提示 / 状态区
├─ 控制区
│  ├─ 推荐范围筛选
│  ├─ 观看状态筛选
│  ├─ 自定义偏好开关
│  ├─ 设置图标按钮
│  └─ 换一批按钮
└─ 推荐卡片区
   └─ Movie Recommendation Card
      ├─ 海报与电影名
      ├─ 状态 / 标签
      ├─ 想看 / 不想看动作
      └─ 推荐理由滚动区域
```

要求：

- 整体视觉遵循旧截图的卡片构成和空间关系
- 推荐理由区域可以独立滚动，避免长理由撑破卡片
- 推荐内容区应保持稳定，不因 loading 或短暂状态导致明显跳动
- 需要补入当前真实必要的状态标签，但不能将页面重构为信息密度失控的运营列表

---

## 5. 控制区

控制区视觉使用旧设计，但筛选语义采用当前真实能力：

| 控件 | 最终选项或行为 |
|---|---|
| 推荐范围 | `不区分来源` / `仅已有播放源` / `仅外部候选` |
| 观看状态 | `全部` / `已看` / `未看` |
| 自定义偏好 | 开关控制是否应用已保存偏好 |
| 偏好编辑 | 使用旧设计的设置图标按钮，不使用纯文字按钮作为最终形态 |
| 换一批 | 按当前推荐流程获取另一批 Movie 推荐 |

交互要求：

- 请求运行中时，冲突筛选和换一批按钮可 disabled，并显示原因
- 更改筛选后只改变 Movie 推荐集合，不引入 TV
- 配置缺失或服务错误时，控制区仍保持可识别，允许用户前往设置或重试

---

## 6. 推荐卡片矩阵

| 区域 | 必需内容 | 说明 |
|---|---|---|
| 主视觉 | 海报、电影名 | 无海报时使用统一占位图 |
| 基础信息 | 年份、时长、评分（数据存在时） | 字段缺失可隐藏，不填充假数据 |
| 标签 | 类型、情绪、场景标签（数据存在时） | 可结合当前真实 Tags 展示 |
| 可用性 | 已有播放源 / 外部候选 | 补入当前真实状态 |
| 观看状态 | 已看 / 未看，以及想看状态 | 与全局电影状态语义一致 |
| 操作 | 想看、不想看 | 不想看语义按现有业务规则处理 |
| 推荐理由 | 推荐解释文本 | 区域可独立滚动 |
| 详情入口 | 海报或电影名 | 进入电影详情 |

要求：

- 推荐卡片只承载 Movie，不提供 TV 标识或剧详情入口
- 卡片整体风格延续旧设计；新增可用性、观看状态和不想看动作时保持层级克制
- 无播放源 / 外部候选仍可进入电影详情的对应承载状态

---

## 7. 自定义推荐偏好弹窗

偏好弹窗视觉采用旧设计，功能逻辑按当前能力表达。

### 结构

- 标题：`自定义推荐偏好`
- 说明文本
- 多行偏好输入框
- 字数或长度限制提示
- 清空按钮
- 取消按钮
- 确认按钮

### 交互

| 操作 | 行为 |
|---|---|
| 打开 | 从设置图标按钮打开弹窗，并加载当前偏好 |
| 清空 | 清空偏好并给出反馈；是否立即关闭按当前功能逻辑处理 |
| 取消 | 不保存草稿，关闭弹窗 |
| 确认 | 保存偏好，成功后关闭弹窗 |
| 保存成功 | 提示偏好已保存；偏好在下一次刷新或 `换一批` 时生效 |
| 保存失败 | 保持弹窗可见，显示错误信息并允许重试 |

说明：

- 不要求保存偏好后立即刷新推荐列表
- 不把保存行为改写成业务逻辑变更
- 弹窗样式应复用全局 Dialog 和图标按钮基线

## 8. 状态矩阵

| 状态 | 展示内容 | 可用动作 |
|---|---|---|
| Initial / Loading | 正在分析偏好或生成推荐的轻量 loading | 禁用重复生成和冲突筛选 |
| MissingSeed | `先标记几部影片，AI 才能理解你的偏好` 类提示 | 保留进入媒体库或继续浏览的入口（如全局设计允许） |
| Empty | 当前筛选下暂无推荐结果 | 调整筛选、换一批 |
| CandidateRefillRunning | 正在后台补充推荐候选 | 保留已有结果时不清空卡片区 |
| CandidateRefillFailed | 候选补充失败 | 重试，或继续查看已保留结果 |
| Error | 推荐生成失败 | 重试；不展示伪造推荐 |
| PreservedResult | 新请求未完成但已有结果保留 | 明确说明结果为上一次可用内容 |
| Disabled | 当前动作暂不可用 | 显示不可用原因 |
| ConfigMissing / API key missing | AI 配置或 API key 缺失 | 页面内提示并可提供进入设置入口 |
| RateLimited / Busy | 服务繁忙或请求受限 | 页面内提示和稍后重试 |

要求：

- 状态文案统一使用中文
- 状态以页面内提示、EmptyState 或轻量状态带展示，不使用打断式信息弹窗作为默认失败呈现
- ConfigMissing 不与 Empty 混同
- 候选补充中若已有结果，优先保持结果区稳定

---

## 9. 与首页和详情页的关系

- 首页推荐预览和本 Tab 使用相同的 Movie-only 推荐语义
- 首页 `发现更多影片` 进入本 Tab，而非搜索 Tab 或独立推荐页面
- 推荐卡片点击进入 `movie-detail-page.md` 定义的电影详情
- metadata-only Movie 或外部候选可按电影详情承载状态展示，不意味着当前已有播放源
- TV 详情入口只来自搜索或榜单，由 `tv-detail-page.md` 定义

---

## 10. Phase 7 实施注意事项

- 以本规格为最终 AI 推荐 UI 目标，不以当前纯文字偏好按钮作为最终形态
- 不把隐藏 Recommendations 路由加入主导航
- 不扩展 AI 推荐算法或 Movie-only 业务边界
- 不将 TV 接入推荐、画像、fingerprint 或 persona
- 复用全局卡片、图标按钮、Dialog、EmptyState、Loading、Error、Disabled 与 ConfigMissing 基线
- 保留当前偏好保存后在下一次刷新或换一批生效的功能语义

---

## 11. 验收标准

- AI 推荐以影片发现页第三个 Tab 呈现
- 首页 `发现更多影片` 进入该 Tab
- 主导航中不新增独立 AI 推荐页面
- 推荐只包含 Movie，不出现 TV 推荐候选
- 推荐范围使用当前准确语义：不区分来源、仅已有播放源、仅外部候选
- 观看筛选覆盖全部、已看、未看
- 偏好编辑入口使用设置图标按钮
- 偏好弹窗采用旧设计视觉，并支持清空、取消、确认和保存失败状态
- 保存偏好后提示下一次刷新或换一批生效，不强制立即刷新
- 推荐卡片覆盖来源、可用性、观看状态、推荐理由、想看和不想看动作
- 推荐理由区域可滚动，整体布局保持旧设计结构
- MissingSeed、候选补充、retry、结果保留、loading、empty、error、disabled 与 config missing 均有设计状态
- ConfigMissing、RateLimited 和一般错误以页面内状态提示呈现
