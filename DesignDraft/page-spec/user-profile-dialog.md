# 个人资料弹窗 UI 实施说明

## 1. 模块名称

个人资料弹窗

建议文件名：

- `user-profile-dialog.md`

建议实现对象：

- `UserProfileDialog`
- 或 `UserProfileDialog.xaml`

---

## 2. 模块用途

个人资料弹窗用于展示和编辑本地个人资料信息。

Phase 6.0b-B 确认后的最终设计：

- 使用旧设计中的完整个人资料弹窗
- 不保留当前占位弹窗作为最终设计
- 个人资料是本地数据
- 不接复杂账号系统
- 不接登录后端
- 不做云同步
- 不把“退出登录”或账号菜单状态反向解释成真实登录体系

用户从首页左下角的头像和账号区域进入：

1. 点击首页左下角头像 / 账号区域
2. 弹出用户菜单
3. 点击菜单中的“个人资料”
4. 打开个人资料弹窗

该弹窗不是全屏页面，内容较少，不需要做成大页面，也不需要滚动。

说明：

- “真实页面”指本地资料查看和编辑 UI
- 本文档不要求在 Phase 6 实现资料保存逻辑
- Phase 7 实施时可以在现有本地设置、配置或用户资料存储能力内落地，不新增云账号后端

---

## 3. 展示方式

弹窗应以居中 Dialog 形式展示。

弹窗打开时，主界面背景应进行弱化处理，可采用以下方式之一：

- 背景变暗
- 背景轻微模糊
- 背景透明度降低
- 使用 WPF UI Dialog 默认遮罩效果

要求：

- 弹窗必须明显浮于主界面之上
- 背景处理不能影响弹窗内容可读性
- 不要使用强烈遮罩或高饱和背景
- 弹窗应适配浅色 / 深色主题

---

## 4. 弹窗尺寸规则

弹窗尺寸不写死具体像素，由 Codex 根据内容、WPF UI 默认 Dialog 行为和实际窗口大小处理。

但必须满足：

- 弹窗不是全屏
- 弹窗不是过小浮窗
- 弹窗大小应稳定
- 同一台电脑、同一窗口尺寸下，每次打开弹窗大小应一致
- 弹窗应占屏幕或主窗口的固定合理比例
- 内容不需要滚动
- 所有内容应在弹窗内完整展示

建议：

- 使用固定的最大宽度 / 最小宽度约束
- 使用固定的最大高度 / 最小高度约束
- 或使用相对主窗口比例计算
- 避免由内容随机撑开导致每次大小变化

## 5. 页面结构

弹窗从上到下包含 3 个主要区域：

1. 顶部标题区
2. 用户概览卡片
3. 基础信息卡片

整体结构：

```text
个人资料弹窗
├─ 顶部标题区
│  ├─ App Logo
│  ├─ 标题：个人资料
│  ├─ 副标题：管理你的账号信息
│  └─ 关闭按钮 X
│
├─ 用户概览卡片
│  ├─ 头像
│  └─ 用户名
│
└─ 基础信息卡片
   ├─ 标题：基础信息
   ├─ 编辑资料 / 确定按钮
   ├─ 左列信息
   │  ├─ 用户名
   │  ├─ 账号
   │  ├─ 手机号
   │  └─ 邮箱
   ├─ 分割线
   └─ 右列信息
      ├─ 性别
      ├─ 年龄
      └─ 个性签名
```

## 6. 顶部标题区

### 内容

顶部标题区包含：

- App Logo
- 主标题
- 副标题
- 关闭按钮

### App Logo

截图左上角的黑色矩形为 App Logo 区域。

要求：

- Logo 应为正方形展示
- 使用资源策略中的 `app-logo.svg` 或等效 Logo 资源
- 不要拉伸变形
- 保持清晰
- Logo 尺寸应与标题区协调
- Logo 与标题文本之间保持适当间距

### 标题

主标题：

- `个人资料`

副标题：

- `管理你的账号信息`

说明：

- 这里的账号信息指本地个人资料信息
- 不代表接入远端账号、登录系统或云同步

### 关闭按钮

右上角显示关闭 X。

要求：

- 使用 WPF UI 默认关闭图标或等效图标
- 点击关闭弹窗
- 支持键盘 Esc 关闭更佳
- 关闭按钮不应抢视觉主焦点
- 深浅主题下均需清晰可见

---

## 7. 用户概览卡片

### 用途

展示用户的头像和用户名，用于快速识别当前本地用户资料。

### 内容

包含：

- 用户头像
- 用户名

### 布局

参考截图：

- 头像在左侧
- 用户名在头像右侧
- 卡片横向布局
- 内容垂直居中
- 卡片留白充足

### 头像规则

默认头像不使用图片资源，由 WPF / XAML 生成。

头像显示规则：

1. 如果用户有自定义头像，显示用户头像
2. 如果用户没有头像但有用户名，显示用户名首字母
3. 如果用户没有头像且没有用户名，显示 WPF UI 默认用户图标

默认头像视觉：

- 圆形头像容器
- 偏纯色灰粉 / 紫灰背景
- 可有极轻微渐变，但渐变不要明显
- 居中显示首字母
- 不区分性别
- 不使用 AI 人物头像

### 用户名

用户名应显示为视觉重点。

如果用户名为空：

- 可以留空
- 不显示“未设置”

---

## 8. 基础信息卡片

### 用途

展示和编辑用户基础信息。

### 标题

卡片标题：

- `基础信息`

### 操作按钮

右上角按钮有两种状态。

#### 查看模式

按钮文案：

- `编辑资料`

点击后：

- 弹窗切换为编辑模式
- 基础信息中的字段值变为可输入控件
- 按钮文案变为“确定”

#### 编辑模式

按钮文案：

- `确定`

点击后：

- 保存当前输入
- 调用已有保存资料逻辑 / Command
- 保存成功后切换回查看模式
- 保存失败时显示错误提示，并保持编辑模式

如项目已有对应 Command，应优先绑定现有 Command。  
不要擅自新增后端业务逻辑。

---

## 9. 基础信息字段

基础信息分为左右两列。

### 左列字段

- 用户名
- 账号
- 手机号
- 邮箱

### 右列字段

- 性别
- 年龄
- 个性签名

字段说明：

- 以上字段均为本地资料字段
- `账号` 是本地显示字段，不代表登录账号或云账号
- 手机号、邮箱只作为本地资料文本，不要求接入验证、绑定或找回流程
- 性别、年龄、个性签名只作为本地展示和编辑内容

### 展示格式

查看模式下，字段采用：

- `字段名：字段值`

示例：

- `用户名：张三`
- `账号：user_001`
- `手机号：13800000000`
- `邮箱：example@example.com`
- `性别：女`
- `年龄：20`
- `个性签名：喜欢深夜看悬疑片`

如果字段没有值：

- 字段值留空
- 不显示“未设置”
- 不显示占位文案

例如：

- `手机号：`

---

## 10. 编辑模式

点击“编辑资料”后，在同一个弹窗内切换为编辑模式。

### 编辑方式

基础信息中的字段值变为可输入控件：

- 用户名：文本框
- 账号：文本框，是否可编辑由现有业务逻辑决定
- 手机号：文本框
- 邮箱：文本框
- 性别：可使用下拉框 / 选择控件 / 文本框，优先按现有业务逻辑
- 年龄：数字输入框或文本框，优先按现有业务逻辑
- 个性签名：文本框，可允许较长文本

业务边界：

- 编辑模式只编辑本地资料
- 不接入登录后端
- 不触发云同步
- 不新增复杂账号系统
- 不做手机号 / 邮箱验证码流程

### 布局要求

编辑模式下：

- 字段名保持不变
- 字段值区域替换为输入控件
- 不应导致弹窗大幅跳动
- 不应导致弹窗尺寸每次不一致
- 输入控件宽度应统一
- 两列布局仍然保持
- 字段为空时输入框内容为空，不显示占位文字，除非现有 UI 规范统一要求

### 按钮

编辑模式下：

- `编辑资料`

变为：

- `确定`

可选但建议：

- 如果有取消逻辑，可以保留右上角 X 关闭弹窗
- 是否需要“取消”按钮按现有业务逻辑决定
- 如果没有取消按钮，关闭弹窗时应按现有逻辑处理未保存内容

---

## 11. 可滚动区域

该弹窗不可滚动。

要求：

- 不要给弹窗整体添加滚动条
- 不要给基础信息卡片添加滚动条
- 内容应在弹窗固定尺寸内完整展示
- 如果未来字段增加，再单独评估是否需要滚动

---

## 12. 状态处理

### 加载状态

如果用户资料需要异步加载：

- 可在卡片区域显示轻量加载状态
- 避免全屏 loading
- 加载完成后显示资料内容

### 空值状态

字段无值时：

- 留空
- 不显示“未设置”
- 不显示占位文字

### 保存成功

点击“确定”保存成功后：

- 切换回查看模式
- 可使用 WPF UI Snackbar / Toast 显示轻提示
- 保存语义是本地资料保存

示例：

- `资料已更新`

### 保存失败

保存失败时：

- 保持编辑模式
- 使用 WPF UI Snackbar / MessageBar / Dialog 显示错误
- 不清空用户已输入内容
- 错误提示只说明本地保存失败，不提示云账号或登录状态

---

## 13. 深色 / 浅色主题注意事项

弹窗视觉遵循 DESIGN.md。

### 浅色主题

- 背景使用浅色 Dialog 表面
- 卡片使用白色或轻微渐变
- 文字使用深灰 / 深蓝灰
- 主按钮使用灰粉 / 浅粉点缀

### 深色主题

- 弹窗使用深灰表面
- 卡片使用更深或略亮的灰色层级
- 使用边框和背景层级区分内容
- 主按钮使用稍亮的灰粉 / 浅粉
- 不使用纯黑大面积背景

负责：

- 标题
- 副标题

- 基础信息标题

- 从首页左下角头像 / 账号区域菜单可以打开个人资料弹窗

- 弹窗大小稳定，不会每次打开尺寸不同
- 背景有遮罩、模糊或透明度降低效果
- 顶部显示正方形 App Logo
- 顶部显示“个人资料”和“管理你的账号信息”
- 右上角显示关闭 X
- 用户概览卡片显示头像和用户名
- 基础信息卡片显示标题“基础信息”
- 基础信息分为左右两列
- 左列包含用户名、账号、手机号、邮箱
- 右列包含性别、年龄、个性签名

- 点击“编辑资料”后在当前弹窗内切换为编辑模式

- 点击“确定”后调用保存逻辑
- 保存成功后返回查看模式
- 保存失败时保留输入并提示错误

- 不出现“个人资料功能将在后续阶段完善”占位态
- 弹窗无滚动条
- 浅色 / 深色主题下均可读
- 项目可以编译通过

## 返回按钮修正说明

该说明不适用于个人资料弹窗顶部标题区。

页面左上角的正方形图标区域在部分全屏页面中不是 App Logo，而是返回按钮。

适用模块：

- 扫描任务页面
- 设置页面
- 影片详情页面

要求：

- 使用 WPF UI 返回图标或等效图标
- 点击后返回上一页 / 上一个主界面状态
- 不作为 App Logo 展示
- App Logo 只在侧边栏品牌区、关于 XFVerse 行或需要品牌露出的区域出现
