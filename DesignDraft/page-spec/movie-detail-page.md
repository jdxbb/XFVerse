# 影片详情页 UI 实施说明

## 1. 模块名称

影片详情页

建议文件名：

- `movie-detail-page.md`

建议实现对象：

- `MovieDetailPage`
- 或 `MovieDetailPage.xaml`

---

## 2. 页面用途

影片详情页用于展示单部影片的完整信息，并提供播放、状态标记、播放源管理和影片识别修正能力。

用户可以从以下入口进入影片详情页：

- 首页继续观看影片海报
- 首页最近新增影片海报
- 首页 AI 推荐影片海报或电影名
- 媒体库海报卡片
- 媒体库列表项
- 其他影片列表入口

该页面是完整页面，不是弹窗。

---

## 3. 截图说明

本模块截图包含 2 张：

1. `01-影片详情页-全屏`
2. `02-影片详情页-影片识别修正-弹窗`

截图中的颜色不参与最终 UI 设计。

截图颜色只用于区分区域和信息结构。  
最终视觉必须遵循：

- `DESIGN.md`
- `resources-note.md`
- `codex-ui-rules.md`

实际 UI 应使用：

- WPF UI
- 深色 / 浅色双主题
- 灰粉 / 中性色视觉体系
- 低饱和语义色
- WPF UI 默认图标优先

## 4. 页面整体结构

影片详情页主要包含：

1. 返回按钮
2. 海报区
3. 影片基础信息区
4. 评分区
5. 主创信息区
6. 标签区
7. 简介区
8. 操作按钮区
9. 播放源区域

整体结构：

```text
影片详情页
├─ 返回按钮
├─ 海报区
├─ 影片信息区
│  ├─ 影片名
│  ├─ 原名
│  ├─ 上映时间
│  ├─ 时长
│  ├─ TMDB 评分
│  ├─ IMDb 评分
│  └─ 简介
│
├─ 右侧信息区
│  ├─ 主创信息
│  └─ 标签
│
├─ 操作按钮区
│  ├─ 播放默认源
│  ├─ 标记想看 / 标记喜爱 / 取消想看 / 取消喜爱
│  ├─ 标记已看 / 标记未看
│  ├─ 标记不想看 / 取消不想看
│  └─ 影片识别修正
│
└─ 播放源区域
   ├─ 入库影片：播放源列表
   └─ 未入库影片：居中文字提醒
```

---

## 5. 返回按钮

页面左上角的正方形区域是返回按钮，不是 App Logo。

要求：

- 使用 WPF UI 返回图标或等效图标
- 点击后返回上一页
- 不显示 App Logo
- 返回按钮位置与截图左上角一致
- 深色 / 浅色主题下都要清晰可见
- 不抢占页面主视觉

---

## 6. 海报区

左侧大卡片区域展示影片海报。

要求：

- 显示影片海报
- 海报保持 2:3 近似比例
- 不拉伸变形
- 圆角和阴影遵循 `DESIGN.md`
- 如果影片没有海报，使用 `poster-placeholder.png`

---

## 7. 影片基础信息区

影片基础信息位于页面中上区域。

包含：

- 影片名
- 原名
- 上映时间
- 时长

### 影片名

影片名是主标题。

要求：

- 字号最大
- 层级最高
- 过长时按布局省略或换行，避免撑破页面

### 原名

字段：

- `原名`

显示影片原名。

### 上映时间

字段：

- `上映时间`

显示影片上映日期或年份。

### 时长

字段：

- `时长`

显示影片时长。

---

## 8. 评分区

评分区包含两个评分卡片：

- TMDB 评分
- IMDb 评分

每个评分卡片包含：

- 语义图标
- 评分名称
- 分数
- 五星评分
- 票数

### 分数格式

示例：

- `X.X / 10`

### 五星评分

评分下面的区域显示五颗星星。

要求：

- 使用 5 颗星
- 支持半星显示
- 单位为半颗星
- 根据 10 分制评分换算为 5 星制
- 不使用截图中的黑色长方形占位样式

### 票数

字段：

- `票数`

显示评分票数。

---

## 9. 简介区

简介区包含：

- 标题：`简介`
- 简介内容

要求：

- 删除“展开”功能
- 不显示“展开”按钮
- 简介内容区域本身可滚动
- 简介过长时，只在简介区域内部滚动
- 不撑高整个页面
- 不影响播放源列表区域布局

简介区域滚动要求：

- 滚动条使用 WPF UI / 项目统一滚动条样式
- 简介内容为空时可以留空或显示轻量提示，按现有逻辑处理

---

## 10. 主创信息区

右上主创信息卡片包含：

- 导演
- 编剧
- 主演
- 国家 / 地区
- 语言
- 制片公司

显示方式：

```text
导演        字段值
编剧        字段值
主演        字段值
国家/地区   字段值
语言        字段值
制片公司     字段值
```

要求：

- 左侧为字段名
- 右侧为字段值
- 字段值过长时按布局换行或省略
- 不使用截图颜色

---

## 11. 标签区

标签区包含三类标签：

- 类型
- 情绪
- 场景

每个标签是胶囊标签，内部显示文字。

要求：

- 标签区域的黑色占位块应替换为真实标签胶囊
- 类型标签使用低饱和灰蓝
- 情绪标签使用灰粉 / 粉紫体系，并根据情绪变化
- 场景标签使用低饱和紫灰
- 标签数量过多时按布局换行或省略
- 不使用高饱和颜色
- 不使用图片作为标签

---

## 12. 操作按钮区

操作按钮区位于影片信息区下方、播放源区域上方。

按钮包括：

- 播放默认源
- 标记想看 / 标记喜爱 / 取消想看 / 取消喜爱
- 标记已看 / 标记未看
- 标记不想看 / 取消不想看
- 影片识别修正

按钮左侧的黑色正方形是语义相关图标。

要求：

- 图标使用 WPF UI 默认图标或补充图标库
- 图标与按钮语义对应
- 主操作按钮需要更突出
- 状态按钮需要有当前状态反馈
- 不使用截图中的黑色占位图标

---

## 13. 播放默认源按钮

按钮文案：

- `播放默认源`

点击后：

- 播放当前影片的默认播放源

说明：

- 不需要处理没有默认源的情况
- 默认源由后端或现有业务逻辑保证存在
- 播放逻辑按现有播放器逻辑处理

---

## 14. 观看状态按钮

观看状态按钮在以下两种状态间切换：

### 未看状态

按钮显示：

- `标记已看`

点击后：

- 将影片标记为已看
- 按钮变为 `标记未看`
- 偏好按钮从 `标记想看` 变为 `标记喜爱`
- 如果原本有想看状态，应按现有状态规则处理

### 已看状态

按钮显示：

- `标记未看`

点击后：

- 将影片标记为未看
- 按钮变为 `标记已看`
- 偏好按钮从 `标记喜爱` 变为 `标记想看`
- 如果原本有喜爱状态，应按现有状态规则处理

---

## 15. 想看 / 喜爱按钮

该按钮根据影片已看 / 未看状态动态变化。

### 未看影片

未看影片显示：

- `标记想看`

点击后：

- 标记想看
- 按钮变为 `取消想看`
- 如果当前是“不想看”，则取消不想看
- `取消不想看` 按钮变回 `标记不想看`

再次点击：

- 取消想看
- 按钮变回 `标记想看`

### 已看影片

已看影片显示：

- `标记喜爱`

点击后：

- 标记喜爱
- 按钮变为 `取消喜爱`
- 如果当前是“不想看”，则取消不想看
- `取消不想看` 按钮变回 `标记不想看`

再次点击：

- 取消喜爱
- 按钮变回 `标记喜爱`

---

## 16. 不想看按钮

不想看按钮在以下状态间切换：

### 普通状态

按钮显示：

- `标记不想看`

点击后：

- 标记不想看
- 按钮变为 `取消不想看`
- 如果原本是想看，则取消想看
- 如果原本是喜爱，则取消喜爱
- 想看 / 喜爱按钮变回对应的“标记想看”或“标记喜爱”

### 已经不想看

按钮显示：

- `取消不想看`

点击后：

- 取消不想看
- 按钮变回 `标记不想看`
- 想看 / 喜爱按钮按当前已看 / 未看状态显示

---

## 17. 偏好状态互斥规则

以下三个偏好状态互斥：

- 不想看
- 想看
- 喜爱

规则：

- 三者任意两个不可能共存
- 不想看会覆盖想看和喜爱
- 点击不想看时，必须取消想看或喜爱
- 点击想看时，必须取消不想看
- 点击喜爱时，必须取消不想看
- 未看影片使用想看
- 已看影片使用喜爱

---

## 18. 影片识别修正按钮

按钮文案：

- `影片识别修正`

点击后：

- 打开影片识别修正弹窗

说明：

- 弹窗内容见后续“影片识别修正弹窗”部分
- 不跳转新页面
- 不关闭详情页

---

## 19. 播放源区域

播放源区域位于页面底部。

标题：

- `播放源`

播放源区域分为两种状态：

1. 入库影片：显示播放源列表
2. 未入库影片：显示居中文字提醒

---

## 20. 未入库影片播放源区域

如果当前影片是未入库影片，则播放源区域不显示播放源列表。

显示方式：

- 在播放源区域居中显示文字提醒

提醒文案建议：

- `未入库影片暂无播放源`

或按现有业务文案处理。

要求：

- 文案居中
- 不显示空列表
- 不显示播放源表头
- 不显示播放源按钮
- 保持区域视觉稳定

---

## 21. 入库影片播放源列表

入库影片显示播放源列表。

播放源列表是可滚动区域。

表头字段包括：

- 格式
- 分辨率
- 帧率
- 码率
- 视频编码
- 音频编码
- 文件大小
- 识别状态
- 最近播放
- 上次播放位置

每条播放源行包含：

- 文件图标
- 文件名
- 文件路径
- 格式
- 分辨率
- 帧率
- 码率
- 视频编码
- 音频编码
- 文件大小
- 识别状态
- 最近播放
- 上次播放位置
- 播放按钮
- 默认源状态按钮
- 重置识别按钮

---

## 22. 播放源行按钮

每条播放源行右侧包含：

- 播放
- 已为默认 / 设为默认
- 重置识别

### 播放

按钮文案：

- `播放`

点击后：

- 播放该播放源

### 默认源按钮

默认播放源显示：

- `已为默认`

要求：

- 不可点击
- 视觉上表现为已选中 / 已禁用
- 不执行操作

非默认播放源显示：

- `设为默认`

点击后：

- 将该播放源设置为默认播放源
- 当前行变为 `已为默认`
- 原默认源变为 `设为默认`

### 重置识别

按钮文案：

- `重置识别`

点击后：

- 将该播放源重置为未识别影片
- 或清除当前播放源与影片的识别关联
- 具体业务逻辑按现有后端实现

---

## 23. 播放源列表滚动

播放源列表是可滚动区域。

要求：

- 播放源数量较多时，仅播放源列表内部滚动
- 不让整个页面被播放源撑高
- 表头保持清晰
- 滚动条使用 WPF UI / 项目统一样式

---

# 影片识别修正弹窗

## 24. 弹窗用途

影片识别修正弹窗用于根据文件名、路径、影片名和年份搜索 TMDB 或使用 AI 辅助识别，并将播放源修正为正确影片。

弹窗由影片详情页的 `影片识别修正` 按钮打开。

弹窗不是全屏页面。

---

## 25. 弹窗结构

弹窗包含：

1. 标题区
2. 文件信息区
3. 搜索区
4. 修正候选列表

结构：

```text
影片识别修正弹窗
├─ 标题区
│  ├─ 标题：影片识别修正
│  └─ 关闭 X
│
├─ 文件信息区
│  ├─ 文件名：字段值
│  └─ 文件路径：字段值
│
├─ 搜索区
│  ├─ 搜索影片名输入框
│  ├─ 上映年份输入框（可选）
│  ├─ 搜索 TMDB 按钮
│  └─ AI 辅助识别按钮
│
└─ 修正候选列表
   └─ 候选影片卡片
```

---

## 26. 弹窗标题区

标题：

- `影片识别修正`

右上角：

- 关闭 X

要求：

- 点击 X 关闭弹窗
- 支持 Esc 关闭更佳
- 关闭弹窗不影响详情页
- 弹窗视觉遵循 WPF UI Dialog / ContentDialog 风格

---

## 27. 文件信息区

文件信息区显示：

- 文件名
- 文件路径

你确认：

- 文件名和文件路径不是占位符
- 右侧应跟随具体字段值

显示格式：

```text
文件名：具体文件名
文件路径：具体文件路径
```

要求：

- 文件路径过长时省略
- 鼠标悬浮显示完整路径 Tooltip
- 文件名过长时省略或换行，按布局处理

---

## 28. 搜索区

搜索区包含：

- 搜索影片名输入框
- 上映年份输入框（可选）
- 搜索 TMDB 按钮
- AI 辅助识别按钮

### 搜索影片名

字段：

- `搜索影片名`

输入控件：

- 文本输入框

可带清空按钮。

### 上映年份

字段：

- `上映年份（可选）`

输入控件：

- 文本输入框或数字输入框

可带清空按钮。

### 搜索 TMDB

按钮文案：

- `搜索 TMDB`

点击后：

- 根据影片名和可选年份搜索 TMDB
- 搜索结果显示在候选列表

### AI 辅助识别

按钮文案：

- `AI 辅助识别`

点击后：

- 使用 AI 根据文件名 / 文件路径辅助识别
- 结果显示在候选列表
- 具体业务逻辑按现有后端处理

---

## 29. 修正候选列表

修正候选列表是可滚动区域。

要求：

- 候选数量较多时，仅候选列表内部滚动
- 不让弹窗无限增高
- 滚动条使用 WPF UI / 项目统一样式

---

## 30. 候选影片卡片

每个候选影片卡片包含：

- 影片名
- 原名
- 简介
- 年份
- 国家
- TMDB ID
- 操作按钮

结构示例：

```text
候选影片卡片
├─ 影片名
├─ 原名
├─ 简介
├─ 年份
├─ 国家
├─ TMDB ID
└─ 选择影片 / 已修正为该影片
```

### 简介

候选列表中的简介不限制行数。

但要求：

- 简介内容区域内部可滚动
- 不让候选卡片无限增高
- 不让整个弹窗被单条简介撑开
- 简介滚动条应轻量

### 操作按钮

如果该候选不是当前修正影片：

- 显示 `选择影片`

点击后：

- 将当前播放源修正为该影片
- 不关闭弹窗
- 当前候选按钮变为 `已修正为该影片`
- 详情页数据按现有逻辑刷新或待关闭弹窗后刷新

如果该候选已经是当前修正影片：

- 显示 `已修正为该影片`

要求：

- 已修正按钮不可点击或表现为已选中状态
- 其他候选仍可点击选择
- 选择其他候选后，新的候选变为 `已修正为该影片`

---

## 31. 可滚动区域

该模块明确包含两个可滚动区域：

1. 播放源列表
2. 修正候选列表

另外：

- 简介区域内部可滚动
- 候选卡片中的简介内容区域内部可滚动

要求：

- 页面整体不应因为播放源数量过多被撑开
- 弹窗整体不应因为候选数量过多被撑开
- 滚动区域边界要清晰
- 不要给不需要滚动的区域添加滚动

---

## 32. 状态处理

### 无海报

使用：

- `poster-placeholder.png`

### 未入库影片

播放源区域显示居中文字提醒，不显示播放源列表。

### 评分缺失

如果 TMDB 或 IMDb 评分缺失：

- 分数可留空或显示轻量提示
- 星级评分为空态
- 不显示错误样式

### 标签缺失

标签缺失时：

- 对应标签行可以留空
- 不显示无意义占位文字

### 搜索中

影片识别修正搜索中：

- 搜索按钮可显示 loading
- 候选列表可显示轻量加载状态

### 无候选结果

修正候选列表无结果时：

- 使用 EmptyState 组件
- 不使用独立插画图片

## 38. 验收标准

该页面完成后，需要满足：

### 基础验收

- 从媒体库、首页或其他影片入口可以进入影片详情页
- 页面左上角显示返回按钮
- 点击返回按钮可以返回上一页
- 页面显示影片海报
- 海报缺失时使用 poster-placeholder.png
- 页面显示影片名、原名、上映时间、时长
- 页面支持浅色 / 深色主题
- 项目可以编译通过

### 评分验收

- 显示 TMDB 评分卡片
- 显示 IMDb 评分卡片
- 每个评分卡片显示分数和票数
- 每个评分卡片显示五颗星星
- 星星支持半星显示

### 信息和标签验收

- 显示导演、编剧、主演、国家 / 地区、语言、制片公司
- 显示简介区域
- 简介区域内部可滚动
- 不显示“展开”按钮
- 显示类型、情绪、场景三类标签
- 标签以胶囊形式展示

### 状态按钮验收

- 未看影片显示“标记已看”
- 点击“标记已看”后变为“标记未看”
- 未看影片显示“标记想看”
- 已看影片显示“标记喜爱”
- 点击想看后显示“取消想看”
- 点击喜爱后显示“取消喜爱”
- 点击不想看后显示“取消不想看”
- 不想看会取消想看或喜爱
- 点击想看或喜爱会取消不想看
- 不想看、想看、喜爱三者互斥，任意两个不能共存

### 播放源验收

- 入库影片显示播放源列表
- 播放源列表可滚动
- 播放源列表显示格式、分辨率、帧率、码率、视频编码、音频编码、文件大小、识别状态、最近播放、上次播放位置
- 每个播放源显示文件名和文件路径
- 每个播放源有播放按钮
- 默认播放源显示“已为默认”
- “已为默认”不可点击
- 非默认播放源显示“设为默认”
- 点击“设为默认”后该播放源变为默认源
- 每个播放源有重置识别按钮
- 未入库影片播放源区域显示居中文字提醒，不显示播放源列表

### 识别修正弹窗验收

- 点击“影片识别修正”打开识别修正弹窗
- 弹窗右上角显示关闭 X
- 弹窗显示文件名和文件路径字段值
- 文件路径过长时省略并支持 Tooltip
- 弹窗包含搜索影片名输入框
- 弹窗包含上映年份可选输入框
- 弹窗包含搜索 TMDB 按钮
- 弹窗包含 AI 辅助识别按钮
- 修正候选列表可滚动
- 候选卡片显示影片名、原名、简介、年份、国家、TMDB ID
- 候选简介区域内部可滚动
- 点击“选择影片”后不关闭弹窗
- 被选择的候选按钮变为“已修正为该影片”
- “已修正为该影片”表现为已选中或不可点击状态
