# 影片详情页 UI 实施说明

## 1. 模块名称

影片详情页

建议文件名：

- `movie-detail-page.md`

建议实现对象：

- `MovieDetailPage`
- 或 `MovieDetailPage.xaml`

---

## 2. 页面用途

影片详情页用于展示单部影片的完整信息，并提供播放、状态标记、播放源管理和影片识别修正能力。

用户可以从以下入口进入影片详情页：

- 首页继续观看影片海报
- 首页最近新增影片海报
- 首页 AI 推荐影片海报或电影名
- 媒体库海报卡片
- 媒体库列表项
- 影片发现中的 Movie 搜索或榜单项
- 影片发现 AI 推荐 Tab / 内部兼容推荐入口
- 观影历史中的 Movie 历史项
- 收藏夹中的 Movie 收藏项
- 修正完成后跳转到 Movie detail 的结果入口
- 无播放源、metadata-only 或外部候选电影的详情承载入口

该页面是完整页面，不是弹窗。

## 3.1 Phase 6.0d-B 新版覆盖规则

本节根据详情链路审计后的确认结果补充并覆盖本文中与其冲突的旧假设。

- 电影详情继续采用旧设计的信息结构与视觉层级：海报、标题、原名、年份、时长、评分、简介、类型标签、播放动作和用户状态均保留。
- 导演、编剧、演员等主创字段在数据可返回时按旧设计展示；整组数据均缺失时隐藏主创卡片，只有个别字段缺失时仅隐藏该字段行，不显示伪造占位内容。
- 详情页允许存在 `无播放源`、`无默认源`、`仅 metadata`、`外部候选`、`已移出可恢复` 与播放按钮 disabled 状态；不再假设默认播放源始终存在。
- 详情页最终设计不展示字幕状态，不新增在线字幕入口；字幕选择和在线字幕搜索属于播放器链路。
- 修正入口保留弹窗式体验，并统一引用 `correction-flow.md`。当前成品若存在详情页内修正 Tab / 面板，仅记录为当前实现，不作为 Phase 7 最终结构。
- 详情页不新增删除记录或移出媒体库入口。若当前对象已有加入、恢复等入口，仅按现有事实承载，不扩大业务范围。
- Phase 6.0i 审计确认当前 `MovieDetailPage` 成品尚未呈现返回按钮，也未暴露通用返回命令；本文原有返回按钮为目标设计而非当前已实现事实。
- Phase 7 中电影详情与剧、季、集详情统一使用紧凑详情标题栏左侧的返回图标按钮；无播放源、metadata-only、外部候选和修正跳转后的电影详情同样显示该按钮。

## 4. 页面整体结构

影片详情页主要包含：

1. 返回按钮
2. 海报区
3. 影片基础信息区
4. 评分区
5. 主创信息区
6. 标签区
7. 简介区
8. 操作按钮区
9. 播放源区域

整体结构：

```text
影片详情页
├─ 返回按钮
├─ 海报区
├─ 影片信息区
│  ├─ 影片名
│  ├─ 原名
│  ├─ 上映时间
│  ├─ 时长
│  ├─ TMDB 评分
│  ├─ IMDb 评分
│  └─ 简介
│
├─ 右侧信息区
│  ├─ 主创信息
│  └─ 标签
│
├─ 操作按钮区
│  ├─ 播放默认源
│  ├─ 标记想看 / 标记喜爱 / 取消想看 / 取消喜爱
│  ├─ 标记已看 / 标记未看
│  ├─ 标记不想看 / 取消不想看
│  └─ 影片识别修正
│
└─ 播放源区域
   ├─ 入库影片：播放源列表
   └─ 未入库影片：居中文字提醒
```

## 5. 返回按钮

页面 Hero / 内容标题区左上角的正方形区域是统一详情返回按钮，不是 App Logo，也不是侧边栏展开 / 收起按钮。

当前成品事实：

- `MovieDetailPage` 当前没有可见返回按钮。
- `MovieDetailViewModel` 当前没有统一返回命令。
- 当前 `NavigationStateService` 可请求进入详情，但不保存来源页面或返回栈。

Phase 7 目标规格：

要求：

- 使用 WPF UI 返回箭头图标或等效图标的紧凑图标按钮
- 位置固定为电影详情路由紧凑标题栏左侧，不占用全局壳层左上角的侧边栏按钮位置
- 默认 Tooltip 为 `返回上一页`；仅在进入 fallback 路径时可明确为 `返回媒体库`
- 点击后优先返回进入本详情的上一界面，并恢复其可保存的 Tab、筛选、榜单、推荐或滚动状态
- 不显示 App Logo
- 深色 / 浅色主题下都要清晰可见
- 不抢占页面主视觉
- hover、pressed、focus、disabled 和 Tooltip 视觉使用全局统一详情返回按钮基线

返回行为：

1. 有可靠来源状态时，返回媒体库、影片发现搜索 / 榜单、AI 推荐 Tab、首页、观影历史或收藏夹中的原入口状态。
2. 由修正完成跳转至电影详情时，有可靠来源则优先返回来源；没有来源时返回媒体库。
3. 没有可靠上一界面或来源已失效时，电影详情 fallback 返回媒体库。

实现差距：

- 当前导航服务未记录来源路由、来源页面状态或返回栈，上述来源恢复属于 Phase 7 实施前需要补齐的导航能力。

字段：

- `原名`

显示影片原名。

### 上映时间

字段：

- `上映时间`

显示影片上映日期或年份。

### 时长

字段：

- `时长`

显示影片时长。

## 8. 评分区

评分区包含两个评分卡片：

- TMDB 评分
- IMDb 评分

每个评分卡片包含：

- 语义图标
- 评分名称
- 分数
- 五星评分
- 票数

### 分数格式

示例：

- `X.X / 10`

### 五星评分

评分下面的区域显示五颗星星。

要求：

- 使用 5 颗星
- 支持半星显示
- 单位为半颗星
- 根据 10 分制评分换算为 5 星制
- 不使用截图中的黑色长方形占位样式

### 票数

字段：

- `票数`

显示评分票数。

简介区包含：

## 11. 标签区

标签区包含三类标签：

- 类型

操作按钮区位于影片信息区下方、播放源区域上方。

按钮包括：

- 播放默认源

- 标记不想看 / 取消不想看
- 影片识别修正

按钮左侧的黑色正方形是语义相关图标。

要求：

- 图标使用 WPF UI 默认图标或补充图标库
- 图标与按钮语义对应
- 主操作按钮需要更突出
- 状态按钮需要有当前状态反馈
- 不使用截图中的黑色占位图标

## 13. 播放默认源按钮

按钮文案：

- `播放默认源`

点击后：

- 播放当前影片的默认播放源

说明：

- 存在默认源时，点击后播放默认源
- 有播放源但没有默认源时，按钮 disabled，并提示先设置默认播放源或从播放源列表播放
- 没有播放源或仅有 metadata 时，按钮 disabled，并显示不可播放原因
- 播放逻辑按现有播放器逻辑处理

---

## 14. 观看状态按钮

观看状态按钮在以下两种状态间切换：

### 未看状态

按钮显示：

- `标记已看`

点击后：

- 将影片标记为已看
- 按钮变为 `标记未看`
- 偏好按钮从 `标记想看` 变为 `标记喜爱`
- 如果原本有想看状态，应按现有状态规则处理

### 已看状态

按钮显示：

- `标记未看`

点击后：

- 将影片标记为未看
- 按钮变为 `标记已看`
- 偏好按钮从 `标记喜爱` 变为 `标记想看`
- 如果原本有喜爱状态，应按现有状态规则处理

---

## 15. 想看 / 喜爱按钮

该按钮根据影片已看 / 未看状态动态变化。

### 未看影片

未看影片显示：

- `标记想看`

点击后：

- 标记想看
- 按钮变为 `取消想看`
- 如果当前是“不想看”，则取消不想看
- `取消不想看` 按钮变回 `标记不想看`

再次点击：

- 取消想看
- 按钮变回 `标记想看`

### 已看影片

已看影片显示：

- `标记喜爱`

点击后：

- 标记喜爱
- 按钮变为 `取消喜爱`
- 如果当前是“不想看”，则取消不想看
- `取消不想看` 按钮变回 `标记不想看`

再次点击：

- 取消喜爱
- 按钮变回 `标记喜爱`

---

## 16. 不想看按钮

不想看按钮在以下状态间切换：

### 普通状态

按钮显示：

- `标记不想看`

点击后：

- 标记不想看
- 按钮变为 `取消不想看`
- 如果原本是想看，则取消想看
- 如果原本是喜爱，则取消喜爱
- 想看 / 喜爱按钮变回对应的“标记想看”或“标记喜爱”

### 已经不想看

按钮显示：

- `取消不想看`

点击后：

- 取消不想看
- 按钮变回 `标记不想看`
- 想看 / 喜爱按钮按当前已看 / 未看状态显示

---

## 17. 偏好状态互斥规则

以下三个偏好状态互斥：

- 不想看
- 想看
- 喜爱

规则：

- 三者任意两个不可能共存
- 不想看会覆盖想看和喜爱
- 点击不想看时，必须取消想看或喜爱
- 点击想看时，必须取消不想看
- 点击喜爱时，必须取消不想看
- 未看影片使用想看
- 已看影片使用喜爱

## 19. 播放源区域

播放源区域位于页面底部。

## 20. 未入库影片播放源区域

如果当前影片没有活动播放源，或处于只有 metadata 的详情承载状态，则播放源区域不显示播放源列表。

显示方式：

- 在播放源区域居中显示文字提醒

提醒文案建议：

- `暂无播放源`
- `已有影片资料，但当前不可播放`

或按现有业务文案处理。

要求：

- 文案居中
- 不显示空列表
- 不显示播放源表头

- 保持区域视觉稳定

## 22. 播放源行按钮

每条播放源行右侧包含：

- 播放
- 已为默认 / 设为默认
- 探测
- 拆分来源（如当前对象支持）
- 修正

### 播放

按钮文案：

- `播放`

点击后：

- 播放该播放源

### 默认源按钮

默认播放源显示：

- `已为默认`

要求：

- 不可点击
- 视觉上表现为已选中 / 已禁用
- 不执行操作

非默认播放源显示：

- `设为默认`

点击后：

- 将该播放源设置为默认播放源
- 当前行变为 `已为默认`
- 原默认源变为 `设为默认`

### 探测

按钮文案：

- `立即探测`

点击后：

- 探测当前播放源信息
- 该动作属于普通操作，不需要确认弹窗

- 将播放源从当前对象中拆分属于警示确认操作
- 必须引用 `global-dialogs.md` 的来源拆分警示确认规格
- 确认文案明确该动作不会删除真实本地文件或 WebDAV 文件

### 修正

- 使用 `correction-flow.md` 定义的统一修正弹窗
- 不在播放源行展开常驻修正面板

# 影片识别修正弹窗

## 24. 弹窗用途

影片识别修正弹窗用于根据文件名、路径、影片名和年份搜索 TMDB 或使用 AI 辅助识别，并将播放源修正为正确影片。

弹窗由影片详情页的 `影片识别修正` 按钮打开。

弹窗不是全屏页面。

---

## 25. 弹窗结构

弹窗包含：

1. 标题区
2. 文件信息区
3. 搜索区
4. 修正候选列表

结构：

```text
影片识别修正弹窗
├─ 标题区
│  ├─ 标题：影片识别修正
│  └─ 关闭 X
│
├─ 文件信息区
│  ├─ 文件名：字段值
│  └─ 文件路径：字段值
│
├─ 搜索区
```

## 32. 状态处理

### 无海报

使用：

- `poster-placeholder.png`

### 未入库影片

播放源区域显示居中文字提醒，不显示播放源列表。

### 无播放源 / metadata-only

- 详情信息仍可正常展示
- 播放按钮 disabled，并显示 `暂无可播放来源` 或等价中文提示
- metadata-only 表示有影片资料但当前没有活动播放源，不作为错误状态

### 有播放源但无默认源

- 播放源列表仍然显示
- 主播放按钮 disabled，提示用户在列表中选择播放源或设置默认源
- 不自动假定第一条来源为默认源

### 外部候选 / 已移出可恢复

- 外部候选显示为可浏览但暂不可播放的详情承载状态
- 已移出可恢复只显示当前已经存在的恢复或加入动作，不新增删除或移出功能

### 加载、错误与 disabled

- 加载中保持详情骨架区域稳定，并禁用依赖数据的动作
- 加载失败显示中文错误提示和当前已有的重试动作
- disabled 按钮必须提供不可操作原因，不仅依赖颜色表达

### 评分缺失

如果 TMDB 或 IMDb 评分缺失：

- 分数可留空或显示轻量提示
- 星级评分为空态
- 不显示错误样式

### 标签缺失

标签缺失时：

- 对应标签行可以留空
- 不显示无意义占位文字

### 搜索中

影片识别修正搜索中：

- 搜索按钮可显示 loading
- 候选列表可显示轻量加载状态

### 无候选结果

修正候选列表无结果时：

- 使用 EmptyState 组件
- 不使用独立插画图片

### 字幕范围

- 详情页不展示本地字幕状态
- 详情页不新增在线字幕入口
- 字幕选择、绑定和在线搜索统一由播放器链路承载

## 39. Phase 7.3 后续：影片修正弹窗结构覆盖

本节覆盖前文中仍将修正弹窗描述为单一 Movie 表单的旧规格。

当前影片修正弹窗采用统一标题区和单卡片内容区：

- 标题使用较大的 `影片识别修正` 文案，右上角只保留关闭图标按钮。
- 标题下方使用文件图标和播放源下拉框；默认显示默认播放源，下拉项展示包含文件名的完整播放源路径。
- 修正卡片第一行始终保留目标类型下拉框，支持 `修正为电影 / 修正为电视剧集 / 加入已有未识别季`。
- Movie 模式没有底部操作按钮时不保留 footer 空白；修正卡片下边缘与弹窗边框的距离和标题上边缘与弹窗边框的距离一致。
- 工具栏与结果列表之间保留状态文本行；无状态文本时折叠，不占布局空间。
- 输入框使用弱化占位提示；有内容时显示末尾清空图标按钮。
- 结果列表使用分隔线，不为每一行增加独立圆角背景卡片；列表和候选简介使用现代化自动显示滚动条。

三种模式的结果结构：

- Movie：显示 2:3 海报、片名、原名、内部滚动简介、上映日期、时长、导演、国家 / 地区、语言和 `修正为TA / 已为该影片`。
- TV Episode：显示可展开电视剧行和季行；电视剧行显示剧名、原名、首播日期、总季数、导演、类型、国家 / 地区和语言；季行显示黑点、剧名、季号、首播日期、集数和修正动作。Season 0 显示为 `特别篇`。
- Unknown Season：直接在当前弹窗内联展示已有未识别剧 / 季，不再要求打开第二层选择弹窗；选中目标季后输入集号并执行 `加入到该季`。

数据边界：

- Movie 候选展示复用已有 TMDB Movie 候选字段。
- Movie / TV 候选会在搜索后补充详情字段。Movie 最多补充 10 条，TV 最多补充 12 条，并发上限为 4；单条详情失败时保留基础搜索结果，不清空整批候选。
