# 媒体库页面 UI 实施说明

## 1. 模块名称

媒体库

建议文件名：

- `media-library-page.md`

建议实现对象：

- `LibraryPage`
- `LibraryViewModel`

关联规格：

- `global-shell.md`
- `global-dialogs.md`
- `media-library-special-items.md`

---

## 2. 页面用途

媒体库页面用于浏览、搜索、筛选、排序和批量管理当前媒体内容。

新版设计覆盖：

- 电影
- 电视剧总览
- 电视剧季
- 未识别媒体和其它项目
- 有播放源 / 无播放源项目
- 已移出媒体库项目的查看与恢复

本规格只定义 UI 结构、状态和交互语义，不新增或修改业务功能。

媒体库业务语义必须保持：

- 移出媒体库只影响可见性 / hide，不删除真实文件，不清除用户状态，不删除 metadata
- 删除记录只清除软件中的记录及相关软件状态，不删除真实本地文件或 WebDAV 文件

## 5. 页头与全局入口

媒体库是普通主页面，页面标题和副标题遵循 `global-shell.md` 的壳层标题关系。

标题建议：

- 页面标题：`媒体库`
- 页面副标题：`查看你的影片资源`

### 扫描入口

媒体库页头最终不保留 `扫描媒体库` 快捷按钮。

要求：

- 扫描任务继续由全局导航、账号菜单或既有流程入口承载
- 不在媒体库页头新增扫描动作
- 不根据旧截图恢复页内扫描按钮

### 主题入口

主题切换属于全局壳层工具，不作为媒体库私有按钮重新设计。

## 6. 导航栏状态与批量入口

页面支持全局侧边栏的展开和收起状态。

### 展开导航栏状态

- 显示完整主导航
- 当前选中项为媒体库
- 支持搜索、筛选、海报 / 列表查看
- 按旧设计，不显示批量选择入口和批量工具栏

### 收起导航栏状态

- 页面内容区扩展
- 显示 `批量选择` 入口
- 支持进入批量选择模式

### 批量规则

批量入口采用旧设计的壳层关系：

- 批量选择只在侧边栏收起状态出现
- 进入批量模式后，按钮文案改为 `完成`
- 批量模式期间卡片点击用于选择，不进入详情
- 退出批量模式后恢复普通卡片详情入口

该规则是 Phase 7 的目标设计；如果当前成品在展开状态亦可进入批量模式，应在 Phase 7 统一到本规格。

## 7. 搜索、筛选与排序

### 搜索

搜索沿用旧设计的提交式交互。

组件：

- 搜索输入框
- `搜索` 按钮
- 可选清空输入动作

提示文案：

- `搜索影片、导演、演员`

触发方式：

- 点击 `搜索` 按钮后执行搜索
- 在搜索输入框中按 Enter 后执行搜索
- 输入文字本身不触发即时搜索

要求：

- 搜索执行后更新媒体结果区和结果摘要
- 清除筛选时同时清除已提交的搜索条件
- Phase 7 不沿用当前输入即筛选行为

### 内容类型筛选

内容类型筛选必须包含：

- 全部
- 电影
- 电视剧
- 其他

其中：

- 普通模式下 `电视剧` 结果优先展示 Series 总览
- 批量模式下 `电视剧` 结果展开至可操作的 Season 粒度
- `其他` 包含未识别、placeholder 和其它待处理媒体项

### 播放源状态与来源筛选

旧设计中的 `库内影片 / 库外影片` 范围不纳入新版设计。

筛选结构改为：

- 播放源状态：全部 / 有播放源 / 无播放源
- 来源类型：全部来源 / 本地 / 网盘；如果后续来源类型增加，可沿同级扩展

说明：

- `无播放源` 是可见状态筛选，也是结果摘要统计项
- `已移出媒体库` 不通过筛选范围表达，而通过独立面板管理

### 其它筛选

保留并结合当前能力补齐：

- 标签：类型 / 情绪 / 场景
- 年代
- 收藏状态：全部 / 喜爱 / 想看 / 不想看
- 观看状态：全部 / 已看 / 未看，必要时可纳入不想看语义

### 不提供的可见筛选

新版设计不提供可见的 `识别状态` 筛选入口。

说明：

- 后端或 ViewModel 可继续保留内部能力
- 不在 toolbar、下拉菜单或空状态中提示用户可按识别状态筛选
- 未识别项仍通过内容类型、卡片状态和详情链路可见

### 排序与清除筛选

排序至少覆盖当前可用方向：

- 最近更新
- 标题
- 年份
- 评分
- 顺序切换

`清除筛选` 操作：

- 清除已提交的搜索关键词
- 重置内容类型、播放源、来源、标签、年代、收藏和观看状态
- 重置排序到默认状态
- 刷新结果区和统计摘要

## 8. 结果摘要与工具动作

结果摘要沿用旧设计的单行结构，并调整统计语义。

结构：

```text
共找到 xx 项媒体 · 已看 xx · 未看 xx · 无播放源 xx        [已移出媒体库] [切布局] [批量选择/完成]
```

要求：

- 第三项固定为 `无播放源`，不显示旧设计中的 `未识别`
- 统计随已提交搜索和当前筛选结果变化
- 摘要覆盖 Movie / TV / Other 的当前展示粒度
- 普通模式的 TV 统计按 Series 结果项口径呈现
- 批量模式的 TV 统计按展开后的 Season 结果项口径呈现

统计不可用时的设计规则：

- 如果某类结果暂时无法提供稳定的已看或未看聚合口径，对应值显示 `--`
- `--` 使用次要文字色，并可提供 tooltip：`当前类型暂不支持该统计`
- 不因统计不可用而隐藏整个摘要位置，避免布局跳动
- `无播放源` 只要能判断活动播放源状态，就正常显示数值

动作顺序：

- `已移出媒体库` 按钮位于旧设计 `切布局` 按钮左侧
- `切布局` 位于批量入口之前
- `批量选择` 仅在侧边栏收起状态可见

## 9. 海报 / 列表布局

### 布局切换

媒体库采用旧设计的真实布局切换：

- 海报布局
- 列表布局

Phase 7 必须实现：

- 点击 `切布局` 后在两种真实布局之间切换
- 记住用户最后选择的布局
- 下次进入媒体库时恢复该布局
- 切换布局不清除搜索、筛选、选中状态或统计口径

当前成品中的布局占位提示不是最终设计。

### 结果区滚动与大列表

- 媒体结果区独立滚动
- 海报网格可采用虚拟化布局
- 列表布局应在大数据量下保持字段可读和操作可见
- 加载与 render-ready 状态见第 16 节

## 10. 卡片与列表字段矩阵

海报卡片延续旧设计的视觉基线：

- 海报或统一占位图
- 左上来源 / 播放源状态信息
- 右上用户状态或待处理状态
- 标题与关键辅助信息
- 标签、观看进度或类型摘要
- 普通模式下可进入详情

列表视图与海报视图使用相同的数据语义，但可以展示更高信息密度。

| 内容类型 | 海报卡片必显字段 | 列表项可追加字段 | 状态 / 标签 | 普通模式详情入口 |
|---|---|---|---|---|
| Movie | 电影名、海报、年份 / 日期、来源、用户状态、观看进度 | 原名、导演、演员、时长、评分、类别 / 情绪 / 场景标签、进度百分比 | 已看、未看、喜爱、想看、不想看、无播放源 | 电影详情页 |
| Series | 剧名、海报、首播年、总季数、有播放源季数、用户状态摘要 | 类型标签、总体来源摘要、整体观看摘要 | 电视剧、无播放源、用户状态摘要 | 剧集总览详情页 |
| Season | 剧名 + 季号、季名、海报、集数、播放源状态、用户状态 | 已看集数 / 总集数、来源数量、年份、识别提示 | 电视剧季、无播放源、用户状态 | 电视剧季详情页 |
| Unknown Movie / Orphan | 文件名或推测标题、占位海报、来源摘要 | 文件来源、可用播放源数、识别说明 | 未识别电影、待修正、无播放源 | 未识别电影对应详情页 |
| Unknown Season / Grouped Placeholder | 剧名候选或分组名、季号（如有）、占位海报、文件数、集号覆盖摘要 | 来源摘要、候选范围、集数 / 文件数、识别说明 | 未识别季、待修正、无播放源 | Grouped 先进入未识别剧详情，再进入未识别季详情；Unknown Season 进入未识别季详情 |
| Other | 显示标题 / 文件摘要、占位海报、来源状态 | 类型原因、来源数量、可用信息 | 其他、placeholder、未识别、无播放源 | 对应 Other / 占位详情页 |

### 字段原则

- Movie 卡片采用旧设计信息层级，保留导演、演员、时长、进度、评分和标签等可表达字段
- Series 与 Season 使用和 Movie 一致的海报、标题、来源、状态语言，但以剧集语义替换电影专属字段
- 未识别项不得因为 metadata 不完整而失去详情入口
- 无海报统一使用 `poster-placeholder.png`
- 字段不可用时显示为空或明确的状态标签，不用虚构内容补位

### 入口原则

- 普通模式下，每一种媒体卡片和列表项都有详情入口
- 未识别电影、未识别季、grouped placeholder、orphan 和 Other 也进入各自对应详情页
- 修正动作属于详情页内部操作
- 人工聚合属于批量模式操作
- 媒体库普通卡片上不新增 `修正` 或 `聚合` 主入口

特殊项目详情和状态说明见 `media-library-special-items.md`。

## 11. 状态标签

通用用户状态沿用既有优先级：

- 已看项目：不想看 > 喜爱 > 已看
- 未看项目：不想看 > 想看 > 未看
- 不想看、想看、喜爱相互排斥

系统或数据状态可以和用户状态并列展示，但不得互相替代：

- 无播放源
- 未识别 / 待修正
- 未识别季
- 其他

要求：

- 用户状态回答“我如何标记了该内容”
- 数据状态回答“该内容现在能否播放或是否需要补充信息”
- 无播放源不能被误写为未识别
- 未识别不等于无详情

## 12. 已移出媒体库面板与恢复

### 入口

`已移出媒体库` 是摘要工具区的独立按钮：

- 位置在 `切布局` 左侧
- 不作为 `库内 / 库外影片` 筛选项存在

### 面板结构

```text
RemovedLibraryPanel
├─ Title: 已移出媒体库
├─ StatusMessage
├─ Empty / Loading / Error State
└─ RemovedGroups
   ├─ Movie / Other item
   └─ TV group
      ├─ Season items
      ├─ RestoreAll
      └─ DeleteAllRecords
```

覆盖对象：

- Movie
- Season
- 通过当前能力形成的 MediaFile / placeholder / Other 隐藏项
- TV 分组项

单项动作：

- `恢复`
- `详情`
- `删除记录`

分组动作：

- `恢复全部`
- `删除全部记录`

### 业务语义

- 移出媒体库只是隐藏项目，不删除真实文件，不清用户状态，不删除 metadata
- 恢复会使该项目重新出现在媒体库可见结果中
- 删除记录会清除软件记录和关联的软件状态
- 删除记录不删除真实本地文件或 WebDAV 文件

### 面板状态

- Loading：正在加载已移出项目
- Empty：暂无已移出媒体库项目
- Error：加载失败，可重试或关闭面板
- Disabled：操作运行期间禁用重复提交

## 13. 批量选择与批量操作

### 批量模式结构

批量模式沿用旧设计，仅在侧边栏收起状态可进入。

进入后：

- `批量选择` 改为 `完成`
- 卡片或列表项显示选择控件
- 展示当前选中数量
- 显示批量工具栏
- 卡片点击切换选中，不进入详情

工具栏包含：

- `全选当前列表`
- `清空选择`
- `标记为已看`
- `标记为未看`
- `AI 辅助识别`
- `取消`，仅操作可取消时显示或启用
- `移出媒体库`
- `删除记录`
- `人工聚合为季`，仅选中项目满足条件时可用

无选中项目时：

- 业务动作按钮 disabled
- `全选当前列表` 可在列表非空时启用
- disabled 状态必须明确可辨

### TV 展示粒度

用户确认的 TV 粒度规则：

- 普通模式展示 Series，总览适合浏览与进入详情
- 批量模式展开为 Season，批量操作作用于可执行的季粒度

视觉提示：

- 进入批量模式时显示轻量提示：`电视剧已按季展开，批量操作作用于具体季度。`
- Season 卡片显示剧名、季号和季名，避免被误认为重复 Series
- 不在批量栏提供 Series 总览级删除或移出歧义入口

### 批量反馈状态

批量操作必须覆盖：

- 未开始 / 无选择
- 操作进行中
- 可取消中
- 已取消
- 全部成功
- 部分成功 / 部分失败
- 全部失败

规则：

- 操作运行中禁用会导致重复提交或选中集变化的动作
- 部分失败时保留失败项的选中状态，方便再次处理
- 结果摘要指出成功、跳过、失败和取消数量

## 14. 批量危险操作与 AI

### 批量移出媒体库

- 需要确认弹窗
- 使用 `global-dialogs.md` 的移出媒体库确认变体
- 确认文案明确 hide-only、可恢复、保留用户状态和 metadata、不会删除真实文件

### 批量删除记录

- 需要二次确认
- 使用 `global-dialogs.md` 的危险确认变体
- 文案明确清软件记录，不删除真实本地文件或 WebDAV 文件

### 批量 AI 辅助识别

覆盖状态：

- 准备处理单元
- 正在处理，显示进度和当前项目
- 成功
- 跳过 / 无结果
- 失败
- 取消中 / 已取消
- 部分失败保留项

AI 识别结果只改变当前既有业务流程允许的识别结果，不在本设计文档扩大业务语义。

### 人工聚合为未识别季

- 仅作为批量模式工具栏动作存在
- 不作为普通卡片的主入口
- 弹窗结构、资格条件和状态见 `media-library-special-items.md`

## 15. 空状态、错误态与 disabled

页面使用统一公共状态组件，不使用独立插画。

| 状态 | 展示区域 | 建议文案 / 交互 |
|---|---|---|
| 无媒体数据 | 结果区 | `暂无媒体`；引导用户通过全局扫描入口添加内容 |
| 筛选无结果 | 结果区 | `没有找到符合条件的媒体`；提供清除筛选按钮 |
| 无播放源筛选无结果 | 结果区 | `没有找到无播放源项目` |
| 加载失败 | 结果区 | 说明加载失败，可提供重试 |
| 已移出面板为空 | 已移出面板 | `暂无已移出媒体库项目` |
| 批量按钮 disabled | 批量栏 | 未选择、项目类型不适用或操作运行中时置灰 |
| 特殊项详情 fallback | 项目入口 | 只在无法定位详情对象的边界情况下显示提示；不得将 grouped placeholder 定义为无详情项 |

状态视觉遵循全局 `EmptyState`、`Loading`、`Error` 和 `Disabled` 基线。

## 16. 性能与大列表状态

媒体库可能承载大量项目，设计必须预留以下状态：

- 初次加载：结果区展示 Loading 状态，保留工具栏结构
- 筛选中 / 搜索中：提交查询后提供轻量忙碌反馈，避免重复提交
- Render-ready：列表已可浏览时结束加载反馈，不让用户等待诊断信息
- 批量运行中：锁定相关动作并展示进度和取消状态
- 部分失败：反馈失败数量，并保留需要重新处理的项目

说明：

- 海报网格可以使用虚拟化实现，不改变卡片设计语义
- Skeleton 可以作为 Phase 7 Deferred 方案评估，不要求必须实现
- 本规格不修改当前性能实现

## 19. 验收标准

### 页面与工具区

- 媒体库页头没有页内 `扫描媒体库` 快捷按钮
- 扫描可继续通过全局 / 导航入口进入
- 搜索仅在点击按钮或按 Enter 后执行
- 内容类型包含全部、电影、电视剧、其他
- 筛选包含有播放源 / 无播放源及来源类型
- 不存在可见识别状态筛选入口
- 不存在库内 / 库外影片范围筛选

### 结果、布局和卡片

- 摘要显示已看、未看、无播放源
- `已移出媒体库` 位于 `切布局` 左侧
- 海报 / 列表能够真实切换并记忆布局
- Movie / Series / Season / Unknown / Other 按字段矩阵展示
- 所有类型项目在普通模式都有详情入口
- 媒体库卡片没有修正或人工聚合主入口

### 批量与隐藏项管理

- 批量入口只在侧边栏收起状态出现
- 普通模式 TV 展示 Series，批量模式 TV 展开为 Season
- 批量栏覆盖全选、清空、状态标记、AI、移出、删除、取消和人工聚合
- 已移出面板覆盖恢复、详情和删除记录操作
- 移出确认说明 hide-only 和可恢复
- 删除确认说明只清软件记录且不删除真实文件

### 状态与性能

- 空、加载、错误、disabled、批量运行中和部分失败均有设计覆盖
- 大列表存在 render-ready 反馈规则
- 深色 / 浅色主题均遵循全局 token 和公共控件基线
