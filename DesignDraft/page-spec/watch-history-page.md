## 1. 模块名称

观影历史

- `watch-history-page.md`

## 2. 页面用途

观影历史页面用于按日期展示用户的历史观影记录。

该页面展示用户曾经观看过的电影与剧集单集记录，并显示观看进度、来源可用性、状态和标签等真实信息。视觉方向延续旧设计的海报卡片与日期分组，但字段以当前 Movie / Episode 能力为准。

用户可以从以下入口进入观影历史页面：

- 左侧导航栏点击“观影历史”
- 首页“继续观看”区域右侧箭头
- 观影洞察“观影日历”点击指定日期后跳转并定位

该页面是完整页面，不是弹窗。

## 6. 导航栏状态

观影历史页面支持：

- 展开导航栏状态
- 关闭导航栏状态

### 展开导航栏状态

展开状态下：

- 左侧显示完整导航栏
- 当前选中菜单为“观影历史”
- 主内容区位于导航栏右侧
- 影片区域按日期分组展示

### 关闭导航栏状态

关闭状态下：

- 左侧导航栏隐藏或收缩
- 顶部左侧显示导航栏展开按钮
- 页面标题显示为“观影历史”
- 副标题显示为“记录你的惬意时刻”
- 主内容区向左扩展

说明：

- 关闭导航栏状态不支持批量选择
- 不显示批量选择按钮
- 不显示批量操作栏

## 7. 日期筛选与历史区域

历史记录区域是页面主要内容区。旧设计的日期分组视觉保留，并补入当前真实日期筛选、日历定位与刷新能力。

### 日期筛选与刷新

最终筛选控件包含：

- `全部`
- `今天`
- `本周`
- `本月`
- `指定日期`
- `刷新`

规则：

- 选择预设范围后仅展示对应范围内的历史记录。
- 选择指定日期后展示当天的记录，并与来自观影日历的目标日期定位共用状态。
- 刷新为观影历史最终可见操作；刷新中需显示 loading，并禁用重复触发。
- 本页不因此新增关键词搜索、排序或批量操作栏。

内容按日期分组展示，每个分组包含：

- 日期标题
- 分割线
- Movie / Episode 历史卡片网格

### 日期标题

日期格式示例：

- `yyyy-mm-dd`

日期分组依据：

- 按关闭影片的时间分组

排序规则：

- 最新日期在上
- 较早日期在下

### 分割线

每个日期标题下方有分割线，用于区分日期分组。

要求：

- 分割线颜色低调
- 不要使用高饱和颜色
- 深色 / 浅色主题下均需清晰

## 8. 历史项卡片

观影历史项复用旧设计的媒体库海报卡片视觉方向，并以当前真实能力承载 Movie 与 Episode 两种记录。

每张卡片的共用字段包含：

- 来源标签
- 状态标签
- 海报 / 封面
- 标题
- 年份或剧集信息
- 最后观看时间
- 观看进度条
- 百分比
- 播放源状态，包括无播放源或播放源不可用
- 详情入口

结构示例：

```text
历史项卡片
├─ 左上：来源标签
├─ 右上：状态标签
├─ 海报 / 封面
├─ 标题
├─ 年份或剧集信息
├─ 最后观看时间 yyyy-mm-dd hh:mm
├─ 类别标签 | 情绪标签（数据存在时）
├─ 播放源可用性
├─ 观看进度条
└─ 百分比
```

### Movie / Episode 字段矩阵

| 类型 | 主标题 | 辅助信息 | 详情目标 | 标签规则 |
|---|---|---|---|---|
| Movie | 电影名 | 年份、来源、最后观看时间、进度 | Movie detail | 类别 / 情绪标签在数据存在时展示 |
| Episode | 集名或剧名 + 集号 | 剧名、季集号、最后观看时间、进度、来源状态 | Season / Episode 相关详情并定位 Episode（当前路由支持时） | 使用剧集适用标签，不强制与 Movie 一致 |

### 来源标签

截图中的 `来源标签` 是具体来源字段的占位。

实际显示应替换为具体来源值。

可能来源包括当前可安全展示的本地来源与 WebDAV 来源摘要。来源、文件名或路径遵循全局长字段与脱敏规则，不裸露完整 WebDAV URL。

具体显示内容按现有后端字段处理。

### 状态标签

右上角状态标签遵循媒体库页面中的状态标签规则。

Movie 记录的收藏 / 观看状态规则：

- 影片基础观看状态为：已看 / 未看
- 已看影片才能标记喜爱
- 未看影片才能标记想看
- 已看和未看影片都可以标记不想看
- 不想看 / 想看 / 喜爱 三个状态互斥，任意两个不能共存
- 不想看会覆盖想看和喜爱
- 不想看优先级最高

已看影片状态优先级：

1. 不想看
2. 喜爱
3. 已看

未看影片状态优先级：

1. 不想看
2. 想看
3. 未看

状态标签颜色遵循 `DESIGN.md`：

- 已看：蓝色
- 未看：淡灰边框，不填充
- 想看：浅粉色
- 喜爱：比想看更深的浅粉
- 不想看：低饱和橙红，不刺眼

Episode 记录显示其当前可用的已看 / 未看、播放进度与播放源状态；除非现有数据能力明确提供对应 Season / Episode 收藏状态，不强行套用 Movie 的喜爱 / 想看 / 不想看标签。

### 海报缺失

如果历史项没有海报或 still：

- 使用 `poster-placeholder.png`

### 播放源不可用

- 无播放源或来源不可用时，卡片显示可识别的不可播放状态。
- 不可播放状态不阻止用户进入详情承载页。
- 本页不将不可播放状态误写为记录已删除或媒体文件被删除。

## 9. 点击行为

观影历史中的历史项卡片可以点击。

点击行为：

- Movie 历史项进入电影详情页。
- Episode 历史项进入季 / 单集相关详情，并在当前路由支持时定位对应 Episode。
- 无播放源或播放源不可用项仍可进入详情。

要求：

- 不需要单独继续播放按钮
- 不需要删除按钮
- 不需要批量选择
- 点击卡片只进入对应详情页
- 不直接开始播放，除非现有后端 / 路由逻辑已有明确规定

## 11. 观影日历定位链路

从 `watch-insights-page.md` 的观影日历点击某天后：

- 跳转至观影历史页。
- 自动应用该日的指定日期筛选。
- 滚动定位至目标日期分组，并以短暂高亮反馈定位结果。
- 若当天无记录，显示指定日期的空状态或提示，不静默失败。

该交互由观影洞察页与观影历史页共同定义；仅补设计语义，不在本阶段改导航或筛选代码。

## 14. 深色 / 浅色主题注意事项

页面视觉遵循 `DESIGN.md`。

### 浅色主题

- 页面背景为淡粉白或统一浅色背景
- 影片卡片使用浅色或清晰表面
- 日期分割线使用低透明度中性色
- 状态标签使用低饱和语义色
- 滚动条使用统一样式

### 深色主题

- 页面背景使用深灰
- 影片卡片使用深灰 / 黑灰表面
- 日期分割线使用深色主题下可见的中性色
- 使用边框和透明度区分卡片层级
- 不使用强烈发光效果
- 不使用纯黑大面积背景，除非是海报占位或播放器区域

## 20. 验收标准

该页面完成后，需要满足：

### 基础验收

- 从左侧导航栏点击“观影历史”可以进入观影历史页面
- 首页“继续观看”右侧箭头可以跳转到观影历史页面
- 从观影日历点击日期可跳转到观影历史页面
- 当前导航项高亮为“观影历史”
- 页面显示标题“观影历史”
- 页面显示副标题“记录你的惬意时刻”
- 页面右上角显示主题切换按钮
- 页面不显示“扫描媒体库”按钮
- 页面支持浅色 / 深色主题
- 项目可以编译通过

### 分组验收

- 历史记录按日期分组
- 日期分组依据为关闭影片的时间
- 日期标题格式为 yyyy-mm-dd 或现有格式
- 最新日期分组显示在上方
- 每个日期分组下显示该日期的 Movie / Episode 历史项
- 日期分组之间有清晰分割线
- 支持全部、今天、本周、本月、指定日期筛选和刷新
- 来自日历的指定日期可滚动定位并短暂高亮；无记录时显示反馈

### 滚动验收

- 影片区域可滚动
- 顶部标题区不滚动
- 左侧导航栏不滚动
- 日期分组和影片卡片随影片区域一起滚动

### 历史项卡片验收

- 历史项显示来源标签、来源不可用状态和观看状态
- 历史项显示海报 / 封面、标题和最后观看时间
- Movie 与 Episode 显示各自适用的辅助字段和标签
- 历史项显示观看进度条和百分比
- 海报缺失时使用 poster-placeholder.png
- Movie 历史项进入电影详情
- Episode 历史项进入季 / 集详情并在支持时定位 Episode
- 无播放源项仍可进入详情

### 状态标签验收

- 已看影片无其他状态时显示“已看”
- 未看影片无其他状态时显示“未看”
- 已看影片标记喜爱时显示“喜爱”
- 未看影片标记想看时显示“想看”
- 已看或未看影片标记不想看时显示“不想看”
- 不想看优先级最高
- 不想看、想看、喜爱三个状态互斥，任意两个不能共存
- 不想看会覆盖想看和喜爱

### 不支持功能验收

- 不显示继续播放按钮
- 不显示删除单条历史按钮
- 不显示批量选择按钮
- 不显示批量操作栏
- 不显示关键词搜索、排序或批量操作区
