# 观影历史页面 UI 实施说明

## 1. 模块名称

观影历史

建议文件名：

- `watch-history-page.md`

建议实现对象：

- `WatchHistoryPage`
- 或 `WatchHistoryPage.xaml`

---

## 2. 页面用途

观影历史页面用于按日期展示用户的历史观影记录。

该页面展示用户曾经观看过的影片，并显示每部影片的观看进度、来源、状态和标签等信息。

用户可以从以下入口进入观影历史页面：

- 左侧导航栏点击“观影历史”
- 首页“继续观看”区域右侧箭头

该页面是完整页面，不是弹窗。

## 5. 顶部标题区

顶部标题区包含：

- 页面标题
- 页面副标题
- 主题切换按钮

### 页面标题

标题文本：

- `观影历史`

### 页面副标题

副标题文本：

- `记录你的惬意时刻`

说明：

- 展开导航栏状态下，截图中主要显示标题
- 关闭导航栏状态下，标题旁显示副标题

页面右上角显示主题切换按钮。

要求：

- 图标可根据当前主题显示日间 / 夜间图标
- 不应抢占页面主视觉

### 不显示扫描媒体库按钮

观影历史页面不显示“扫描媒体库”按钮。

要求：

- 删除截图中展开导航栏右上角的“扫描媒体库”按钮
- 观影历史页顶部只保留必要标题和主题切换按钮
- 不提供扫描入口

## 6. 导航栏状态

观影历史页面支持：

- 展开导航栏状态
- 关闭导航栏状态

### 展开导航栏状态

展开状态下：

- 左侧显示完整导航栏
- 当前选中菜单为“观影历史”
- 主内容区位于导航栏右侧
- 影片区域按日期分组展示

### 关闭导航栏状态

关闭状态下：

- 左侧导航栏隐藏或收缩
- 顶部左侧显示导航栏展开按钮
- 页面标题显示为“观影历史”
- 副标题显示为“记录你的惬意时刻”
- 主内容区向左扩展

说明：

- 关闭导航栏状态不支持批量选择
- 不显示批量选择按钮
- 不显示批量操作栏

---

## 7. 影片历史区域

影片历史区域是页面主要内容区。

内容按日期分组展示，每个分组包含：

- 日期标题
- 分割线
- 影片卡片网格

### 日期标题

日期格式示例：

- `yyyy-mm-dd`

日期分组依据：

- 按关闭影片的时间分组

排序规则：

- 最新日期在上
- 较早日期在下

### 分割线

每个日期标题下方有分割线，用于区分日期分组。

要求：

- 分割线颜色低调
- 不要使用高饱和颜色
- 深色 / 浅色主题下均需清晰

---

## 8. 影片卡片

观影历史中的影片卡片复用媒体库海报卡片风格。

每张卡片包含：

- 来源标签
- 状态标签
- 海报
- 电影名
- 日期
- 类别标签
- 情绪标签
- 观看进度条
- 百分比

结构示例：

```text
影片卡片
├─ 左上：来源标签
├─ 右上：状态标签
├─ 海报
├─ 电影名
├─ 日期 yyyy-mm-dd
├─ 类别标签 | 情绪标签
├─ 观看进度条
└─ 百分比
```

### 来源标签

截图中的 `来源标签` 是具体来源字段的占位。

实际显示应替换为具体来源值。

可能来源包括：

- 库内影片
- 库外影片
- WebDAV 来源
- 其他现有来源字段

具体显示内容按现有后端字段处理。

### 状态标签

右上角状态标签遵循媒体库页面中的状态标签规则。

状态规则：

- 影片基础观看状态为：已看 / 未看
- 已看影片才能标记喜爱
- 未看影片才能标记想看
- 已看和未看影片都可以标记不想看
- 不想看 / 想看 / 喜爱 三个状态互斥，任意两个不能共存
- 不想看会覆盖想看和喜爱
- 不想看优先级最高

已看影片状态优先级：

1. 不想看
2. 喜爱
3. 已看

未看影片状态优先级：

1. 不想看
2. 想看
3. 未看

状态标签颜色遵循 `DESIGN.md`：

- 已看：蓝色
- 未看：淡灰边框，不填充
- 想看：浅粉色
- 喜爱：比想看更深的浅粉
- 不想看：低饱和橙红，不刺眼

### 海报缺失

如果影片没有海报：

- 使用 `poster-placeholder.png`

## 9. 点击行为

观影历史中的影片卡片可以点击。

点击行为：

- 点击整张影片卡片进入影片详情页

要求：

- 不需要单独继续播放按钮
- 不需要删除按钮
- 不需要批量选择
- 点击卡片只进入详情页
- 不直接开始播放，除非现有后端 / 路由逻辑已有明确规定

---

## 10. 不支持的操作

观影历史页面不需要以下功能：

- 不需要继续播放按钮
- 不需要删除单条历史记录
- 不需要批量选择
- 不需要批量操作栏
- 不需要扫描媒体库按钮
- 不需要搜索筛选区

如果后续新增功能，需要单独补充规格，不由 Codex 自行添加。

## 11. 可滚动区域

观影历史页面的可滚动区域是：

- 影片区域

要求：

- 顶部标题区不滚动
- 左侧导航栏不滚动
- 影片历史区域内部滚动
- 日期分组和影片卡片随影片区域一起滚动
- 滚动条使用 WPF UI / 项目统一样式
- 不要给每个日期分组单独添加滚动

## 12. 空状态

如果没有观影历史记录，使用 EmptyState 组件。

建议文案：

- 标题：`暂无观影历史`
- 说明：`开始观看影片后，这里会记录你的观影足迹。`

要求：

- 空状态显示在影片历史区域
- 不使用独立插画图片
- 使用 WPF UI 图标 + 文案 + 可选操作按钮
- 不影响顶部标题区

## 13. 深色 / 浅色主题注意事项

页面视觉遵循 `DESIGN.md`。

### 浅色主题

- 页面背景为淡粉白或统一浅色背景
- 影片卡片使用浅色或清晰表面
- 日期分割线使用低透明度中性色
- 状态标签使用低饱和语义色
- 滚动条使用统一样式

### 深色主题

- 页面背景使用深灰
- 影片卡片使用深灰 / 黑灰表面
- 日期分割线使用深色主题下可见的中性色
- 使用边框和透明度区分卡片层级
- 不使用强烈发光效果
- 不使用纯黑大面积背景，除非是海报占位或播放器区域

## 19. 验收标准

该页面完成后，需要满足：

### 基础验收

- 从左侧导航栏点击“观影历史”可以进入观影历史页面
- 首页“继续观看”右侧箭头可以跳转到观影历史页面
- 当前导航项高亮为“观影历史”
- 页面显示标题“观影历史”
- 页面显示副标题“记录你的惬意时刻”
- 页面右上角显示主题切换按钮
- 页面不显示“扫描媒体库”按钮
- 页面支持浅色 / 深色主题
- 项目可以编译通过

### 分组验收

- 历史记录按日期分组
- 日期分组依据为关闭影片的时间
- 日期标题格式为 yyyy-mm-dd 或现有格式
- 最新日期分组显示在上方
- 每个日期分组下显示该日期的影片卡片
- 日期分组之间有清晰分割线

### 滚动验收

- 影片区域可滚动
- 顶部标题区不滚动
- 左侧导航栏不滚动
- 日期分组和影片卡片随影片区域一起滚动

### 影片卡片验收

- 影片卡片显示来源标签
- 影片卡片显示状态标签
- 影片卡片显示海报
- 影片卡片显示电影名
- 影片卡片显示日期
- 影片卡片显示类别标签和情绪标签
- 影片卡片显示观看进度条和百分比
- 海报缺失时使用 poster-placeholder.png
- 点击整张影片卡片进入影片详情页

### 状态标签验收

- 已看影片无其他状态时显示“已看”
- 未看影片无其他状态时显示“未看”
- 已看影片标记喜爱时显示“喜爱”
- 未看影片标记想看时显示“想看”
- 已看或未看影片标记不想看时显示“不想看”
- 不想看优先级最高
- 不想看、想看、喜爱三个状态互斥，任意两个不能共存
- 不想看会覆盖想看和喜爱

### 不支持功能验收

- 不显示继续播放按钮
- 不显示删除单条历史按钮
- 不显示批量选择按钮
- 不显示批量操作栏
- 不显示搜索筛选区
