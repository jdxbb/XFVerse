# XFVerse 1.0 安装说明

Last updated: 2026-06-21

## 1. 文档定位

本文面向需要安装、升级、修复或卸载 XFVerse 的 Windows 用户。

本文只说明软件部署和数据保留，不介绍媒体扫描、识别、播放、字幕、推荐等功能操作。功能使用方法见[软件使用说明书](使用说明书.md)；具体故障排查见[XFVerse 帮助中心](help/README.md)。

## 2. 适用版本与当前状态

- 产品：XFVerse。
- 版本：1.0.0。
- 安装范围：当前 Windows 用户。
- 发布形式：x64 与 ARM64 两个独立的 self-contained 安装包。
- 数字签名：当前安装包未签名。
- 安装生命周期：两个架构均已通过静默安装、同版本修复和卸载验证。
- ARM64 已通过原生首次启动、数据库初始化和 libmpv 基础媒体载入；x64 已在 Windows ARM64 仿真环境通过相同的进程级验证。
- 当前 Windows 11 ARM64 设备的安装和播放器人工验收已由用户确认通过；Windows 11 x64 原生和 Windows 10 x64 仍未通过最终 RC，不应把当前候选包视为 GA。

## 3. 系统要求

### 3.1 操作系统与架构

当前正式候选目标为：

- Windows 10 / Windows 11 x64。
- Windows ARM64。

不支持 Windows X86（32 位）。

当前仅有 Windows 11 ARM64 原生 RC 证据。Windows 11 x64 原生和 Windows 10 x64 仍是发布 Blocker；最终最低 Windows 版本和完整支持范围以这些 Blocker 的关闭结果为准。

### 3.2 运行时

正式安装包已经包含对应架构的 .NET 8、WPF、libmpv 和 ffprobe 运行组件。

普通用户不需要安装：

- Visual Studio。
- .NET SDK。
- 单独的 .NET Desktop Runtime。
- 单独的 mpv 或 ffmpeg。

### 3.3 磁盘与网络

- 建议安装前至少保留 1 GB 可用空间。
- 安装后程序文件约占 520 MB，后续数据库、海报、字幕和视频缓存会继续占用空间。
- 使用本地媒体不要求 WebDAV 或外部 API。
- WebDAV、影片发现、在线识别、在线字幕和 AI 功能分别需要网络及相应服务配置。

## 4. 选择正确的安装包

在 Windows 中打开“设置”→“系统”→“系统信息”或“关于”，查看“系统类型”。

| 系统类型 | 应下载的文件 |
| --- | --- |
| 基于 x64 的处理器 | `XFVerse-Setup-1.0.0-win-x64.exe` |
| 基于 ARM64 的处理器 | `XFVerse-Setup-1.0.0-win-arm64.exe` |
| 32 位操作系统或 X86 处理器 | 不支持 |

不要把两个架构的安装包同时安装为两套程序。两者使用相同的正式产品身份、安装目录和用户数据格式。

## 5. 正式安装包与校验值

| 架构 | 文件大小 | SHA-256 |
| --- | --- | --- |
| x64 | 266,950,555 bytes / 254.58 MiB | `6D7641FBEB7E20FC282EE23BF81DF7ECA1CE81DFE6D4366ED1DE38D167F04A15` |
| ARM64 | 240,257,892 bytes / 229.13 MiB | `6C75397ADAD4CEDF6374CA936D367D989C817ABB0ADAF9A0A31ECD6DADA52BC8` |

产物和构建记录见[XFVerse 1.0 正式打包记录](release-v1.0/XFVERSE_1_0_PACKAGING_RECORD.md)。

## 6. 校验安装包

建议在运行安装包前检查 SHA-256，确认文件完整且没有下载错误。

### 6.1 使用 PowerShell

1. 在安装包所在文件夹空白处按住 Shift 并单击鼠标右键。
2. 选择“在终端中打开”或打开 PowerShell 后进入该目录。
3. 根据下载的架构执行：

```powershell
Get-FileHash .\XFVerse-Setup-1.0.0-win-x64.exe -Algorithm SHA256
```

或：

```powershell
Get-FileHash .\XFVerse-Setup-1.0.0-win-arm64.exe -Algorithm SHA256
```

4. 将输出的 `Hash` 与第 5 节对应值逐字比较。

如果校验值不一致：

- 不要继续运行安装包。
- 删除该文件并从可信发布位置重新下载。
- 不要通过关闭安全软件来绕过校验失败。

## 7. 安装前准备

1. 确认安装包架构与 Windows 系统类型一致。
2. 完成 SHA-256 校验。
3. 如果 XFVerse 正在运行，先保存当前操作并退出应用。
4. 升级已有安装时，建议先备份用户数据目录，尤其是数据库文件。
5. 不需要卸载旧版本后再安装；正常升级应直接覆盖安装。

## 8. 安装步骤

1. 双击正确架构的安装包。
2. 首次安装时，在“选择安装位置”页面确认目录，或单击“浏览”选择其他当前用户可写目录。
3. 覆盖安装时，安装器会优先沿用原安装目录，安装位置页可能不再显示。
4. 在“选择附加任务”页面决定是否创建桌面快捷方式。该选项默认不勾选。
5. 在“准备安装”页面检查安装位置和附加任务。
6. 单击“安装”。
7. 等待文件解压和快捷方式创建完成。
8. 在完成页面按需选择“启动 XFVerse”，然后单击“完成”。

默认程序目录为：

```text
%LOCALAPPDATA%\Programs\XFVerse
```

安装器只要求当前用户权限，不需要管理员权限。

## 9. Windows 安全提示

当前 XFVerse 1.0 候选安装包未进行 Authenticode 数字签名，因此 Windows 可能显示“未知发布者”或 Microsoft Defender SmartScreen 提示。

遇到提示时：

1. 先确认文件名、来源和 SHA-256。
2. 只有在校验值完全一致且文件来自可信发布位置时才继续。
3. 不要关闭 Defender、防火墙或其他安全软件。
4. 如果文件来源不明或校验失败，应取消安装。

## 10. 首次启动

根据当前代码，首次启动会：

1. 创建当前用户的 XFVerse 数据目录。
2. 创建或打开 SQLite 数据库。
3. 应用当前版本包含的数据库 migrations。
4. 初始化主题和应用服务。
5. 根据已有设置决定是否安排启动时 WebDAV 扫描。

如果初始化失败，应用会显示“启动失败”及错误摘要，然后退出，不应继续进入不可预测状态。

首次启动不会自动要求配置所有在线服务。TMDB、OMDb、OpenSubtitles、AI 和 WebDAV 均按对应功能独立配置，不是浏览本地媒体库的共同前置条件。

## 11. 第一次配置的最短入口

启动后可通过用户菜单进入：

- “扫描任务”：添加本地目录或 WebDAV 扫描路径并开始扫描。
- “设置”：按需配置主题、关闭行为、TMDB、OMDb、OpenSubtitles、AI 和缓存。

安装说明不展开这些功能的完整操作步骤，详见[软件使用说明书](使用说明书.md)。

## 12. 程序文件与用户数据

### 12.1 程序文件

默认位置：

```text
%LOCALAPPDATA%\Programs\XFVerse
```

该目录由安装器管理，包含应用程序集、目标架构原生组件、资源和许可证。

### 12.2 用户数据

默认位置：

```text
%LOCALAPPDATA%\MediaLibrary
```

主要内容包括：

| 内容 | 默认位置或形式 |
| --- | --- |
| 软件数据库 | `media-library.db` |
| 视频缓存 | `VideoCache` |
| 海报缓存 | `PosterCache` |
| 在线字幕缓存 | `OnlineSubtitles` |
| 诊断日志 | `Logs` |
| 本地用户资料 | 用户资料 JSON 和头像目录 |
| 界面与播放器偏好 | JSON 配置及数据库设置 |

目录名 `MediaLibrary` 是为兼容既有数据保留的内部名称。

用户数据目录和程序目录相互独立。覆盖安装、修复安装、架构切换及默认卸载都不应删除用户数据。

## 13. 升级到新版本

1. 退出正在运行的 XFVerse。
2. 建议备份 `%LOCALAPPDATA%\MediaLibrary`。
3. 下载与当前 Windows 架构匹配的新安装包并校验 SHA-256。
4. 直接运行新安装包。
5. 保持原安装目录完成覆盖安装。
6. 启动新版本后，应用会按代码中的数据库初始化流程应用必要 migrations。

不要为了升级而先删除用户数据目录，也不要把删除数据库作为常规升级步骤。

XFVerse 1.0 当前没有自动更新器，升级需要手动下载安装包。

## 14. 在 x64 与 ARM64 包之间切换

在 Windows ARM64 设备上，如需从 x64 包切换为原生 ARM64 包，或反向切换：

1. 退出 XFVerse。
2. 备份用户数据目录。
3. 运行目标架构安装包并使用原安装目录覆盖安装。
4. 安装器会删除另一架构的 mpv 和 ffprobe 程序目录。
5. 用户数据库、设置、缓存和资料保持在原用户数据目录中。

Phase 8.4 已验证 x64→ARM64 和 ARM64→x64 的程序文件切换及用户数据目录不变。

不要手工混合复制两个 staging 的 DLL 或 EXE。

## 15. 修复安装

遇到程序文件缺失或被安全软件隔离后恢复的情况，可进行同版本修复：

1. 退出 XFVerse。
2. 重新取得同版本、同架构安装包。
3. 校验 SHA-256。
4. 运行安装包并选择原安装目录。
5. 完成覆盖安装。

修复安装只重写或补齐程序文件，不重置数据库和配置。

## 16. 卸载

可通过 Windows“设置”→“应用”→“已安装的应用”找到 XFVerse 并卸载。

默认卸载会：

- 删除程序文件。
- 删除开始菜单快捷方式。
- 删除由安装器创建的桌面快捷方式。
- 保留 `%LOCALAPPDATA%\MediaLibrary` 用户数据。

默认卸载不会：

- 删除本地媒体文件。
- 删除 WebDAV 服务器上的文件。
- 自动删除数据库、设置、播放历史或缓存。

## 17. 完全清除 XFVerse 软件数据

只有在确认不再需要数据库、设置、播放历史、用户状态、资料和缓存时才执行。

1. 退出 XFVerse。
2. 先复制 `%LOCALAPPDATA%\MediaLibrary` 到安全位置作为备份。
3. 通过 Windows 正常卸载 XFVerse。
4. 确认备份可用后，手工删除 `%LOCALAPPDATA%\MediaLibrary`。

删除该目录会清除 XFVerse 的软件数据，但不会删除：

- 用户自行保存的本地影片、电视剧或字幕源文件。
- WebDAV 服务器上的媒体文件。

如果无法判断某个目录是否为真实媒体目录，不要删除，先查看[缓存、备份与恢复问题](help/缓存备份与恢复.md)或寻求人工确认。

## 18. 安装或启动失败

### 18.1 安装包无法运行

- 核对系统架构和安装包架构。
- 重新检查 SHA-256。
- 确认文件来自可信发布位置。
- 不要关闭系统安全功能。

### 18.2 安装过程中提示文件被占用

- 退出 XFVerse。
- 在任务管理器中确认 XFVerse 已结束。
- 再次运行安装包。

### 18.3 首次启动提示“应用初始化失败”

- 记录错误摘要。
- 确认当前用户可以写入 `%LOCALAPPDATA%\MediaLibrary`。
- 不要直接删除数据库。
- 如已有数据，先备份整个用户数据目录。
- 查看 `%LOCALAPPDATA%\MediaLibrary\Logs` 中与问题时间对应的脱敏日志。

### 18.4 播放、扫描或在线服务失败

这类问题已经超出安装说明范围。请保留错误信息和脱敏日志，按[XFVerse 帮助中心](help/README.md)中的对应主题处理。

## 19. 隐私与问题反馈

提交问题时不要公开：

- API Key、Token 或密码。
- 完整 WebDAV URL、账号或远端目录。
- 完整本地用户路径。
- 用户数据库。
- 私有媒体清单或未经处理的日志。

可以提供：

- XFVerse 版本。
- Windows 版本与系统架构。
- 安装包文件名和 SHA-256。
- 经过脱敏的错误摘要。
- 与问题时间对应的少量脱敏日志。
