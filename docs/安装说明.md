# 安装说明

## 环境要求

- Windows 10/11
- .NET 8 Desktop Runtime 或 .NET 8 SDK
- 可访问 WebDAV 服务的网络环境

## 启动方式

1. 使用 Visual Studio 打开 `MediaLibrary.sln`。
2. 将 `MediaLibrary.App` 设置为启动项目。
3. 首次启动时程序会自动创建或迁移本地 SQLite 数据库。
4. 在“设置”页配置 WebDAV、TMDB、OMDb 和 AI 接口。

## 注意事项

- API Key、Token 和 WebDAV 密码只应通过设置页保存，不要写入源码。
- 如果播放器无法播放远程 WebDAV 文件，请确认服务器支持 HTTP Range 请求和基础认证。
- 如数据库结构更新，启动应用时会自动执行 EF Core Migration。

## 测试安装包

当前仓库提供测试安装包脚本：

```powershell
.\scripts\packaging\Build-TestInstaller.ps1
```

如果只需要生成当前 Windows PC 使用的 x64 测试包，并保留当前数据库里的画像缓存数据，可使用：

```powershell
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\scripts\packaging\Build-TestInstaller.ps1 -RuntimeIdentifiers win-x64 -PreserveProfileCache
```

脚本会执行以下操作：

- 以仓库根目录 `logo.svg` 为唯一外部品牌图标源，生成软件 EXE、任务栏、托盘、快捷方式和安装包使用的多尺寸 `app-logo.ico`。
- 生成 Windows x64 和 Windows ARM64 自包含发布目录。
- 内置当前需要的 mpv 播放器内核与 ffprobe 组件。
- 复制当前用户数据快照，并排除 `VideoCache`、旧数据库备份和日志。
- 通过数据库副本注入 2026 年 4 月、5 月的史诗世界观偏好测试观影数据。
- 生成当前用户安装的 Inno Setup 安装包，并输出到仓库根目录。

测试安装包会覆盖当前用户的 XFVerse 本地数据目录，且会包含测试快照中的 API Key、Token、WebDAV 账号、播放记录和画像数据。该包只用于内部验证，不应作为正式公开发布包分发。

Known Issues：

- Blocker：无。
- Deferred：修改 `logo.svg` 后需要重新运行图标生成步骤并重新构建应用；Windows 可能需要刷新图标缓存后才显示新图标。
- Deferred：正式发布包仍需拆分真实用户数据与示例数据，避免携带私有凭据。
- Noise：首次生成安装包时可能需要下载 Inno Setup 编译器到仓库临时目录。
