# 安装与启动问题

Last updated: 2026-06-21

## 适用范围

本文处理安装器无法运行、Windows 安全提示、架构不匹配、首次启动失败、数据库初始化失败和升级后无法启动。

正常安装、升级和卸载步骤见[安装说明](../安装说明.md)。

## 1. 安装器无法打开或立即退出

### 现象

- 双击安装包没有反应。
- Windows 提示文件损坏或无法运行。
- 安装器刚出现便退出。

### 常见原因

- 安装包下载不完整。
- x64 与 ARM64 安装包选错。
- 文件被浏览器、安全软件或系统标记为不可信。
- 安装包不是正式发布文件。

### 安全检查

1. 核对正式文件名是否包含正确架构：
   - `XFVerse-Setup-1.0.0-win-x64.exe`
   - `XFVerse-Setup-1.0.0-win-arm64.exe`
2. 不要运行名称异常、被二次打包或来源不明的副本。
3. 在安装包所在目录打开 PowerShell，执行：

```powershell
Get-FileHash -LiteralPath ".\XFVerse-Setup-1.0.0-win-x64.exe" -Algorithm SHA256
```

ARM64 包应把文件名替换为 `XFVerse-Setup-1.0.0-win-arm64.exe`。

4. 将输出与正式发布页或[安装说明](../安装说明.md)中的 SHA-256 完整比对。

### 修复步骤

1. SHA-256 不一致时，删除当前安装包并从可信发布位置重新下载。
2. 确认架构正确后重新运行。
3. 如果文件被 Windows 阻止，先核对来源和哈希，再查看文件属性中的安全提示。
4. 仍无法运行时，记录 Windows 提示文字和安装包哈希。

### 不要做什么

- 不要为了运行来源不明的安装包而关闭系统安全功能。
- 不要忽略哈希不一致。
- 不要把 x64 与 ARM64 程序文件手工混合复制。

### 仍未解决时提供

- 安装包文件名、大小和 SHA-256。
- Windows 版本和系统架构。
- Windows 显示的脱敏错误文字。

## 2. SmartScreen 或“未知发布者”提示

### 现象

Windows 显示“Windows 已保护你的电脑”或发布者未知。

### 原因

XFVerse 1.0 当前正式候选安装包未配置 Authenticode 数字签名，因此 Windows 可能显示未知发布者。这不等于文件一定安全，也不等于文件一定有问题。

### 安全检查

1. 确认下载来源可信。
2. 核对正式文件名。
3. 核对完整 SHA-256。
4. 确认架构与当前 Windows 一致。

### 处理方式

只有上述检查全部通过时，才按 Windows 提供的界面继续。不同 Windows 版本的按钮文字可能不同。

### 不要做什么

- 不要长期关闭 SmartScreen。
- 不要为安装 XFVerse 关闭防火墙或杀毒软件。
- 不要运行哈希不一致的文件。

## 3. 如何确认 Windows 架构

### 图形界面

打开“设置 → 系统 → 系统信息”，查看“系统类型”。

### PowerShell

```powershell
[System.Runtime.InteropServices.RuntimeInformation]::OSArchitecture
```

常见输出：

- `X64`：使用 win-x64 安装包。
- `Arm64`：使用 win-arm64 安装包。

XFVerse 1.0 不提供 X86 安装包。不要根据处理器品牌猜测架构。

## 4. 安装时提示文件被占用

### 现象

- 安装或修复时无法替换程序文件。
- Windows 提示 XFVerse 或相关文件正在使用。

### 常见原因

- 主窗口已关闭，但程序仍缩小在系统托盘。
- 播放器或 XFVerse 进程尚未退出。
- 安全软件正在扫描新程序文件。

### 修复步骤

1. 保存当前操作。
2. 从 XFVerse 托盘菜单明确退出。
3. 打开任务管理器，确认 XFVerse 进程已结束。
4. 关闭仍打开的播放器窗口。
5. 等待短暂的文件扫描完成后重试安装。

不要直接删除正在使用的程序目录。

## 5. 安装成功但找不到程序

### 检查步骤

1. 在 Windows 开始菜单搜索“XFVerse”。
2. 如果安装时选择创建桌面快捷方式，检查桌面。
3. 默认程序目录为：

```text
%LOCALAPPDATA%\Programs\XFVerse
```

4. 确认安装器显示完成，而不是中途取消。

### 修复步骤

- 程序文件存在但快捷方式丢失时，可执行同版本修复安装。
- 程序目录不完整时，重新运行与当前架构匹配的安装包。
- 不要从其他电脑复制部分 DLL 覆盖。

## 6. 首次启动显示“应用初始化失败”

### 影响范围

应用可能在进入首页前退出。真实本地和 WebDAV 文件不会因此被删除。

### 常见原因

- 用户数据目录不可写。
- 数据库初始化或 migration 失败。
- 用户数据目录中的文件被占用。
- 程序文件不完整或架构不匹配。
- 旧数据与当前候选版本存在尚未验证的兼容问题。

### 安全检查

1. 确认 XFVerse 已完全退出。
2. 检查当前用户能否访问：

```text
%LOCALAPPDATA%\MediaLibrary
```

3. 不要删除 `media-library.db`。
4. 如果已有数据，先复制整个用户数据目录作为备份。
5. 查看以下目录中与失败时间对应的日志：

```text
%LOCALAPPDATA%\MediaLibrary\Logs
```

6. 确认程序目录和安装包架构一致。

### 修复步骤

1. 先运行同版本修复安装，补齐程序文件。
2. 保持用户数据目录不变，再次启动。
3. 如果错误明确指向数据库，停止重复启动并保留数据库副本。
4. 按[缓存备份与恢复](缓存备份与恢复.md)中的数据库恢复流程处理。

### 不要做什么

- 不要把删除数据库作为第一步。
- 不要用空数据库覆盖唯一数据副本。
- 不要在 XFVerse 运行时复制或替换数据库。

### 仍未解决时提供

- XFVerse 版本、安装包架构和 Windows 架构。
- 错误摘要。
- 失败时间附近的脱敏日志。
- 是否为首次安装、覆盖升级或跨架构切换。

## 7. 数据库 migration 或升级后启动失败

### 现象

- 旧版本可以启动，新版本启动失败。
- 错误摘要提到数据库、migration、SQLite 或 schema。

### 处理步骤

1. 停止继续反复启动。
2. 退出所有 XFVerse 进程。
3. 备份整个 `%LOCALAPPDATA%\MediaLibrary`。
4. 记录升级前后的 XFVerse 版本。
5. 保留失败后的数据库和日志，不要修改。
6. 尝试同版本修复安装；修复安装只补程序文件，不应重置用户数据。
7. 修复后仍失败时，按备份恢复流程回到已知可用数据副本。

数据库迁移失败属于需要保留证据的问题。不要手工编辑 SQLite 表结构。

## 8. 覆盖安装或跨架构切换后无法启动

### 常见原因

- 使用了与 Windows 不匹配的目标架构。
- 安装过程中 XFVerse 未完全退出。
- 程序目录中残留了手工复制的其他架构原生文件。
- 用户数据升级失败。

### 修复步骤

1. 确认 Windows 架构。
2. 退出 XFVerse。
3. 运行正确架构的同版本安装包进行修复。
4. 不手工混合 `win-x64` 和 `win-arm64` 原生目录。
5. 用户数据已备份时，可先正常卸载程序，再安装正确架构。
6. 默认卸载保留用户数据；不要勾连到真实媒体目录进行清理。

## 9. 完全重装与完全清除的区别

### 完全重装程序

适用于程序文件损坏：

1. 备份用户数据。
2. 正常卸载 XFVerse。
3. 保留 `%LOCALAPPDATA%\MediaLibrary`。
4. 安装正确架构的软件。

媒体库、设置和历史应继续保留。

### 完全清除软件数据

只有确定不再需要媒体记录、设置、状态和历史时才执行。必须先备份，然后在 XFVerse 已卸载且不再运行时手工删除用户数据目录。

完全清除 XFVerse 软件数据仍不会删除存放在其他位置的本地媒体或 WebDAV 文件。
