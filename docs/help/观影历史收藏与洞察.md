# 观影历史、收藏与洞察问题

Last updated: 2026-06-21

## 适用范围

本文处理观影历史为空或无法打开、收藏状态不一致、电影与电视剧状态规则、观影统计和 AI 画像问题。

正常功能说明见[软件使用说明书第 10、11 章](../使用说明书.md)。

## 1. 播放后观影历史为空

### 常见原因

- 只打开了详情，没有实际开始播放。
- 播放器在进度写入前异常退出。
- 当前日期筛选不包含该记录。
- 播放来源未成功进入可播放状态。
- 对应软件记录已被删除。

### 检查步骤

1. 将历史筛选切换到“全部”。
2. 刷新观影历史。
3. 正常播放一段时间。
4. 正常关闭播放器。
5. 再次刷新历史。

强制结束进程可能导致最后一段进度尚未保存。

## 2. 历史记录无法打开

### 常见提示

- 播放源不可用。
- 剧集历史缺少所属季信息。
- 关联媒体记录已被删除。

### 处理步骤

1. 确认真实来源仍存在。
2. 本地或 WebDAV 来源恢复后重新扫描。
3. 从媒体库重新打开对应电影或剧集。
4. 记录历史条目的类型和错误摘要。

历史页面不会为了打开旧记录而创建虚假播放源。

## 3. 历史日期筛选结果不符合预期

历史支持全部、今天、本周、本月和指定日期。

检查：

- Windows 日期、时间和时区是否正确。
- 当前选择的是哪种日期范围。
- 指定日期是否选中正确日期。
- 播放发生时间是否跨越午夜。

观影洞察日历跳转到历史时，也会应用对应日期范围。

## 4. 收藏夹中找不到内容

### 检查步骤

1. 确认当前页签是“喜爱”还是“想看”。
2. 刷新收藏夹。
3. 在详情页确认当前状态。
4. 检查电影是否因状态互斥被自动调整。
5. 电视剧应检查 Season 状态，而不是只看 Series。

收藏夹可以展示电影和电视剧季，不把 Episode 当作电影收藏项。

## 5. 从收藏夹移除后内容是否删除

不会。移除只取消当前状态：

- “喜爱”页移除会取消喜爱。
- “想看”页移除会取消想看。
- 不删除媒体记录。
- 不删除播放源。
- 不删除真实文件。
- 不自动清除其他不冲突状态。

## 6. 电影状态自动发生变化

电影状态规则：

| 操作 | 自动结果 |
| --- | --- |
| 标记已看 | 清除想看 |
| 标记未看 | 清除喜爱 |
| 设置喜爱 | 电影必须已看 |
| 设置想看 | 电影必须未看 |
| 设置不想看 | 与喜爱、想看互斥 |

这属于产品规则，不是数据丢失。

## 7. 电视剧状态与电影不同

电视剧按层级保存状态：

- Season：喜爱、想看、不想看和整季已看。
- Episode：单集已看。
- Series：组织电视剧资料和季列表。

因此可能出现：

- 某集已看，但整季未完成。
- 季出现在收藏夹，Series 本身没有相同按钮。
- Episode 不提供电影级喜爱或想看。

## 8. 标记整季已看后结果异常

### 检查步骤

1. 确认打开的是正确季。
2. 核对当前季的集列表。
3. 检查是否存在 metadata-only Episode。
4. 刷新季详情。
5. 查看失败状态消息。

整季操作会影响该季中的集观看状态。识别或季集归属错误时，应先修正结构。

## 9. 观影统计为空

### 常见原因

- 没有已保存的电影观看记录。
- 当前选择的月份没有电影历史。
- 只有电视剧观看数据。
- 播放没有成功写入历史。

### 检查步骤

1. 切换“本月”和“全部”。
2. 检查观影历史中是否有电影记录。
3. 通过播放器正常观看电影并关闭。
4. 刷新统计。

XFVerse 1.0 的观影统计为 Movie-only。电视剧不计入统计。

## 10. 统计数据与媒体库数量不同

媒体库数量表示软件中的媒体记录；观影统计依据实际保存的电影播放历史。两者不是同一指标。

以下内容不会自动成为观影统计：

- 仅扫描入库的电影。
- 仅标记想看的电影。
- 电视剧或 Episode 历史。
- 没有实际播放历史的元数据条目。

## 11. 观影统计加载失败

### 处理步骤

1. 保留页面显示的错误摘要。
2. 切换范围后重试一次。
3. 检查用户数据目录是否可写。
4. 不要删除数据库。
5. 查看 `watch-insights-perf.log` 中问题时间附近内容。

如果页面保留上一次统计结果，应以界面显示的刷新状态为准。

## 12. 画像显示“未配置”

AI 服务或画像路由没有可用模型。

处理：

1. 打开“设置 → API 配置”。
2. 保存并测试 AI 服务。
3. 选择可用默认模型。
4. 确认观影画像路由使用可用模型。
5. 返回观影洞察刷新。

## 13. 画像显示“数据不足”

画像需要足够的电影观看输入。应正常积累电影历史，而不是手工编辑数据库。

电视剧、Season 和 Episode 不进入 XFVerse 1.0 画像输入。

## 14. 画像刷新失败但仍显示旧内容

这是缓存回退：

- 新请求失败。
- 上一次可用画像仍被保留。
- 页面应提示刷新失败或使用缓存。

旧画像不代表本次刷新成功。恢复 AI 服务后再次刷新。

## 15. AI 画像内容不准确

### 处理方式

1. 检查电影识别是否正确。
2. 检查观看历史和状态。
3. 修正错误影片记录。
4. 积累更多电影观看数据。
5. 再刷新画像。

画像是辅助性总结，不应作为身份、健康或专业判断。

## 16. 本地个人资料无法保存

### 检查步骤

1. 确认签名没有超过界面限制且不包含换行。
2. 头像使用可读取的 PNG、JPG、BMP、GIF 或 TIFF。
3. 确认用户数据目录可写。
4. 换用较小的本地图片重试。

个人资料只保存在本机，不提供云同步。

## 17. “退出登录”没有清除账号

XFVerse 1.0 没有云账号和真实登录会话。“退出登录”是保留入口，不代表：

- 注销在线账户。
- 清除本地资料。
- 删除外部服务凭据。
- 跨设备退出。

## 仍未解决时提供

- XFVerse 版本。
- 问题属于历史、收藏、统计还是画像。
- 当前日期范围或页签。
- 条目类型：电影、Season 或 Episode。
- 脱敏错误摘要。
- 是否配置 AI。
- 问题时间附近的洞察日志片段。
