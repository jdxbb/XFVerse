# 缓存、备份与恢复问题

Last updated: 2026-06-21

## 适用范围

本文处理用户数据目录、缓存占用、缓存清理失败、数据库备份、恢复、卸载后数据和完全清除。

正常数据管理说明见[软件使用说明书第 13 章](../使用说明书.md)和[安装说明](../安装说明.md)。

## 1. XFVerse 用户数据在哪里

默认目录：

```text
%LOCALAPPDATA%\MediaLibrary
```

主要内容：

| 内容 | 名称 |
| --- | --- |
| SQLite 数据库 | `media-library.db` |
| 视频缓存 | `VideoCache` |
| 海报缓存 | `PosterCache` |
| 在线字幕缓存 | `OnlineSubtitles` |
| 诊断日志 | `Logs` |
| 本地资料和设置 | JSON、头像及数据库设置 |

内部目录名 `MediaLibrary` 是兼容历史保留名称，不代表产品名发生变化。

## 2. 程序目录与用户数据目录有什么区别

- 程序目录：应用 EXE、DLL、mpv、ffprobe 和资源。
- 用户数据目录：数据库、设置、状态、历史、资料和缓存。

默认程序目录：

```text
%LOCALAPPDATA%\Programs\XFVerse
```

覆盖安装、修复安装、架构切换和默认卸载都不应删除用户数据目录。

## 3. 如何安全备份

### 推荐步骤

1. 正常关闭播放器。
2. 从 XFVerse 托盘菜单退出应用。
3. 在任务管理器确认 XFVerse 已结束。
4. 复制整个 `%LOCALAPPDATA%\MediaLibrary` 目录到安全位置。
5. 在备份名称或说明中记录日期和 XFVerse 版本。
6. 确认备份中至少存在 `media-library.db`。

### 为什么要备份整个目录

数据库、设置、本地资料、头像和部分缓存之间可能存在关联。只复制数据库可能无法完整恢复界面偏好和本地资料。

### 不要做什么

- 不要在 XFVerse 正在运行时替换数据库。
- 不要只保留唯一数据库副本。
- 不要把备份保存在即将清理的用户数据目录内部。

## 4. 什么时候应备份

- 覆盖升级前。
- x64 与 ARM64 架构切换前。
- 重装 Windows 前。
- 手工恢复数据前。
- 完全清除软件数据前。
- 数据库出现异常但仍能复制时。

正常清缓存通常不要求备份数据库，但重要数据仍建议定期备份。

## 5. 如何恢复完整备份

### 安全步骤

1. 安装与备份兼容的 XFVerse 版本。
2. 启动一次后正常退出，确认程序可以运行。
3. 确认 XFVerse 和播放器进程已结束。
4. 先备份当前 `%LOCALAPPDATA%\MediaLibrary`。
5. 将完整备份恢复到用户数据目录。
6. 启动 XFVerse。
7. 检查设置、媒体库、来源、收藏和历史。
8. 重新测试 WebDAV 和外部服务凭据。

### 凭据限制

凭据使用当前 Windows 用户上下文的本机保护机制。备份恢复到其他电脑、Windows 用户或重装系统后，可能需要重新输入凭据。

## 6. 数据库疑似损坏或 migration 失败

### 现象

- 启动提示数据库或 SQLite 错误。
- 升级后提示 migration 失败。
- 媒体库无法加载且日志指向数据库。

### 安全处理

1. 停止反复启动。
2. 退出所有 XFVerse 进程。
3. 复制整个用户数据目录作为故障副本。
4. 保留错误发生时间和日志。
5. 不修改原数据库。
6. 如果有已知可用备份，按完整恢复流程恢复。

### 不要做什么

- 不要直接删除 `media-library.db`。
- 不要用空数据库覆盖故障副本。
- 不要手工编辑 migration 表。
- 不要从不同时间的备份随意拼接数据库和配置。

数据库结构问题应保留证据并等待可验证的恢复方案。

## 7. 海报缓存占用过大

设置页提供海报缓存状态、容量上限和清理。

### 处理步骤

1. 打开“设置 → 通用”。
2. 刷新软件缓存状态。
3. 调整海报缓存容量上限。
4. 保存设置。
5. 按需执行清理海报缓存。

清理后海报会在后续页面访问时重新获取或生成。该操作不删除电影、电视剧、状态、历史、视频缓存或真实媒体。

## 8. “其他缓存”是什么

“其他缓存”表示 XFVerse 标记为可清理的其他缓存记录，例如部分外部 metadata 缓存。

清理其他缓存：

- 不删除媒体记录。
- 不删除收藏或历史。
- 不删除本地或 WebDAV 文件。
- 可能导致后续页面重新请求外部数据。

## 9. 在线字幕缓存如何清理

设置页的在线字幕清理只针对孤立缓存：

- 没有被 Movie 绑定引用。
- 没有被 Episode 绑定引用。
- 没有被当前播放源绑定引用。

仍在使用的绑定字幕应被保留。扫描发现的本地外挂字幕不属于在线字幕孤立缓存。

## 10. 缓存清理失败

### 常见原因

- 文件正被播放器使用。
- XFVerse 无法读取字幕绑定引用。
- 部分缓存文件没有删除权限。
- 安全软件正在扫描文件。

### 处理步骤

1. 关闭播放器。
2. 退出 XFVerse 后重新打开。
3. 刷新缓存状态。
4. 再执行一次对应类别的清理。
5. 记录页面显示的错误摘要。

如果界面提示无法读取在线字幕绑定引用，软件会停止清理以避免误删仍在使用的字幕，这是安全保护。

不要直接对整个用户数据目录执行递归删除来代替分类清理。

## 11. 视频缓存占用空间

视频缓存默认位于：

```text
%LOCALAPPDATA%\MediaLibrary\VideoCache
```

视频缓存用于辅助部分 WebDAV 播放。它不是原始远端文件，也不等于媒体库记录。

处理原则：

- 播放中不要手工删除正在使用的缓存。
- 先退出播放器和 XFVerse。
- 优先使用软件提供的缓存管理能力。
- 不要把真实媒体目录误认成 `VideoCache`。

清理视频缓存不会删除 WebDAV 服务器上的文件，但可能使后续远端播放重新下载或缓冲。

## 12. 日志占用空间

日志位于：

```text
%LOCALAPPDATA%\MediaLibrary\Logs
```

Phase 8.7 只记录日志位置和安全提交方法。XFVerse 1.0 当前没有面向用户的统一日志保留周期设置。

手工处理前：

1. 退出 XFVerse。
2. 保留最近问题对应日志。
3. 不删除仍需用于故障分析的唯一证据。
4. 不把完整日志公开上传。

## 13. 卸载后数据还在

这是默认行为。卸载主要删除：

- 程序文件。
- 安装器创建的快捷方式。

默认保留：

- 数据库。
- 设置。
- 用户资料。
- 播放历史和状态。
- 缓存和日志。

重新安装相同或兼容版本后，应用应继续使用这些数据。

## 14. 卸载后如何恢复使用

1. 安装与 Windows 架构匹配的 XFVerse。
2. 使用原 Windows 用户启动。
3. 确认 `%LOCALAPPDATA%\MediaLibrary` 仍存在。
4. 启动后检查媒体库和设置。
5. 外部服务凭据失败时重新输入并测试。

如果用户数据已手工删除，只能从已有备份恢复。

## 15. 如何完全清除 XFVerse 软件数据

只有确认不再需要数据库、设置、状态、历史、资料和缓存时执行：

1. 退出 XFVerse。
2. 备份整个用户数据目录。
3. 正常卸载 XFVerse。
4. 确认备份可用。
5. 手工删除 `%LOCALAPPDATA%\MediaLibrary`。

该操作会清除 XFVerse 软件数据，但不会删除：

- 用户自行保存于其他目录的本地媒体。
- 本地原始字幕文件。
- WebDAV 服务器上的媒体。

如果不能确认目录用途，不要删除。

## 16. 恢复后来源显示不可用

备份只恢复 XFVerse 软件记录，不会恢复：

- 已移动或删除的本地文件。
- 离线磁盘。
- 已改变的 WebDAV 地址或权限。
- 已失效的凭据。

恢复真实来源后重新扫描，或重新配置相应连接。

## 仍未解决时提供

- XFVerse 版本和 Windows 架构。
- 操作类型：备份、恢复、清缓存、卸载或升级。
- 用户数据目录的大致大小，不提供完整路径。
- 错误摘要。
- 是否已退出 XFVerse。
- 是否有完整备份。
- 问题时间附近的脱敏日志。
