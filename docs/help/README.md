## 开始排查前

按以下顺序完成最小检查：

1. 记录当前页面和完整错误摘要。
2. 记录问题发生时间，精确到分钟。
3. 在“设置 → 关于”记录 XFVerse 版本。
4. 确认安装包架构与 Windows 架构一致。
5. 判断问题只影响某个来源、某项外部服务，还是整个应用。
6. 正常退出 XFVerse；如果选择了缩小到托盘，应从托盘退出。
7. 重新打开后只重试一次，避免连续重复写操作。
8. 需要日志时，只截取问题时间附近的少量脱敏内容。

## 常见问题速答

### 删除记录会删除视频吗

不会。“删除记录”清理 XFVerse 中的相关软件记录，不删除本地或 WebDAV 文件。来源仍在启用扫描路径中时，后续扫描可能再次发现它。

### 移出媒体库与删除记录有什么区别

- 移出媒体库：隐藏记录，保留 metadata、观看状态和收藏状态，可恢复。
- 删除记录：删除相关软件记录和用户数据，不删除真实文件。

### 清理缓存会删除影片吗

不会。缓存清理只处理 XFVerse 管理的缓存或缓存记录，不应进入真实媒体目录。

### 为什么发现页有影片但不能播放

发现页可以建立只有元数据的条目。没有真实本地或 WebDAV 播放源时，只能查看资料或设置适用状态。

### 为什么电视剧不出现在观影洞察或 AI 推荐中

XFVerse 1.0 的观影洞察、AI 推荐画像和相关输入为 Movie-only。电视剧不进入这些能力。

### “退出登录”为什么没有云账号效果

XFVerse 1.0 只有本地用户资料，没有云账号、跨设备同步或真实登录系统。“退出登录”是保留入口。

### 外部服务未配置时软件还能用吗

可以。不同服务只影响对应能力：

- TMDB：在线搜索、榜单、识别和元数据。
- OMDb：部分补充评分。
- OpenSubtitles：在线字幕搜索和下载。
- AI：辅助识别、电影推荐和电影画像。
- WebDAV：远端扫描和播放。

本地媒体库与基础播放不要求同时配置所有外部服务。

## 高风险操作规则

### 不要直接删除数据库

`media-library.db` 保存媒体记录、来源、状态和历史。启动失败时，删除数据库不是首选修复。

正确顺序：

1. 退出 XFVerse。
2. 备份整个用户数据目录。
3. 保留原数据库副本。
4. 收集错误摘要和脱敏日志。
5. 再按[缓存备份与恢复](缓存备份与恢复.md)执行恢复。

### 不要混用程序目录和用户数据目录

- 程序目录保存应用文件。
- `%LOCALAPPDATA%\MediaLibrary` 保存当前用户的软件数据。

覆盖安装或默认卸载不应删除用户数据。不要把真实媒体文件放入 XFVerse 用户数据目录。

### 不要关闭系统安全功能作为长期方案

安装器未签名时 Windows 可能显示未知发布者。应核对来源、文件名和 SHA-256，而不是长期关闭 SmartScreen、防火墙或安全软件。

### 不要公开敏感信息

提交问题时不要提供：

- API Key、Token、密码。
- 完整 WebDAV URL、账号或远端路径。
- 完整本地用户路径。
- 完整数据库、配置文件或日志。
- 私有媒体清单和未处理截图。

## 需要提交问题时

建议提供：

- XFVerse 版本。
- 安装包架构：x64 或 ARM64。
- Windows 版本和系统架构。
- 问题发生时间。
- 影响的功能和最短复现步骤。
- 页面显示的脱敏错误摘要。
- 问题时间附近的少量脱敏日志。
- 是否只影响本地、WebDAV 或某项外部服务。

完整模板见[诊断与隐私](诊断与隐私.md)。

## 当前已知限制

- 正式安装包当前可能显示未知发布者。
- XFVerse 1.0 没有自动更新器。
- x64 与 ARM64 应使用对应独立安装包。
- Windows 11 x64 原生和 Windows 10 x64 尚未完成最终 RC。
- 当前 Windows 11 ARM64 设备的 UI、播放器和业务人工验收已由用户确认通过。
- 大体积 WebDAV 视频、HEVC 4K 和不同网络环境的表现仍需最终 RC 记录。
- 本轮人工验收未形成自动化截图归档；后续发布截图仍需保持脱敏。
