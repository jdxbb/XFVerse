# 媒体库与数据操作问题

Last updated: 2026-06-21

## 适用范围

本文处理媒体库空结果、条目无法打开、批量操作、移出与恢复、删除记录、重复内容和用户状态更新问题。

完整功能说明见[软件使用说明书第 6、7、10 章](../使用说明书.md)。

## 1. 媒体库为空或搜索不到内容

### 常见原因

- 尚未完成扫描。
- 搜索关键词或筛选组合过严。
- 当前只显示电影、电视剧或某一来源。
- 条目位于“其他”或已移出内容。
- 来源暂时不可用。

### 检查步骤

1. 清空搜索关键词。
2. 将内容类型切换到“全部”。
3. 清除年代、观看状态、收藏状态和标签筛选。
4. 同时允许本地和网盘来源。
5. 清除“有播放源/无播放源”条件。
6. 切换海报和列表视图确认。
7. 查看“已移出内容”。
8. 返回扫描任务检查最近扫描结果。

清除筛选只改变当前显示，不会修改媒体数据。

## 2. 条目存在但无法打开详情

### 可能原因

- 条目是未识别剧集候选，尚无稳定的 Series、Season 或 Episode 目标。
- 元数据记录缺少稳定标识。
- 关联记录已被删除或来源状态变化。
- 页面加载外部资料失败。

### 处理步骤

1. 刷新媒体库。
2. 确认条目属于电影、电视剧还是其他。
3. 对未识别内容使用识别修正或人工聚合。
4. 外部资料失败时先测试 TMDB，但不要删除现有来源。
5. 如果只在某个筛选下异常，记录筛选组合。

## 3. 条目有资料但不能播放

这通常是“元数据条目”：

- 有标题、海报和简介。
- 没有真实本地或 WebDAV 播放源。
- 可以查看资料或设置适用状态。
- 不能直接播放。

检查详情页的播放源区域。不要通过复制虚假路径或手工编辑数据库来制造播放源。

## 4. 批量操作按钮不可用

### 检查步骤

1. 确认已进入选择模式。
2. 确认至少选择一个当前可见项目。
3. 检查选择中是否混入不支持该操作的内容。
4. 人工聚合只适用于有播放源的未识别电视剧式项目。
5. 喜爱、想看等状态受电影或季状态约束。

退出选择模式或更改筛选后，应重新核对选择数量。

## 5. 批量操作部分失败

### 现象

结果显示成功、跳过、失败或已取消数量不一致。

### 处理步骤

1. 保存结果摘要。
2. 查看界面列出的前几个失败项。
3. 刷新媒体库，确认成功项已经生效。
4. 对失败项单独重试。
5. 如果失败集中在同一来源，先恢复该来源。
6. AI 或 TMDB 失败时保留原识别，不要先删除记录。

批量操作不是全有或全无。不要因为部分失败就重复执行整个选择集。

## 6. 移出媒体库后内容去了哪里

“移出媒体库”会：

- 从正常媒体库隐藏。
- 保留 metadata。
- 保留想看、喜爱、不想看和已看状态。
- 保留观看历史。
- 保留播放源记录。
- 不删除本地或 WebDAV 文件。

在媒体库的已移出内容区域查找并恢复。

## 7. 无法从已移出内容恢复

### 常见原因

- 相关软件记录已经被删除。
- 电视剧总览没有独立隐藏状态，需要恢复季或来源。
- 当前加载已移出列表失败。
- 关联数据处于不完整状态。

### 处理步骤

1. 刷新已移出内容。
2. 尝试恢复单个项目。
3. 电视剧按季或分组恢复。
4. 检查操作后的状态消息。
5. 如果记录已删除但真实文件仍在启用路径中，可重新扫描。

重新扫描可能重新建立记录，但不会保证恢复原有历史或收藏状态。

## 8. 删除记录会发生什么

“删除记录”会清除相关 XFVerse 软件数据，例如：

- 媒体实体和来源关联。
- 相关观看历史。
- 相关收藏和用户状态。

它不会：

- 删除本地文件。
- 删除 WebDAV 文件。
- 删除整个扫描目录。

如果真实文件仍位于启用扫描路径，后续扫描可能再次发现它。

## 9. 删除记录后又出现

### 原因

来源文件仍存在，且对应扫描路径仍启用。

### 处理方式

- 只是不想在媒体库显示：使用“移出媒体库”。
- 暂时不想扫描某个目录：停用对应扫描路径。
- 不再使用某个路径配置：删除扫描路径配置。

上述操作都不删除真实媒体。XFVerse 1.0 不提供删除本地或 WebDAV 原文件的媒体库命令。

## 10. 重复电影或重复剧集

### 常见原因

- 同一内容位于多个来源。
- 父目录和子目录被重复添加。
- 同一文件有不同路径表示。
- 识别结果不一致，未被聚合为多播放源。
- 来源暂时消失后形成异常记录。

### 安全处理

1. 先比较每个条目的来源。
2. 确认是真重复还是不同版本。
3. 同一实体的不同文件应优先作为多播放源保留。
4. 错误合并时使用“拆分”。
5. 错误识别时执行人工修正。
6. 不确定时先移出，不要直接批量删除记录。

## 11. 电影状态无法设置

电影状态存在互斥规则：

- 标记已看会清除想看。
- 改为未看会清除喜爱。
- 喜爱只适用于已看的电影。
- 已看的电影不使用想看。
- 不想看与喜爱、想看互斥。

按钮不可用或状态自动调整时，先检查这些规则。

## 12. 电视剧状态与预期不同

XFVerse 1.0 的电视剧状态层级为：

- Season：喜爱、想看、不想看、整季已看。
- Episode：单集已看。
- Series：主要用于组织资料和季列表。

不要期待 Episode 出现电影级的喜爱或想看按钮。标记整季状态前应核对当前季的集列表。

## 13. 收藏夹中移除是否会删除内容

不会。收藏夹的移除操作只取消当前“喜爱”或“想看”状态：

- 不删除媒体记录。
- 不删除播放源。
- 不删除真实文件。
- 不自动移出媒体库。

## 14. 状态更新失败

### 检查步骤

1. 确认条目有稳定的软件记录。
2. 检查状态是否与已看、喜爱、想看或不想看冲突。
3. 刷新详情页或收藏夹。
4. 避免连续重复点击。
5. 记录界面显示的安全错误摘要。

状态更新失败不应导致真实媒体丢失。

## 15. 删除、移出、路径删除和缓存清理对照

| 操作 | 软件记录 | 用户状态 | 真实媒体 |
| --- | --- | --- | --- |
| 移出媒体库 | 保留并隐藏 | 保留 | 不删除 |
| 恢复到媒体库 | 重新显示 | 保留 | 不修改 |
| 删除记录 | 删除相关记录 | 清除相关状态 | 不删除 |
| 删除扫描路径 | 删除路径配置 | 已有状态不等同于立即清除 | 不删除 |
| 清理缓存 | 清理缓存或缓存记录 | 保留 | 不删除 |

## 仍未解决时提供

- XFVerse 版本。
- 当前内容分类和筛选条件。
- 条目类型：电影、电视剧、季、集或其他。
- 操作名称和选择数量。
- 成功、跳过、失败、取消数量。
- 脱敏错误摘要。
- 是否为本地或 WebDAV 来源。
