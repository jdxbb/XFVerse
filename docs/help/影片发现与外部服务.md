# 影片发现与外部服务问题

Last updated: 2026-06-21

## 适用范围

本文处理影片搜索、榜单、TMDB、OMDb、OpenSubtitles、AI、网络、认证、额度和模型兼容问题。

各服务用途见[软件使用说明书第 9、12 章](../使用说明书.md)。

## 1. 影片搜索或榜单为空

### 常见原因

- TMDB 未配置或测试失败。
- 当前网络无法访问 TMDB。
- 搜索词、媒体类型或筛选条件过严。
- 外部服务返回空结果。
- 翻页请求失败。

### 检查步骤

1. 打开“设置 → API 配置”。
2. 确认 TMDB 已保存并测试成功。
3. 清除影片发现页的地区、语言、年代、观看和收藏筛选。
4. 核对当前搜索类型是电影还是电视剧。
5. 使用较短的标题关键词重试。
6. 榜单失败时切换榜单类型或时间范围。

影片发现失败不影响本地媒体库和本地播放。

## 2. TMDB 连接失败

### 常见原因

- Token 或 API Key 不完整。
- 使用了错误的凭据类型或粘贴了额外空格。
- 网络、代理、防火墙、DNS 或 TLS 环境异常。
- TMDB 服务暂时不可用。
- 请求被限流。

### 处理步骤

1. 从 TMDB 官方账户重新复制有效凭据。
2. 清空旧值后重新粘贴并保存。
3. 执行连接测试。
4. 检查系统时间、网络和系统代理。
5. 稍后重试服务端错误或限流。

### 影响范围

TMDB 不可用时可能影响：

- 影片搜索和榜单。
- 扫描在线识别。
- 人工识别修正。
- 海报、简介、演职员等在线资料补齐。

已有本地 metadata、媒体库和基础播放仍可使用。

## 3. TMDB 搜索结果不正确

### 处理步骤

1. 核对电影或电视剧类型。
2. 使用原文标题、译名或年份缩小范围。
3. 电视剧修正时确认 Series、Season 和 Episode。
4. 不要只根据海报确认。
5. 应用修正前核对标题、年份、季号和集号。

AI 候选仍需经过 TMDB 或用户确认，不能自动视为最终事实。

## 4. OMDb 连接失败或没有评分

### 常见原因

- API Key 未配置或无效。
- 当日额度已用完。
- 目标影片没有对应数据。
- 网络或服务端暂时失败。

### 影响范围

OMDb 主要补充适用的外部评分或资料。失败时：

- 详情页仍可打开。
- 本地或 WebDAV 播放不受影响。
- 可能只缺少对应评分。

### 处理步骤

1. 重新保存并测试 OMDb API Key。
2. 检查服务账户额度。
3. 确认目标电影已有可匹配外部标识。
4. 稍后刷新详情。

## 5. OpenSubtitles 配置测试失败

### 提示分类

| 提示 | 常见原因 |
| --- | --- |
| API Key 未配置 | 没有填写有效 API Key |
| 鉴权失败 | API Key、登录 token 或账号密码无效 |
| 拒绝访问 | 权限不足、额度限制或接口策略 |
| 服务端错误 | OpenSubtitles 暂时不可用 |
| 网络错误 | 网络、代理或防火墙阻止访问 |

### 处理步骤

1. 重新填写 API Key。
2. 账号和密码不是必填；不需要登录时可清空账号密码，使用 API key-only 模式。
3. 保存后执行“测试在线字幕”。
4. 登录失败但 API Key 能力通过时，根据实际需求决定是否保留账号配置。
5. 服务端或网络错误稍后重试。

不要提交 API Key、密码或登录 token。

## 6. AI 服务测试失败

### 常见原因

- 服务地址不是兼容 API 入口。
- API Key 无效。
- 模型名称不存在或当前账号无权使用。
- 某个路由任务选择了不兼容模型。
- 服务要求与当前兼容接口不同。
- 请求超时、限流或服务端失败。

### 检查步骤

1. 核对服务地址，避免把普通网页地址当成 API 地址。
2. 重新输入密钥并保存。
3. 测试默认模型。
4. 查看测试结果中通过和失败的模型数量。
5. 使用已确认可用的模型作为默认模型。
6. 再逐项设置扫描、修正、推荐和画像路由。

### 修复步骤

- 模型不存在：选择服务实际提供的模型。
- 无权限：检查账号权限和额度。
- 超时：稍后重试，或选择响应更稳定的模型。
- 部分路由失败：只更换失败任务的模型，不必重置全部配置。

## 7. AI 未配置时哪些功能仍可用

仍可使用：

- 本地和 WebDAV 来源配置。
- 基础扫描规则。
- 媒体库浏览。
- 电影、电视剧、季和集详情。
- 本地与 WebDAV 播放。
- 音轨、嵌入字幕和外挂字幕。
- 收藏、想看、已看和历史。

受影响：

- AI 辅助识别。
- 电影 AI 推荐。
- 电影观影画像。
- 部分 AI 标签或修正建议。

AI 不是应用启动和基础播放的必需条件。

## 8. AI 推荐为空或生成失败

### 检查步骤

1. 确认 AI 配置和推荐模型测试通过。
2. 确认媒体库中有足够的电影数据。
3. 检查推荐范围和观看状态筛选。
4. 减少过窄的自定义偏好。
5. 等待限流恢复后刷新。

### 边界

- XFVerse 1.0 的 AI 推荐为 Movie-only。
- 电视剧不会进入推荐输入。
- 缓存结果可能在服务失败时继续显示。
- 推荐理由是辅助信息，不是识别确认。

## 9. 观影画像显示未配置、数据不足或缓存

### 未配置

AI 服务或画像路由没有可用模型。完成设置测试后刷新。

### 数据不足

当前保存的电影观看数据不足以形成稳定画像。正常使用播放器并积累电影历史。

### 显示缓存

刷新失败但有旧快照时，页面可能保留上一次画像。界面应明确说明缓存或刷新失败。

电视剧数据不进入 1.0 画像输入。

## 10. 网络、代理、防火墙或 DNS 问题

### 判断方法

- 所有外部服务同时失败：优先检查网络、系统代理、DNS、系统时间和 TLS 环境。
- 只有一个服务失败：优先检查该服务配置、额度和状态。
- 浏览器能打开网页但 API 失败：不代表 API 地址、凭据或代理规则一定正确。

### 安全处理

1. 检查 Windows 网络状态。
2. 确认系统时间和时区正确。
3. 检查组织网络或代理是否允许对应 API。
4. 使用服务自带测试按钮。
5. 记录 HTTP 状态类别或安全错误摘要。

不要长期关闭防火墙、安全软件或证书验证。

## 11. 认证、额度和限流

### 认证失败

通常需要重新填写凭据并测试，不需要删除数据库。

### 额度不足

等待额度恢复、调整服务计划或降低请求频率。XFVerse 无法绕过服务方额度。

### 限流

暂停重复刷新，稍后重试。连续点击可能延长问题或产生更多失败日志。

## 12. 凭据换电脑后不可用

外部服务凭据使用当前 Windows 用户上下文的本机保护机制保存。复制用户数据到其他电脑、Windows 用户或重装系统后，受保护值不保证可直接解密。

恢复后应：

1. 打开设置页。
2. 重新输入 WebDAV 和外部服务凭据。
3. 分别保存并测试。
4. 不从日志或数据库中尝试提取明文密钥。

## 13. 外部服务降级对照

| 服务 | 失败时主要影响 | 仍可使用 |
| --- | --- | --- |
| WebDAV | 远端扫描和远端播放 | 本地扫描、本地播放、已有本地库 |
| TMDB | 搜索、榜单、在线识别和资料补齐 | 已有本地资料和基础播放 |
| OMDb | 对应补充评分 | 详情、TMDB 资料和播放 |
| OpenSubtitles | 在线字幕搜索和下载 | 嵌入字幕、外挂字幕和播放 |
| AI | AI 识别提示、电影推荐和电影画像 | 基础扫描、媒体库、播放和人工修正 |

## 仍未解决时提供

- XFVerse 版本。
- 失败的服务名称。
- 测试按钮显示的脱敏结果。
- HTTP 状态类别或错误类型。
- 问题发生时间。
- 是否所有服务同时失败。
- 是否使用系统代理。
- 失败的模型名称可以提供，但不要提供密钥。
