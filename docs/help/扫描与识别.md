# 扫描与识别问题

Last updated: 2026-06-21

## 适用范围

本文处理本地目录、WebDAV、扫描任务、文件忽略、识别错误、未识别内容、电视剧分组和媒体技术信息探测问题。

正常流程见[软件使用说明书第 4～6 章](../使用说明书.md)。

## 1. 本地目录无法添加或扫描

### 现象

- 文件夹选择后无法保存。
- 页面提示目录不可访问。
- 扫描立即失败或扫描数为零。

### 常见原因

- 路径不存在或磁盘离线。
- 当前 Windows 用户没有读取权限。
- 移动磁盘盘符已变化。
- 网络映射位置当前未连接。
- 添加了文件而不是文件夹。

### 检查步骤

1. 用文件资源管理器打开该目录。
2. 确认能列出目录并读取其中一个媒体文件。
3. 检查移动磁盘或网络位置是否在线。
4. 回到“扫描任务”，确认路径已启用。
5. 只运行该本地路径，观察当前阶段、扫描数、忽略数和错误数。

### 修复步骤

- 路径变化时编辑或重新添加正确目录。
- 目录暂时离线时，恢复连接后重扫。
- 权限不足时，在文件所有者允许的范围内调整目录权限。
- 不要以管理员身份长期运行 XFVerse来绕过正常权限。

删除本地扫描路径只删除软件配置，不删除目录或文件。

## 2. WebDAV 连接测试失败

### 现象

- 显示“测试失败”。
- 无法打开远端路径选择器。
- 保存连接后路径状态无法刷新。

### 常见原因

- 服务地址格式错误。
- 用户名、密码或授权方式不正确。
- 当前网络、代理、防火墙、DNS 或证书环境阻止连接。
- 服务不允许列出目录。
- 服务暂时不可用。

### 检查步骤

1. 在“设置 → API 配置”确认连接信息已保存。
2. 重新输入凭据，避免复制到前后空格。
3. 确认服务地址指向 WebDAV 入口，而不是普通网页地址。
4. 使用服务方支持的客户端确认同一账号能列出目录。
5. 重新执行“测试连接”。
6. 测试通过后再添加和启用扫描路径。

### 修复步骤

- 认证失败：重新确认账号、密码或服务方授权要求。
- 网络失败：恢复网络或正确配置系统代理后重试。
- 证书失败：确认系统时间正确，并使用服务方有效证书。
- 目录读取失败：确认账号至少有列目录和读文件权限。

### 不要做什么

- 不要把完整 WebDAV URL、账号和密码发到问题报告中。
- 不要为了测试而长期关闭防火墙或证书验证。
- 不要把删除远端文件当作连接修复。

## 3. WebDAV 测试成功但扫描没有结果

### 检查步骤

1. 确认至少有一个 WebDAV 扫描路径。
2. 确认该路径处于启用状态。
3. 确认路径下实际存在 XFVerse 可处理的媒体文件。
4. 在“扫描任务”运行 WebDAV 扫描，而不是本地扫描。
5. 查看扫描数、忽略数、错误数和原因摘要。
6. 检查媒体库的“其他”和已移出内容，不要只查看电影分类。

### 常见解释

- 扫描成功但文件被忽略。
- 文件进入未识别或待修正状态。
- 文件属于电视剧式内容，被组织到季结构。
- 原记录已存在，因此没有新增项目。
- 路径层级选择错误，只扫描了空目录。

不要以“成功绑定多少影片”作为扫描成功的唯一标准。

## 4. 扫描完成但媒体库没有新增

### 检查步骤

1. 清除媒体库搜索和筛选。
2. 切换到“全部”和“其他”。
3. 检查来源筛选是否只选择了本地或网盘。
4. 检查“有播放源/无播放源”筛选。
5. 查看已移出内容。
6. 比对扫描任务中的新增、更新、忽略和错误信息。

### 可能原因

- 文件更新了已有播放源，没有新建媒体。
- 相同内容被合并为同一电影或剧集的额外来源。
- 条目处于已移出状态。
- 识别证据不足，保留为未识别内容。
- 来源不可用，原记录只更新了可用性。

## 5. 文件被忽略

### 影响范围

忽略表示扫描过程决定不把该项目作为新的有效媒体来源处理，不等于文件已被删除。

### 检查步骤

1. 查看扫描原因摘要和近期扫描记录。
2. 确认目标是媒体文件，而不是目录、图片、说明文件或临时文件。
3. 检查文件扩展名和文件大小是否合理。
4. 确认没有重复添加父目录和子目录。
5. 确认来源当前可读取。

仍无法判断时，提供安全文件名样本即可；不要提交完整目录结构或私人媒体清单。

## 6. 扫描中出现失败或部分成功

### 常见原因

- 单个文件不可读取。
- WebDAV 网络中断或超时。
- 外部识别服务失败。
- ffprobe 无法读取媒体技术信息。
- 扫描被用户取消。

### 处理步骤

1. 记录失败数和原因摘要。
2. 判断失败是否集中在同一来源。
3. 先恢复网络、权限或文件可访问性。
4. 对受影响来源重新扫描。
5. 检查失败项目是否仍保留为未识别记录。

外部识别或技术探测失败不应导致真实媒体文件丢失。已有软件记录应尽量保留并允许重试。

## 7. 取消扫描后数据是否安全

取消只停止当前扫描任务：

- 已完成处理的项目可能已经写入或更新。
- 尚未处理的项目留待下次扫描。
- 不删除本地文件或 WebDAV 文件。
- 不应以取消作为数据库回滚工具。

取消后可刷新扫描任务和媒体库，再决定是否重扫。

## 8. 影片识别错误

### 现象

- 标题、年份或海报明显不符。
- 电影被识别为电视剧集，或反之。
- 多个来源被错误合并。

### 安全修复

1. 打开对应电影详情、季详情或集详情。
2. 从识别修正入口搜索正确目标。
3. 核对标题、年份、季号和集号。
4. 明确选择目标后确认修正。
5. 如果只是某个播放源错误合并，先拆分该来源，再修正。

AI 结果只作为搜索提示，不能替代用户确认。修正失败或超时时，界面会尽量回滚事务；不要连续重复点击确认。

## 9. 条目显示未识别、待人工确认或识别失败

### 含义

- 未识别：有真实来源，但证据不足以绑定正式影片。
- 待人工确认：存在低置信候选，需要用户核对。
- 识别失败：本次识别没有形成安全结果。

这些状态优先避免高置信错绑，不代表文件损坏。

### 处理步骤

1. 先确认来源可访问。
2. 检查文件名是否包含可辨识标题、年份、季号和集号。
3. 配置并测试 TMDB。
4. 按需配置 AI 辅助，但仍人工确认。
5. 从详情或批量识别入口重新处理。

不要为了“识别更多”而把不确定候选强行绑定。

## 10. 电视剧季集没有正确分组

### 检查步骤

1. 确认文件属于同一部电视剧和同一季。
2. 核对季号、集号和文件顺序。
3. 检查是否混入已识别项目或无播放源项目。
4. 对符合条件的未识别来源使用“人工聚合为季”。
5. 在聚合界面逐项确认集号。
6. 聚合完成后再执行识别或季级修正。

人工聚合只修改 XFVerse 中的 TV 关系，不移动、重命名或删除真实文件。

目标集已存在来源时，新来源应作为额外播放源保留，而不是覆盖文件。

## 11. 复扫后出现来源消失或重新出现

### 来源暂时离线

XFVerse 可能把来源标记为不可用。恢复磁盘、网络或 WebDAV 后重新扫描，系统会尝试重新关联。

### 安全复扫规则

- 不需要先删除记录。
- 不需要删除数据库。
- 不要重复添加相同路径。
- 恢复来源后按来源类型运行对应扫描。
- 重扫不会创建指向不存在文件的有效播放源。

如果复扫产生疑似重复项，先保留两项并记录来源信息，不要直接批量删除。

## 12. 媒体技术信息为空或探测失败

### 影响范围

分辨率、时长、码率、编码等字段可能为空，但媒体记录和播放源仍可保留。

### 处理步骤

1. 确认来源文件可读取。
2. 对 WebDAV 来源确认网络稳定。
3. 在详情页使用手动探测。
4. 探测失败后保留原记录，稍后重试。

ffprobe 探测失败不等于视频一定无法播放，也不应触发删除来源。

## 仍未解决时提供

- XFVerse 版本。
- 来源类型：本地或 WebDAV。
- 扫描开始时间和结束时间。
- 扫描数、忽略数、错误数。
- 脱敏的原因摘要。
- 一个安全的文件名样本。
- 是否配置 TMDB 或 AI。
- 问题时间附近的扫描识别日志片段。

日志获取和脱敏方法见[诊断与隐私](诊断与隐私.md)。
