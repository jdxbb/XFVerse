# 诊断信息与隐私

Last updated: 2026-06-21

## 适用范围

本文说明如何获取版本、Windows 架构、日志和错误摘要，以及如何在不泄露路径、账号、媒体清单和凭据的情况下提交问题。

## 1. 先记录问题上下文

在重试或关闭页面前记录：

- 问题发生日期和时间，精确到分钟。
- 当前页面。
- 正在执行的操作。
- 页面显示的完整错误摘要。
- 问题是否可重复。
- 是否只影响本地、WebDAV 或某项外部服务。
- 最近是否升级、切换架构、修改来源或清理缓存。

不要连续重复执行删除、修正、批量或恢复操作。

## 2. 获取 XFVerse 版本

打开：

`用户菜单 → 设置 → 通用 → 关于`

记录显示的软件版本，例如 `1.0.0`。

同时记录使用的安装包架构：

- win-x64。
- win-arm64。

不要只写“最新版”；版本必须使用具体数字。

## 3. 获取 Windows 版本和架构

### 图形界面

打开“设置 → 系统 → 系统信息”，记录 Windows 版本和“系统类型”。

### PowerShell

```powershell
[Environment]::OSVersion.VersionString
[System.Runtime.InteropServices.RuntimeInformation]::OSArchitecture
```

架构通常显示 `X64` 或 `Arm64`。

## 4. 获取安装包 SHA-256

在安装包所在目录打开 PowerShell：

```powershell
Get-FileHash -LiteralPath ".\XFVerse-Setup-1.0.0-win-x64.exe" -Algorithm SHA256
```

ARM64 安装包应替换为对应文件名。提交哈希是安全的，但不要提交来源不明文件本身。

## 5. 日志目录

默认日志目录：

```text
%LOCALAPPDATA%\MediaLibrary\Logs
```

列出日志文件：

```powershell
Get-ChildItem -LiteralPath "$env:LOCALAPPDATA\MediaLibrary\Logs" -File |
    Sort-Object LastWriteTime -Descending |
    Select-Object Name, Length, LastWriteTime
```

该命令只读取文件列表，不修改日志。

## 6. 常见日志文件

日志按功能生成，目录中不一定同时存在全部文件。

| 文件 | 主要用途 |
| --- | --- |
| `mpv-playback.log` | 播放器加载、来源切换、音轨、字幕和播放错误 |
| `video-cache-debug.log` | 视频缓存和 WebDAV 缓存播放 |
| `poster-cache-debug.log` | 海报缓存获取和生成 |
| `scan-identification-debug.log` | 扫描识别阶段 |
| `watch-completion.log` | 观看完成判断 |
| `watch-insights-perf.log` | 观影洞察加载和性能 |
| `ai-perf-debug.log` | AI 请求和性能诊断 |
| `ai-pool-debug.log` | AI 推荐候选池诊断 |

扫描任务页面还会显示保存在软件数据库中的近期扫描记录和原因摘要。

## 7. 查看日志末尾

例如查看播放日志最后 200 行：

```powershell
Get-Content -LiteralPath "$env:LOCALAPPDATA\MediaLibrary\Logs\mpv-playback.log" -Tail 200
```

如果文件不存在，说明当前环境尚未生成该日志，或问题属于其他功能。

只复制与问题时间相邻的少量内容，不要默认提交整个日志文件。

## 8. 如何选择日志

| 问题 | 优先信息 |
| --- | --- |
| 安装器打不开 | 安装包文件名、SHA-256、Windows 提示 |
| 启动失败 | 错误摘要、Logs 中对应时间文件、是否升级 |
| 扫描或识别 | 扫描计数、原因摘要、`scan-identification-debug.log` |
| 播放或字幕 | 来源类型、播放位置、`mpv-playback.log` |
| 视频缓存 | 来源类型、缓存状态、`video-cache-debug.log` |
| 海报问题 | 页面、目标类型、`poster-cache-debug.log` |
| 观影洞察 | 当前页签、状态文字、`watch-insights-perf.log` |
| AI 推荐 | 模型名称、错误类别、AI 相关日志 |

不要为了收集日志而重复执行可能改变数据的操作。

## 9. 必须删除或替换的信息

提交前检查每一行，删除或替换：

- 完整本地路径。
- Windows 用户名。
- 完整 WebDAV URL。
- WebDAV 用户名和远端私有路径。
- API Key、Token、Password、Cookie。
- 私有媒体标题或文件名。
- 电子邮件、手机号和个人签名。
- 私人头像文件名或路径。

## 10. 推荐脱敏形式

| 原信息类型 | 推荐写法 |
| --- | --- |
| 本地路径 | `<本地媒体目录>\子目录\文件.ext` |
| 用户目录 | `%USERPROFILE%\...` |
| WebDAV 地址 | `https://<服务主机>/...` |
| 账号 | `<WebDAV账号>` |
| API Key | `<已删除>` |
| 电影文件名 | `<媒体A>.mkv` |
| 远端路径 | `<远端目录>/Season01/...` |

保留错误类型、状态码、时间、计数和文件扩展名通常足够诊断。

## 11. 凭据泄露处理

如果问题报告、截图或日志中已经泄露 API Key、Token 或密码：

1. 立即从共享位置删除内容。
2. 到对应服务撤销或轮换凭据。
3. 在 XFVerse 设置中更新新凭据。
4. 重新测试服务。
5. 不要只对截图打码后继续使用已泄露密钥。

XFVerse 的本机凭据保护不能防止用户主动复制并公开明文凭据。

## 12. 不要提交的文件

默认不要提交：

- `media-library.db`。
- 整个 `%LOCALAPPDATA%\MediaLibrary`。
- 完整配置文件。
- 完整 Logs 目录。
- 原始媒体或字幕文件。
- 用户资料 JSON 和头像。
- 包含真实媒体库的全屏截图。

只有在明确、可信、安全的诊断流程中，且已理解数据内容后，才考虑提供最小必要文件。

## 13. 截图脱敏

截图前：

1. 关闭包含密钥和密码的设置区域。
2. 隐藏本地路径、WebDAV 地址和账号。
3. 使用演示媒体或遮盖私人标题。
4. 避免拍到头像、邮箱和手机号。
5. 检查 Windows 标题栏、任务栏和通知。
6. 裁剪到与问题直接相关的区域。

不要使用历史设计草图代替当前应用问题截图。

## 14. 最小问题报告模板

```text
XFVerse 版本：
安装包架构：x64 / ARM64
Windows 版本与架构：
问题时间：
问题页面：
执行操作：
实际结果：
期望结果：
是否可重复：
来源类型：本地 / WebDAV / 无来源
最近变更：升级 / 改配置 / 扫描 / 清缓存 / 无
错误摘要：
脱敏日志片段：
```

不要在模板中填写凭据或完整路径。

## 15. 安装问题补充信息

提供：

- 安装包完整文件名。
- 文件大小。
- SHA-256。
- Windows 安全提示文字。
- 首次安装、修复、覆盖升级或跨架构切换。

不要提交安装包中的私人修改版本。

## 16. 扫描问题补充信息

提供：

- 本地或 WebDAV。
- 扫描路径数量，不提供完整路径。
- 扫描、忽略、错误计数。
- 原因摘要。
- 一个脱敏文件名样本。
- 是否配置 TMDB 和 AI。

## 17. 播放问题补充信息

提供：

- 本地或 WebDAV。
- 视频编码、分辨率和大致大小。
- 是否有声音和画面。
- 错误发生的播放位置。
- 是否只有某一个来源失败。
- `mpv-playback.log` 的脱敏片段。

## 18. 外部服务问题补充信息

提供：

- 服务名称。
- 测试结果。
- HTTP 状态类别或错误类型。
- 模型名称或字幕错误类别。
- 是否所有外部服务同时失败。

不要提供 API Key、Token、密码或完整服务地址。

## 19. 数据恢复问题补充信息

提供：

- 操作前后的 XFVerse 版本。
- 是否有完整备份。
- 故障发生于启动、升级、恢复还是清缓存。
- 数据目录大致大小。
- 错误摘要。

不要上传数据库作为第一步。

## 20. 日志保留说明

XFVerse 1.0 当前没有统一的用户可配置日志保留周期。建议：

- 保留最近一次未解决问题的相关日志。
- 问题解决后按需归档脱敏片段。
- 不公开长期积累的完整日志。
- 不在应用运行时批量移动正在写入的日志。
