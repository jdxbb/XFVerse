## 平台与架构

- X86 不受支持。
- 不提供 Portable 免安装包。
- x64 与 ARM64 使用独立安装包，不应混装程序文件。
- 两种架构共用产品版本、正式 AppId 和用户数据格式。
- ARM64 候选已完成原生 PE、ffprobe、安装、修复、卸载、首次启动、数据库初始化和 libmpv 基础媒体载入验证。
- x64 候选已在 Windows ARM64 的 x64 仿真环境完成相同的进程级验证，但尚未完成 Windows x64 原生环境验收。
- 具体 Windows 10/11 支持声明必须服从最终 RC 环境结果。

## 1.0 核心能力

### 媒体来源与扫描

- 本地文件夹扫描。
- WebDAV 连接、路径管理、扫描与远端播放。
- 本地与 WebDAV 独立扫描和取消。
- 安全复扫、来源重新关联和扫描记录。
- 电影、电视剧、季、集和未识别内容管理。
- 本地规则、TMDB、AI hint 和人工确认组成的识别流程。

### 媒体库与详情

- 全部、电影、电视剧和其他分类。
- 海报与列表视图。
- 搜索、来源、类型、年代、状态、标签和播放源筛选。
- 批量观看状态、AI 辅助识别、移出和删除记录。
- Movie、Series、Season、Episode 详情。
- 多播放源、默认源、技术探测、拆分和识别修正。

### 播放与字幕

- 基于 mpv 的本地与 WebDAV 播放。
- 播放、暂停、跳转、音量、全屏和 Episode 导航。
- 多播放源切换。
- 多音轨、嵌入字幕和本地外挂字幕。
- OpenSubtitles 在线搜索、下载、绑定和缓存管理。
- 观看进度、续播、已看判断和观影历史。

### 影片发现与个人内容

- TMDB 电影和电视剧搜索。
- 热门、高分和趋势榜单。
- 喜爱、想看、不想看和已看状态。
- 电影与电视剧季收藏。
- 电影 AI 推荐。
- 电影观影统计和 AI 画像。
- 本地个人资料和主题设置。

观影洞察、AI 推荐画像和相关输入在 1.0 中保持 Movie-only。

## 正式版发布工程变化

与历史测试安装链路相比，1.0 正式候选：

- 使用独立正式打包脚本和正式安装器定义。
- 使用稳定正式 AppId，不复用测试安装器身份。
- 从干净 staging 构建。
- 不读取或打包当前用户数据库、配置、日志、缓存、头像、历史或凭据。
- 每个安装包只携带目标架构的应用入口、libmpv 和 ffprobe。
- 正式 Release 二进制不携带 PDB 和开发机 CodeView 路径。
- 安装、修复、架构切换和默认卸载保留用户数据。
- 默认卸载不删除本地或 WebDAV 媒体。
- 随包提供第三方组件清单和许可证文本。

历史 `XFVerse-TestSetup-*` 仅为内部测试产物，禁止作为 1.0 正式包发布。

## 安装与升级

### 首次安装

1. 确认 Windows 架构。
2. 下载对应 x64 或 ARM64 安装包。
3. 核对完整 SHA-256。
4. 退出所有 XFVerse 实例。
5. 运行安装器。

完整步骤见[安装说明](../安装说明.md)。

### 覆盖升级

- 正常升级直接运行新版正式安装器。
- 不需要先删除用户数据。
- 建议升级前备份 `%LOCALAPPDATA%\MediaLibrary`。
- 应用启动时会按现有数据库初始化流程应用必要 migrations。
- XFVerse 1.0 没有自动更新器。

### 架构切换

- 先确认目标 Windows 架构。
- 使用目标架构安装包覆盖安装。
- 安装器会清理另一架构的 mpv 与 ffprobe 程序目录。
- 用户数据库、设置、状态、历史和缓存保持在原用户数据目录。

## 数据保留语义

- 移出媒体库：隐藏记录，保留 metadata 和用户状态。
- 删除记录：删除相关软件记录，不删除本地或 WebDAV 文件。
- 删除扫描路径：删除配置，不删除目录或远端内容。
- 清理缓存：只处理 XFVerse 缓存。
- 修复安装：补齐程序文件，不重置用户数据。
- 默认卸载：删除程序文件，保留用户数据。

完全清除软件数据必须由用户在备份后显式手工执行。

## 回滚原则

1. 退出 XFVerse，包括托盘实例。
2. 备份当前完整用户数据目录。
3. 保留失败版本的数据库和日志副本。
4. 卸载当前程序或运行目标版本安装器。
5. 不删除真实媒体目录或 WebDAV 内容。
6. 只有确认数据库版本兼容时才使用旧程序打开新数据库。
7. 需要恢复数据时，恢复完整且已知可用的备份，不拼接不同时间的数据库和配置。

数据库 migration 失败时，不要删除 `media-library.db` 作为第一步。

## 外部服务与网络

| 服务 | 对应能力 | 未配置或失败时 |
| --- | --- | --- |
| WebDAV | 远端扫描和播放 | 本地媒体能力仍可使用 |
| TMDB | 搜索、榜单、识别和资料 | 已有本地资料与基础播放仍可使用 |
| OMDb | 部分外部评分 | 只缺少对应补充评分 |
| OpenSubtitles | 在线字幕 | 嵌入字幕、外挂字幕和播放仍可使用 |
| AI 兼容服务 | 辅助识别、电影推荐和画像 | 基础扫描、媒体库、播放和人工修正仍可使用 |

WebDAV 播放依赖远端服务的 Range 支持、网络质量和大文件行为。

## 数字签名

当前两个候选安装包的 Authenticode 状态均为 `NotSigned`。Windows 可能显示 SmartScreen 或未知发布者提示。

用户应核对可信来源、文件名、架构和 SHA-256，不应通过长期关闭系统安全功能解决提示。

## 已知限制

- Phase 8 源码已提交，但当前候选包生成于该提交之前；GA 前仍需从干净提交重新构建并记录新校验值。
- Windows 11 x64 原生环境尚未验收；现有 x64 结果来自 Windows ARM64 仿真。
- Windows 10 x64 尚未验收，最终支持范围尚未冻结。
- 当前 Windows 11 ARM64 设备的主导航、业务功能和播放器人工验收已由用户确认通过；没有自动化截图记录。
- libmpv 基础媒体载入已通过，但不等同于播放器 UI、全屏、音量、轨道、字幕、WebDAV 和续播验收。
- 大体积 WebDAV 视频、HEVC 4K 和不同硬件解码环境仍需记录实测结果。
- 安装包未签名。
- 没有自动更新器。
- 没有云账号、真实登录或跨设备同步。
- TV 不进入 1.0 的观影洞察、AI 推荐画像和 fingerprint。
- 项目自身没有根 LICENSE，不应宣称项目代码已经采用某种开源许可证。
- GPL 原生组件已建立双架构对应源代码包；正式下载页必须把匹配源码包和 SHA-256 与安装包同时提供。
