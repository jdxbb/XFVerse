# Phase 8.9：Release Candidate 验收与 1.0 GA 发布

Last updated: 2026-06-21

## 阶段状态

- 状态：RC 已执行，GA 阻断。
- 前置条件：Phase 8.2～8.8 文档完成，P8-B05 已关闭。
- 当前状态：双架构构建、安装生命周期、首次启动、数据库完整性、升级副本、原生播放核心和发布清单验证已完成；当前 Windows 11 ARM64 设备人工验收已由用户确认通过，规定的 x64 系统矩阵未完成。
- 结论：不建议发布，存在以下 Blocker。

## 目标

用正式打包链路生成 RC，完成干净安装、升级、核心业务、文档和隐私验收；修复阻断问题后生成 XFVerse 1.0.0 GA 产物。

## 发布节奏

### RC1

- 从干净工作区和干净 staging 构建。
- 执行自动检查和全量人工验收。
- 只修 Blocker 和明确影响 1.0 的高优先级问题。

### RC2+

- 仅在 RC1 有代码或安装器修复时生成。
- 每个 RC 都必须重新执行敏感扫描、安装/升级验证和受影响回归。

### GA

- GA 代码与最后一个通过验收的 RC 必须可追踪。
- 如 GA 重新构建，必须确认源码、版本和构建参数一致，并重新生成 SHA-256。
- 只有 Blocker 为零时才能建议发布。

## 验收环境

至少覆盖：

1. 干净 Windows 11 x64 当前用户。
2. 干净 Windows 10 x64 当前用户；若无法提供，必须记录为 Blocker 或明确调整支持范围。
3. 干净 Windows ARM64 当前用户；最低 Windows 版本根据可用设备和 RC 结果冻结。
4. 已安装测试版或上一候选版的升级环境。
5. 有现有数据库副本的升级环境。
6. 无网络或外部 API 未配置环境。
7. 本地目录媒体环境。
8. WebDAV 测试环境。
