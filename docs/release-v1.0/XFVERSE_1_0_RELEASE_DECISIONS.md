# XFVerse 1.0 发布决策记录

Last updated: 2026-06-20

## 1. 目的

本文冻结 XFVerse 1.0 发布边界。Phase 8.4 根据用户新增的 ARM64 正式包要求，已按本文件约定记录并更新 Phase 8.1 的单架构决策。

## 2. 已冻结决策

| ID | 决策 | 状态 | 理由与后续约束 |
| --- | --- | --- | --- |
| RD-001 | 正式产品名为 XFVerse。 | 已冻结 | README、安装器、EXE 信息、设置页和发布说明统一使用该名称。历史内部命名只在兼容路径或代码名中保留。 |
| RD-002 | 正式版本号为 `1.0.0`。 | 已冻结 | Phase 8.3 建立单一版本源，贯穿程序集、文件属性、设置页、安装器和发布说明。 |
| RD-003 | 1.0 正式候选架构为 Windows x64 与 Windows ARM64。 | 已更新 | 两种架构分别生成 self-contained 安装包；最终操作系统版本支持声明仍以 Phase 8.9 RC 环境矩阵为准。 |
| RD-004 | ARM64 以独立正式候选安装包进入 1.0。 | 已更新 | Phase 8.4 已在 Windows ARM64 环境完成原生 publish、PE 架构、ffprobe 执行、安装/修复/卸载及双向架构切换验证；应用首次启动和播放仍转 Phase 8.9。 |
| RD-005 | X86 不支持。 | 已冻结 | 当前原生资源和运行时解析未形成完整 X86 发布链路。 |
| RD-006 | x64 与 ARM64 正式包均采用 self-contained 发布。 | 已更新 | 终端用户无需单独安装 .NET Desktop Runtime；每个安装包只携带自身架构的应用入口、libmpv 和 ffprobe。 |
| RD-007 | 正式版提供相互独立的 x64 与 ARM64 安装包。 | 已更新 | 两包共用稳定正式 AppId、版本和用户数据格式；切换架构时覆盖程序文件并删除另一架构的原生程序目录，不触碰用户数据。 |
| RD-008 | 安装范围为当前用户，不请求管理员权限。 | 已冻结 | 默认安装到当前用户 LocalAppData 下的 Programs 目录；不得写入系统级受保护目录。 |
| RD-009 | 1.0 不提供 Portable 包。 | 已冻结 | 当前数据路径、启动初始化和原生资源模型均按安装版设计。 |
| RD-010 | 正式安装器必须使用全新、稳定的正式 AppId。 | 已冻结 | 不复用 `XFVerse.TestInstaller*.iss` 中的测试 AppId，防止正式包误升级、覆盖或继承测试包危险行为。 |
| RD-011 | 正式打包使用独立脚本和空 staging。 | 已冻结 | 禁止复制当前用户 AppData、数据库、配置、日志、缓存、头像、播放历史或私有媒体信息。 |
| RD-012 | 用户数据默认目录继续使用 `%LOCALAPPDATA%\MediaLibrary`。 | 已冻结 | 这是现有兼容路径。1.0 不为品牌一致性强制改名，避免引入高风险数据迁移；文档需说明该内部目录名。 |
| RD-013 | `XFVERSE_APPDATA_DIR` 仅作为高级诊断/测试覆盖项。 | 已冻结 | 不作为普通用户安装步骤，不应在 README 快速开始中突出。 |
| RD-014 | 覆盖升级必须保留数据库、配置、用户状态、播放历史、缓存和用户资料。 | 已冻结 | 安装器只更新程序文件；应用启动后由现有数据库初始化流程应用 migrations。Phase 8.4/8.9 必须验证旧数据升级。 |
| RD-015 | 修复安装只修复程序文件，默认不清理用户数据。 | 已冻结 | 不将“重置数据”伪装成修复步骤。 |
| RD-016 | 默认卸载保留用户数据。 | 已冻结 | 1.0 安装器不提供自动删除用户数据的选项；完整清理采用帮助文档中的显式手工步骤，并要求用户先确认备份。 |
| RD-017 | 安装器不得删除本地媒体或 WebDAV 内容。 | 已冻结 | 与产品数据语义红线一致。 |
| RD-018 | 1.0 不包含自动更新器。 | 已冻结 | 使用手动下载新版正式安装器进行升级；README、安装说明和帮助文档必须统一描述。 |
| RD-019 | 数字签名不作为 1.0 功能承诺。 | 已冻结 | 当前无证书和签名配置。若发布前提供有效证书，可作为不改变功能范围的发布增强；否则安装说明明确 Windows 可能显示未知发布者。 |
| RD-020 | 正式发布语言为简体中文。 | 已冻结 | 英文 README、英文安装器和英文手册不作为 1.0 GA 条件。 |
| RD-021 | Markdown 是正式文档单一源。 | 已冻结 | PDF 等导出物只能由已验收 Markdown 生成，不得成为独立事实源。 |
| RD-022 | 代码与 RC 行为优先于历史计划和旧截图。 | 已冻结 | 文档发现不一致时先修正文档或记录产品缺陷，不以旧文档倒推当前功能。 |
| RD-023 | 现有测试安装包永远不作为正式发布候选。 | 已冻结 | 测试链路携带并覆盖用户数据，必须与正式链路隔离。 |
| RD-024 | 1.0 的发布形式是安装器，不单独承诺压缩包分发。 | 已冻结 | 如后续提供诊断压缩包，应标记为内部验证产物。 |
| RD-025 | 发布校验至少包含 SHA-256。 | 已冻结 | Phase 8.9 为最终安装器生成并发布校验值；不以文件名或大小替代完整性校验。 |

## 3. 安装、升级与卸载模型

```text
首次安装
  └─ 安装程序文件
      └─ 首次启动创建用户数据目录和数据库

覆盖升级
  └─ 更新程序文件
      └─ 保留用户数据
          └─ 应用启动时执行已有数据库 migrations

修复安装
  └─ 重写/补齐程序文件
      └─ 不重置数据库和配置

默认卸载
  └─ 删除程序文件和快捷方式
      └─ 保留用户数据

完整清理
  └─ 用户明确确认并退出应用
      └─ 按帮助文档手工删除 XFVerse 用户数据目录
```

## 4. 发布安全门

以下问题未解决前不得生成 GA：

1. 正式打包链路仍可能复制真实用户数据。
2. 正式安装器仍可能删除或覆盖用户数据。
3. 应用、文件属性、安装器和发布说明没有统一使用 `1.0.0`。
4. 敏感字段仍只能以 Base64 还原且未有兼容处理或准确风险说明。
5. 第三方分发组件的许可证和随包声明不完整。
6. 正式安装器与测试安装器没有隔离 AppId、输出目录和产物名称。

## 5. 正式产物命名基线

Phase 8.4 可根据工具限制调整分隔符，但含义必须保持：

```text
XFVerse-1.0.0-win-x64-setup.exe
XFVerse-1.0.0-win-x64-setup.exe.sha256
XFVerse-1.0.0-win-arm64-setup.exe
XFVerse-1.0.0-win-arm64-setup.exe.sha256
XFVerse-1.0.0-release-notes.md
THIRD_PARTY_NOTICES.md
```

测试、RC 和 GA 产物不得使用同一无后缀文件名覆盖：

```text
XFVerse-1.0.0-rc.N-win-x64-setup.exe
XFVerse-1.0.0-win-x64-setup.exe
XFVerse-1.0.0-rc.N-win-arm64-setup.exe
XFVerse-1.0.0-win-arm64-setup.exe
```

## 6. 环境和依赖声明

- 本地媒体库和本地播放不要求 WebDAV、TMDB、OMDb、OpenSubtitles 或 AI 配置。
- 在线发现和识别需要 TMDB 与网络。
- OMDb 仅补充外部评分。
- 在线字幕需要 OpenSubtitles 与网络。
- AI 分类、修正建议、推荐和画像按功能降级，不能成为应用启动前置条件。
- WebDAV 播放的稳定性依赖服务器 Range 支持、网络质量和大文件行为。
- 正式包体积和最低可用磁盘空间在 Phase 8.4 实测后填写，不在 8.1 估算为最终要求。

## 7. 仍需后续阶段验证的决策

以下不是范围待决，而是已确定方向但尚需产物验证：

- Windows x64 与 Windows ARM64 的最低可用系统版本、首次启动和完整功能行为。
- self-contained 包的最终体积、启动时间和原生依赖完整性。
- 覆盖升级、修复和卸载的数据保留。
- Windows 未签名安装提示的实际文案与帮助指引。
- 旧数据库迁移到 1.0.0 的成功、失败和备份恢复路径。
