# Phase 8 Known Issues

Last updated: 2026-06-20

## Blocker

### P8-B05 - GPL 对应源代码公开获取安排未完成

- Phase 8.4 已完成第三方组件清单、随包 LICENSE / NOTICE、版本和哈希审计。
- 当前剩余风险：公开分发 GPL 原生组件时，需要确保对应源代码或等价源代码获取方式与下载包同时提供。
- 要求：Phase 8.8 在 README、发布说明和分发页面中提供稳定的对应源代码入口，并复核实际分发方式。
- 目标阶段：8.8。

## Deferred

### P8-D01 - ARM64 正式支持推迟到 1.0 之后

- 当前已有 ARM64 测试发布路径，但尚无真实设备验收证据。
- Phase 8.1 已决定 ARM64 不进入 1.0 GA；可保留内部实验构建，后续有真实设备完整验收后再评估。

### P8-D02 - 数字签名不作为 1.0 功能承诺

- Phase 8.4 正式候选安装包的 Authenticode 状态为 NotSigned。
- 如发布前没有可用证书，安装说明和发布说明必须明确 Windows 可能显示未知发布者，不得引导用户关闭系统安全功能。

### P8-D03 - 正式包体积优化

- Phase 8.4 已将正式 staging 收口到 516.47 MiB，安装包为 242.20 MiB。
- 主要剩余体积来自 libmpv、ffprobe、自包含 .NET/WPF 运行时和高分辨率人格海报。
- 1.0 不启用 trimming、单文件、ReadyToRun 或原生二次压缩；进一步优化推迟到后续版本。

### P8-D04 - 自动更新器

- Phase 8.1 已决定 1.0 不包含自动更新器。
- 用户通过手动下载新版正式安装器覆盖升级，安装器必须保留用户数据。

### P8-D05 - 日志目录与诊断入口尚未完全统一

- 日志目录、公共脱敏和扫描历史隐私保护已在 Phase 8.3 实施。
- 本 Deferred 只保留用户诊断入口、日志打开方式、保留周期和帮助文档组织工作，由 Phase 8.7 完成。

### P8-D06 - 本地用户资料不等于账号系统

- 用户菜单中的“退出登录”当前是未接入占位能力。
- 1.0 不宣称云账号、登录、退出登录或跨设备同步；相关入口需在使用说明书和帮助文档中准确说明。

### P8-D07 - TV 洞察与推荐能力不在 1.0 范围

- TV 不进入 Watch Insights、AI 推荐画像和 fingerprint。
- 使用说明书和 README 不得将 Movie-only 能力泛化为全部媒体。

### P8-D08 - 播放器高负载场景仍需 RC 记录

- 现有阶段文档记录 70GB+ WebDAV 长时间播放可能卡顿、HEVC 4K 依赖硬件解码安全策略。
- Phase 8.9 必须按 RC 环境矩阵验证并在发布说明中保留真实限制。

### P8-D09 - 历史 UI 规格与当前实现存在文档漂移

- `DesignDraft` 记录了 UI 目标和形成过程，部分旧条目已经被 Phase 7 实际实现调整。
- 典型差异包括 WPF UI 库目标未形成包依赖、设置页主题入口调整、扫描占位指标移除和视频缓存 UI 收缩。
- `XFVERSE_1_0_SOFTWARE_DESIGN.md` 以当前代码为事实基线；历史规格不在 Phase 8.2 反向重写。

## Noise

### P8-N01 - 历史测试安装包

- 仓库根目录和 artifacts 中存在本地测试产物。
- 这些文件不作为正式产物，也不应被正式脚本输入。

### P8-N02 - 本地打包工具下载

- Inno Setup 可能由脚本下载到临时目录。
- 正式流程需要锁定来源和校验，但首次准备工具本身不是产品缺陷。

### P8-N03 - 历史草图资源命名和占位表现

- 部分草图包含临时高饱和配色、黑色占位块或非标准扩展名。
- 草图只用于布局追溯，不作为正式 UI 资源或 RC 截图。
