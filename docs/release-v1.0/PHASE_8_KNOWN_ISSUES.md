# Phase 8 Known Issues

Last updated: 2026-06-21

## Blocker

### P8-B06 - GA 构建不可追踪

- Phase 8 源码与文档已提交到 `codex/xfverse-1.0-release`。
- 当前候选包生成时间早于该干净提交，现有发布清单仍记录构建时的脏工作区基线。
- 关闭条件：从已提交分支重新构建双架构候选包，并重新复算、复验和记录提交号。

### P8-B07 - Windows 11 x64 原生环境缺失

- x64 候选仅在 Windows ARM64 的 x64 仿真环境完成安装、启动和原生库进程验证。
- 仿真结果不能替代声明支持的 Windows 11 x64 原生 P0 验收。
- 关闭条件：在 Windows 11 x64 原生环境完成安装、首次启动、升级、播放和卸载矩阵。

### P8-B08 - Windows 10 x64 环境缺失

- 当前产品计划仍声明 Windows 10 / Windows 11。
- 关闭条件：在 Windows 10 x64 完成 P0 验收，或明确移除 Windows 10 支持并同步 README、安装说明、发布说明和阶段矩阵。

### 已关闭：P8-B09 - 完整 UI 与业务矩阵

- 用户于 2026-06-21 确认当前 Windows 11 ARM64 设备人工验收无异常。
- 本轮人工验收未形成自动化截图或逐项机器记录，不作为 Windows x64/Windows 10 兼容性证据。
- P8-B09 关闭；P8-B07 和 P8-B08 继续约束其他声明支持环境。

### 已关闭：P8-B05 - GPL 原生对象来源与对应源代码

- 已于 2026-06-21 关闭。
- x64 与 ARM64 ffprobe 已替换为可验证固定发布。
- 双架构对象、对应源代码包、对象映射、SHA-256 和安装生命周期均已通过。

## Deferred

### P8-D01 - 双架构应用级 RC 仍缺完整页面验收

- 双架构干净首次启动、数据库初始化、正常退出和 libmpv 基础媒体载入已通过。
- ARM64 现有数据副本启动、数据库完整性和日志脱敏抽查已通过。
- 当前 Windows 11 ARM64 页面级业务与播放器人工验收已通过；其他系统环境仍由 P8-B07/P8-B08 阻断。

### P8-D02 - 数字签名不作为 1.0 功能承诺

- Phase 8.4 正式候选安装包的 Authenticode 状态为 NotSigned。
- 如发布前没有可用证书，安装说明和发布说明必须明确 Windows 可能显示未知发布者，不得引导用户关闭系统安全功能。

### P8-D03 - 正式包体积优化

- P8-B05 合规重建后，x64 staging 为 556.31 MiB、安装包 254.59 MiB；ARM64 staging 为 468.72 MiB、安装包 229.13 MiB。
- 主要剩余体积来自 libmpv、ffprobe、自包含 .NET/WPF 运行时和高分辨率人格海报。
- 1.0 不启用 trimming、单文件、ReadyToRun 或原生二次压缩；进一步优化推迟到后续版本。

### P8-D04 - 自动更新器

- Phase 8.1 已决定 1.0 不包含自动更新器。
- 用户通过手动下载新版正式安装器覆盖升级，安装器必须保留用户数据。

### P8-D05 - 诊断入口的最终 RC 复核

- 日志目录、公共脱敏和扫描历史隐私保护已在 Phase 8.3 实施。
- Phase 8.5 安装说明已记录默认日志目录和最短启动失败检查。
- Phase 8.7 已完成按错误现象组织的帮助中心、日志文件说明、当前日志保留状态、脱敏提交模板和安装/使用文档交叉链接。
- Phase 8.9 已抽查 ARM64 升级副本日志，未发现异常、完整用户目录、WebDAV URL 或凭据标签。
- 当前设备的 UI 错误提示与帮助步骤由用户人工验收确认无异常。

### P8-D06 - 本地用户资料不等于账号系统

- 用户菜单中的“退出登录”当前是未接入占位能力。
- Phase 8.6 使用说明书和 Phase 8.7 帮助中心均已明确本地资料、无云同步和“退出登录”占位边界。

### P8-D07 - TV 洞察与推荐能力不在 1.0 范围

- TV 不进入 Watch Insights、AI 推荐画像和 fingerprint。
- Phase 8.6 使用说明书已按 Movie-only 描述；Phase 8.8 README 不得将这些能力泛化为全部媒体。

### P8-D08 - 播放器高负载场景仍需 RC 记录

- 现有阶段文档记录 70GB+ WebDAV 长时间播放可能卡顿、HEVC 4K 依赖硬件解码安全策略。
- Phase 8.9 必须按 RC 环境矩阵验证并在发布说明中保留真实限制。

### P8-D09 - 历史 UI 规格与当前实现存在文档漂移

- `DesignDraft` 记录了 UI 目标和形成过程，部分旧条目已经被 Phase 7 实际实现调整。
- 典型差异包括 WPF UI 库目标未形成包依赖、设置页主题入口调整、扫描占位指标移除和视频缓存 UI 收缩。
- `XFVERSE_1_0_SOFTWARE_DESIGN.md` 以当前代码为事实基线；历史规格不在 Phase 8.2 反向重写。

### P8-D10 - 用户文档截图与最终 RC 实机复核

- Phase 8.6 已按当前代码和功能矩阵完成 13 章、4 个附录的正文，未使用历史 DesignDraft 冒充成品截图。
- Phase 8.7 已完成帮助中心索引和 8 篇主题文档，排障命令已在当前 Windows 环境验证。
- 脱敏截图、页面文案、错误提示、日志名称和最终安装包实机行为的一致性复核纳入 Phase 8.9。
- 如 RC 与正文存在差异，应以 RC 为准更新使用说明书和帮助文档，不得把差异仅记录为已知问题。

### P8-D11 - 官网正式下载入口与 GA 状态同步

- `xfverse.fun` 已部署面向用户的文档中心和新版官网 UI。
- 官网当前明确标注 Release Candidate，不提供虚假的正式下载按钮。
- 关闭条件：P8-B06、P8-B07、P8-B08 处理完成后，从最终 GA 制品复算文件名、大小和 SHA-256，同步官网发布状态与正式下载入口。
- 正式同步前不得把现有 RC 候选包描述为 GA。

## Noise

### P8-N01 - 历史测试安装包

- 仓库根目录和 artifacts 中存在本地测试产物。
- 这些文件不作为正式产物，也不应被正式脚本输入。

### P8-N02 - 本地打包工具下载

- Inno Setup 可能由脚本下载到临时目录。
- 正式流程需要锁定来源和校验，但首次准备工具本身不是产品缺陷。

### P8-N03 - 历史草图资源命名和占位表现

- 部分草图包含临时高饱和配色、黑色占位块或非标准扩展名。
- 草图只用于布局追溯，不作为正式 UI 资源或 RC 截图。

### P8-N04 - ARM64 系统上的 x64 仿真卸载残留

- 在 Windows ARM64 上运行并卸载 x64 仿真候选后，少量 x64 运行时文件可能因仿真进程文件锁而延迟删除。
- ARM64 原生安装包卸载未出现该现象。
- 该现象不能代表 Windows x64 原生结果；由 P8-B07 的原生环境验收最终判断。
