# XFVerse 1.0 文档信息架构

Last updated: 2026-06-21

## 1. 目标

建立 XFVerse 1.0 文档的职责边界、目录、事实来源、交叉链接和维护规则，避免 README、安装说明、软件使用说明书和帮助文档互相复制或定位混淆。

## 4. 事实来源优先级

出现冲突时按以下顺序处理：

1. 正式 RC/GA 包在目标环境中的实际行为。
2. 当前分支代码、项目文件、数据库模型和打包脚本。
3. `XFVERSE_1_0_FEATURE_MATRIX.md` 和发布决策记录。
4. 当前阶段计划、日志和 Known Issues。
5. 历史阶段文档。
6. 旧截图、旧测试包和设计草图。

设计草图用于解释 UI 形成过程和设计意图，不用于覆盖当前实际 UI。软件设计说明书应记录既有流程：先绘制 UI 草图和页面规格，再依据草图修改实际 UI；Phase 8 不承担重新绘制草图或重新改造 UI。

## 5. 内容归属规则

### 5.1 安装信息

- 安装说明保存完整步骤。
- README 只保留最短安装入口和链接。
- 帮助文档只处理安装失败、启动失败、升级冲突和数据恢复。
- 使用说明书从“首次启动后的软件界面”开始，不重复安装器每一步。

### 5.2 功能信息

- 使用说明书是正式功能全量说明。
- 帮助文档按任务和问题拆分，并链接回使用说明书对应章节。
- README 只列核心能力，不列所有按钮和筛选条件。
- 软件设计说明书解释实现边界，不替代用户操作说明。

### 5.3 数据与删除语义

以下定义必须在安装说明、使用说明书和相关帮助文档中保持完全一致：

- 移出媒体库：只改变可见性。
- 删除记录：删除 XFVerse 软件记录，不删除真实媒体。
- 删除扫描路径：删除扫描配置，不删除目录或远端内容。
- 清理缓存：删除 XFVerse 缓存，不删除真实媒体。
- 默认卸载：删除程序文件，保留用户数据。
- 完整清理：用户确认后手工删除 XFVerse 用户数据。

### 5.4 外部服务

- 安装说明只解释是否必需、基础网络要求和首次配置入口。
- 使用说明书解释每项服务支持哪些功能。
- 帮助文档解释连接测试、常见错误和安全检查。
- 软件设计说明书解释服务边界、缓存、节流和降级。
- 任何文档不得包含真实 API Key、Token、账号、密码或完整私有 URL。

## 6. 术语基线

| 统一术语 | 含义和使用规则 |
| --- | --- |
| XFVerse | 对外产品名。 |
| 媒体库 | XFVerse 中可见的电影、电视剧和其他媒体记录集合。 |
| 媒体来源 | 本地文件或 WebDAV 文件在软件中的来源记录。 |
| 本地目录 | 用户选择供 XFVerse 扫描的本地文件夹。 |
| WebDAV 扫描路径 | WebDAV 连接下启用扫描的远端路径配置。 |
| 移出媒体库 | 隐藏记录但保留 metadata 和用户状态。 |
| 删除记录 | 删除软件记录，不删除真实媒体文件。 |
| 清理缓存 | 删除 XFVerse 创建的缓存副本。 |
| 电影 / Movie | 电影实体和对应功能范围。用户文档优先写“电影”。 |
| 电视剧 / TV Series | 剧集总实体。用户文档优先写“电视剧”。 |
| 季 / Season | 电视剧的一季。 |
| 集 / Episode | 某一季中的单集。 |
| 未识别 / 待修正 | 证据不足、需要用户确认的 placeholder 或候选状态。 |
| 默认播放源 | 多来源实体中优先用于播放的来源。 |
| 外挂字幕 | 独立字幕文件。 |
| 在线字幕 | 通过外部服务搜索并下载到 XFVerse 缓存的字幕。 |
| 观影洞察 | 当前 1.0 Movie-only 的统计和画像功能。 |
| RC | Release Candidate，发布候选版本。 |
| GA | General Availability，正式公开版本。 |

避免使用：

- 用“删除影片”同时指软件记录删除和真实文件删除。
- 用“账号”描述当前仅存在的本地用户资料。
- 不把历史 Base64 编码称为加密；当前 DPAPI 保护应准确描述为当前 Windows 用户上下文的本机凭据保护。
- 在只有 ARM64 候选安装包、尚未完成 Phase 8.9 应用级 RC 时，直接宣称具体 Windows ARM64 系统版本已正式支持。
- 用“自动更新”描述手动下载安装器升级。

## 7. 截图规范

- 截图必须来自目标 RC，不能使用旧测试包或设计稿冒充正式界面。
- 所有截图必须脱敏，不显示真实媒体路径、WebDAV URL、账号、Token、API Key、头像或私有媒体清单。
- 优先使用专用演示数据；不得把用户真实数据库复制进截图环境。
- 截图文件名使用稳定章节语义，不使用“最终版2”“新截图”等临时命名。
- UI 文案变更后必须复核截图和替代文本。
- Phase 8.1 不开始大规模截图；截图集中在 RC 界面稳定后完成。

## 8. 交叉链接规则

- README 链接到安装说明、使用说明书、帮助中心、发布说明和第三方声明。
- 安装说明结尾链接到使用说明书“首次启动”和帮助中心“安装与启动”。
- 使用说明书各高风险章节链接到对应帮助文档，例如删除记录、扫描失败、在线字幕、数据恢复。
- 帮助文档每篇提供“相关完整章节”链接，避免复制大段功能说明。
- 发布说明链接到安装说明中的升级章节和 Known Issues。
- 软件设计说明书链接到功能矩阵和发布决策，不把阶段日志复制进正文。

## 9. 文档验收规则

每篇正式文档至少通过以下检查：

1. 页面、按钮、状态名与 RC 一致。
2. 操作前置条件和失败行为已写明。
3. 删除、缓存、卸载和数据保留语义准确。
4. Movie/TV 边界准确。
5. 外部服务被标记为必需或可选。
6. 没有私有路径、URL、凭据或真实样本。
7. 没有把 Deferred 能力写成正式支持。
8. 内部链接有效，重复内容有单一事实源。
9. 截图来自目标 RC 且已脱敏。
10. 与功能矩阵和发布决策一致。
