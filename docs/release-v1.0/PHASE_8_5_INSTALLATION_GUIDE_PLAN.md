## 文档定位

安装说明只负责部署生命周期：安装前准备、下载校验、安装、首次启动、升级、修复和卸载。

它不承担完整功能教程，也不收纳所有故障排查。

## 目标

重写 `docs/安装说明.md`，使普通 Windows 用户能够在不知道 .NET、WPF、SQLite 或仓库结构的情况下安全完成安装和维护。

## 目录要求

1. 文档适用版本。
2. 支持的 Windows 版本和 CPU 架构。
3. 硬件、磁盘和网络建议。
4. 下载正式安装包。
5. 校验 SHA-256。
6. 安装步骤。
7. Windows Defender / SmartScreen 与签名状态。
8. 首次启动会发生什么。
9. 第一次配置的最短入口。
10. 数据、缓存和日志保存位置及用途。
11. 从测试版或旧版本升级。
12. 覆盖安装 / 修复安装。
13. 卸载和用户数据保留。
14. 完全清除软件数据。
15. 安装或启动失败时转到帮助文档。
16. 安装说明不包含的内容及链接。

## 内容规则

- 正式用户不需要 Visual Studio 或 .NET SDK。
- 若使用自包含包，明确不需要另装 .NET Runtime。
- 明确区分 `win-x64` 与 `win-arm64` 两个独立安装包，并说明如何在 Windows“系统类型”中选择正确架构。
- 说明同版本跨架构覆盖安装会替换程序文件、清理另一架构原生目录并保留用户数据；不建议同时保留两套安装。
- 若无数字签名，准确说明未知发布者提示。
- 不要求用户禁用安全软件。
- 不默认建议删除数据库解决问题。
- 数据目录示例必须脱敏，不出现开发机用户名。
- 明确卸载软件不会删除本地媒体或 WebDAV 文件。
- 明确删除用户数据会清除软件数据库、设置和缓存，但仍不会删除媒体源文件。
- 所有安装器按钮名称必须与 RC 一致。

## 截图要求

- 只使用最终 RC 安装器。
- 截图不含开发机路径、用户名或私有数据。
- 每张截图只说明关键决策，不做装饰性堆叠。
- 若安装器 UI 很直观，可减少截图，以文字准确性优先。
